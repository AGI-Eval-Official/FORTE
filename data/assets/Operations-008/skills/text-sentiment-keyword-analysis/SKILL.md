# 文本情感分析与关键词归纳

## 概述

将非结构化的自然语言文本（用户评论、反馈、留言、帖子等）转化为结构化的情感与主题数据，输出情感极性、情感强度分数及核心关键词，便于后续统计分析、舆情监控、客户洞察或业务决策。

## 触发条件与适用场景

### 触发条件

当用户明确要求对文本进行情感判断、关键词提取或观点归纳时触发。典型用户表述包括但不限于：

- “分析这段评论的情感倾向”
- “提取这段话的关键词”
- “判断这条反馈是正面还是负面”
- “总结用户的满意点和不满点”
- “这段留言的情绪强度是多少”
- “帮我做文本情绪识别”

### 具体触发关键词

微博评论分析、推特情绪检测、公众态度判断、热点事件反馈、社媒舆论分析、用户评论挖掘、品牌形象分析、用户反馈归纳、负面评价发现、品牌口碑监测、企业声誉评估、竞品情感对比、客户情绪识别、留言情绪判断、邮件情感分析、优先处理负面情绪、服务体验提升、客服对话归纳、新产品反馈分析、消费者意见挖掘、营销策略调整、产品评价整理、市场需求关键词提取、用户调研总结、分析情感、情感分析、关键词提取、文本情绪识别、评论观点归纳、正负面判断、情感标签输出、置信度评分、反馈文本结构化。

### 典型业务场景

| 场景 | 说明 |
|------|------|
| 社交媒体监测 | 分析微博、推特、小红书、抖音等平台评论，了解大众对事件或产品的态度 |

| 市场调研 | 分析消费者对新产品的反馈，辅助调整营销策略 |
| 产品评价整理 | 对电商、应用商店的评论进行情感标注和主题归类 |
| 舆情监控 | 分析新闻评论、论坛帖子，发现舆论风险点 |
| 竞品对比 | 对比不同产品的用户评价情感分布与关键词差异 |

## 输入要求

| 参数 | 类型 | 是否必填 | 说明 |
|------|------|----------|------|
| `target_text` | 文本 | 必填 | 待分析的原始文本内容（评论、留言、邮件、问卷开放题等） |
| `analysis_dimension` | 文本 | 可选 | 分析维度或关注焦点（如"服务态度""产品质量"），不填则全面分析 |
| `output_fields` | 列表 | 可选 | 需要额外输出的字段（如"满意点""不满意点""服务体验描述"），不填则输出基础字段 |

**输入前提检查：**
- 若用户未提供 `target_text`，立即追问："请提供您希望分析的文本内容。"
- 若输入文本为空或仅含空白字符，返回异常提示（见「异常兜底」）
- 本技能**仅用于分析已有文本**，不生成评论或好评内容

## 执行步骤

### Step 1：接收与校验输入

接收 `target_text`，检查是否满足以下条件：
- 非空文本
- 非纯符号/表情/链接（无实质语义内容）
- 非生成请求（如"帮我写一篇好评"）

若不满足，执行「异常兜底」流程。

### Step 2：文本清洗

对 `target_text` 进行预处理：
- 去除表情符号、特殊字符（保留标点）
- 去除多余空格与换行
- 去除明显广告语句或无关链接
- 保留核心语义内容，不改变原意

### Step 3：情感分析

基于清洗后文本进行语义判断：

- **情感极性**：判断为 `正面` / `负面` / `中性`
  - 若文本同时含正负面信息，以**整体倾向**为主
  - 模棱两可或无法判断时标记为 `中性`
- **情感强度**：给出 -5 到 +5 的整数分数
  - `-5`：极度负面（强烈愤怒、严重投诉）
  - `-1 ~ -3`：轻度至中度负面
  - `0`：中性或混合
  - `+1 ~ +3`：轻度至中度正面
  - `+5`：极度正面（高度赞扬、强烈推荐）

### Step 4：关键词归纳

从清洗后文本中提取 2-5 个最能代表核心主题的名词或短语：
- 优先提取**具体、有业务价值**的词（如"续航差""屏幕显示好""服务态度"）
- 避免虚词、通用词（如"很好""问题""东西"）
- 若文本同时含正负面信息，关键词中体现**正负两方面主题**
- 短文本（一句话）：提取 2-3 个关键词，整体判断
- 长文本（段落/多句）：提取 3-5 个关键词，可覆盖多个主题

### Step 5：附加字段输出（按需）

若用户通过 `output_fields` 指定或在对话中明确要求，额外输出以下字段：

| 字段名 | 说明 |
|--------|------|
| `最满意点` | 用户最满意的 1-2 个方面 |
| `最不满意点` | 用户最不满意的 1-2 个方面 |
| `服务体验描述` | 对服务的具体评价 |
| `质量问题描述` | 具体描述的故障或问题 |
| `推荐意向` | 是否明确推荐（强烈推荐 / 推荐 / 中立 / 不推荐） |

### Step 6：返回结构化结果

以 JSON 格式输出分析结果（见「输出要求」）。

## 核心逻辑规则

- 若文本同时包含正面和负面信息，以整体倾向为主，关键词中体现正负两方面主题
- 短文本（一句话）优先整体判断；长文本（段落）可提取多个主题标签
- 不确定或模棱两可的文本标记为中性，强度为 0，关键词可标注"混合/模糊"
- **不生成原文之外的总结摘要**，仅输出标签和关键词
- 本技能**不用于生成文本**，仅分析已有文本

## 输出要求

### 基础输出格式（JSON）

```json
{
  "情感极性": "正面 | 负面 | 中性",
  "情感强度": 0,
  "提及的关键主题": ["关键词1", "关键词2", "关键词3"]
}
```

### 含附加字段的完整输出格式

```json
{
  "情感极性": "正面 | 负面 | 中性",
  "情感强度": 0,
  "提及的关键主题": ["关键词1", "关键词2"],
  "最满意点": "...",
  "最不满意点": "...",
  "推荐意向": "推荐 | 强烈推荐 | 中立 | 不推荐",
  "服务体验描述": "...",
  "质量问题描述": "..."
}
```

**输出验收标准：**
- `情感极性` 必须为三选一：`正面` / `负面` / `中性`
- `情感强度` 必须为 -5 到 +5 的整数
- `提及的关键主题` 为 2-5 个具体名词或短语，无虚词
- 附加字段仅在用户明确要求时输出，不涉及时填写 `"不涉及"`
- 输出结果不包含原文之外的推断或摘要

## 异常兜底

| 异常情况 | 处理方式 |
|----------|----------|
| `target_text` 为空 | 回复："请提供您希望分析的文本内容，我将为您进行情感分析和关键词归纳。" |
| 文本仅含表情/符号/链接，无实质语义 | 返回 `{"情感极性": "中性", "情感强度": 0, "提及的关键主题": ["内容不足，无法提取"]}` 并提示用户补充文本 |
| 用户请求生成文本（非分析） | 回复："本技能仅用于分析已有文本，不生成评论内容。请提供您希望分析的原始文本。" |
| 文本语言非中文 | 支持多语言分析，按原文语言进行情感判断，关键词以原文语言输出 |
| 文本过长（超过 5000 字） | 提示用户："文本较长，将对全文进行整体情感判断，关键词提取覆盖主要段落。" 并继续执行 |

## 示例

### 正例 1：混合情感文本

**输入：**
```
这款手机电池续航太差了，才用半天就没电，不过屏幕显示效果还不错。
```

**输出：**
```json
{
  "情感极性": "中性",
  "情感强度": -1,
  "提及的关键主题": ["续航差", "屏幕显示好"]
}
```

### 正例 2：含附加字段需求

**输入：**
```
小米SU7的外观设计非常惊艳，内饰质感也超出预期，但冬季续航缩水严重，充电速度也比较慢，整体来说还是推荐的。
```

**输出（用户要求输出满意点和不满意点）：**
```json
{
  "情感极性": "正面",
  "情感强度": 2,
  "提及的关键主题": ["外观设计好", "内饰质感佳", "冬季续航差", "充电速度慢"],
  "最满意点": "外观设计惊艳，内饰质感超出预期",
  "最不满意点": "冬季续航缩水严重，充电速度慢"
}
```

### 正例 3：强烈负面文本

**输入：**
```
售后服务态度极差，打了三次电话没人接，问题拖了两周还没解决，再也不会买这个品牌了。
```

**输出：**
```json
{
  "情感极性": "负面",
  "情感强度": -4,
  "提及的关键主题": ["售后服务差", "响应慢", "问题未解决"]
}
```

### 反例（错误用法）

**输入：**
```
帮我写一篇产品好评。
```

**处理说明：** 本技能不用于生成文本，仅用于分析已有文本。回复用户："本技能仅用于分析已有文本，不生成评论内容。请提供您希望分析的原始文本。"
