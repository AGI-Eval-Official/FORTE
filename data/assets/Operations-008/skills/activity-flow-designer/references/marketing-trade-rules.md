# 营销交易类活动规则 (marketing_trade)

> 场景标识：`marketing_trade`
> 适用场景：线上（限时折扣、满减、优惠券、秒杀）；线下（门店促销、快闪店、市集）

---

## 一、行业评分标准

### 1.1 评分维度与权重

| 评分项 | 权重 | 评分细则 | 满分 |
|-------|------|---------|------|
| **价格规则清晰度** | 20% | 折扣/优惠逻辑是否无歧义，叠加顺序是否明确可执行 | 20 |
| **库存管理完备性** | 15% | 总库存/分时段库存/动态补充/预警机制是否完整 | 15 |
| **线上线下一致性** | 15% | 线上价格与线下价格、权益是否统一，核销链路是否打通 | 15 |
| **用户操作路径** | 15% | 从入口到领奖的完整链路是否闭环，断点是否有兜底 | 15 |
| **风控策略有效性** | 10% | 刷单/黄牛/囤货/薅羊毛等风险是否有对应防控措施 | 10 |
| **异常预案覆盖度** | 10% | 售罄/系统故障/现场突发/退款等场景是否有处理方案 | 10 |
| **合规性** | 15% | 价格法、广告法、消费者权益保护法、明码标价规定是否满足 | 15 |

### 1.2 评分等级

| 等级 | 分数范围 | 含义 | 后续动作 |
|-----|---------|------|---------|
| S | 90-100 | 可直接执行，无需额外补充 | 输出最终文档 |
| A | 80-89 | 整体完善，少量细节需确认 | 与用户确认2-3个细节后定稿 |
| B | 70-79 | 框架完整，部分规则需补充 | 标注缺失项，请用户补全后修订 |
| C | 60-69 | 存在明显缺失，需要较大修订 | 列出问题清单，建议重写薄弱模块 |
| D | <60 | 不合格，需重新设计 | 给出具体改进方向，建议重新走SOP流程 |

### 1.3 扣分速查表

| 问题类型 | 扣分 | 说明 |
|---------|------|------|
| 价格逻辑矛盾 | -5~-10 | 叠加规则冲突、最终价格不明确 |
| 库存规则缺失 | -5 | 无总库存/分时段/预警任一规则 |
| 线上线下权益不一致 | -5~直接D | 同一活动线上线下价格/权益矛盾 |
| 用户路径断点 | -3 | 领奖/核销环节无兜底方案 |
| 风控空白 | -5 | 无任何反作弊/反刷单规则 |
| 售罄处理缺失 | -3 | 无售罄后的用户提示和替代方案 |
| 合规风险点 | -5~直接D | 违反价格法/广告法/明码标价规定 |
| 退换规则不清 | -3 | 已使用奖励的退换逻辑未定义 |

## 二、专属违禁词库

### 2.1 价格表述违禁词

| 违禁词 | 原因 | 替代建议 |
|-------|------|---------|
| 最低价 / 史低价 / 史低 | 无法证实，违反《价格法》/广告法 | "限时特惠""活动价" |
| 全网最低 | 无法证实，涉嫌虚假宣传 | "参考价XX折起" |
| 原价XXX元（虚构） | 虚构原价违法 | 使用真实原价或"建议零售价" |
| 仅限今日最后X件 | 制造虚假紧迫感 | "限量X份，售完即止" |
| 必买 / 不得不买 | 强制性表述 | "推荐""值得入手" |
| 零利润 / 亏本赚吆喝 | 可能构成不正当竞争（倾销） | "超值优惠""诚意回馈" |
| 价格欺骗（划线价虚高） | 虚假折扣，违反《禁止价格欺诈行为的规定》 | 划线价应为真实近30天最低价 |

### 2.2 奖励表述违禁词

| 违禁词 | 原因 | 替代建议 |
|-------|------|---------|
| 100%中奖 | 概率性奖励不可保证 | "有机会获得""抽奖赢取" |
| 价值XXX元（虚高） | 虚高标价涉嫌欺诈 | 按实际市场公允价值标注 |
| 终身有效 / 永久 | 承诺过当 | "长期有效""有效期至XXXX" |
| 人人有份 | 若实际有门槛则属误导 | 明确参与条件后使用 |
| 送现金 / 返现（非金融资质） | 可能涉及非法集资/洗钱风险 | "返券""返积分""赠送等值礼品" |

### 2.3 线下活动专属违禁词

| 违禁词 | 原因 | 替代建议 |
|-------|------|---------|
| 先到先得（无数量） | 信息不完整 | "限量XX份，先到先得" |
| 现场免费送（无限制） | 可能引发秩序混乱 | "凭券领取，每日限量" |
| 保证有货 | 无法100%保证 | "数量有限，先到先得" |
| 排队即可领（无条件） | 信息不完整 | "排队领取，每日限量XX份" |

### 2.4 敏感规避

| 敏感领域 | 规避原因 | 处理方式 |
|---------|---------|---------|
| 价格歧视 | 不同用户不同价可能违规 | 差异化定价需有合理依据并明确说明 |
| 虚假促销（先涨价后打折） | 违反《价格法》及平台规则 | 折扣基于真实原价，保留价格变动记录 |
| 强制捆绑销售 | 侵犯消费者选择权 | 捆绑套餐外必须提供单品购买选项 |
| 不明码标价 | 违反《价格法》第十三条 | 所有商品/服务必须标注清晰价格 |
| 积分/虚拟资产变现 | 可能涉及金融监管 | 避免直接兑换现金，改为实物/券/服务 |

## 三、输出文档结构参考

### 3.1 文档章节框架

基本信息 → 参与规则（资格/入口/流程） → 价格/优惠规则（优惠类型/叠加/一致性） → 奖励规则（内容/条件/使用规则/核销） → 库存管理（总量/分时段/预警/补货） → 风控规则（反作弊/黄牛防控/异常订单） → 异常处理（退款/发放失败/系统故障/现场预案） → 合规审查（价格法/广告法/平台规则） → 附件（FAQ/话术/物料清单）

### 3.2 价格规则明细表头

| 规则项 | 说明 |
|-------|------|
| 优惠类型 | 折扣/满减/优惠券/秒杀 |
| 优惠条件 | 触发门槛 |
| 优惠力度 | 具体优惠内容 |
| 叠加规则 | 是否可与其他优惠同享 |
| 有效期 | 优惠生效时间段 |
| 适用范围 | 商品/用户/渠道限制 |
| 线下适用 | 门店是否同步 |
