# 运营数据处理 Skill

## 概述

本 Skill 提供运营数据的处理与可视化能力（清洗、合并、格式转换、可视化等），支持按需执行指定环节或完整全流程。

## 工作流程

### 确定执行范围

根据用户指令判断需要执行的环节：

- **未指定环节** → 执行完整流程：`数据汇总` → `数据清洗` → `数据分析` → `可视化`
- **指定了某个或某些环节** → 仅执行指定的环节，跳过其他环节
- **指定了输出目标**（如"只需要汇总表"）→ 自动推断所需环节并执行

#### 数据质量前置检查

**无论用户是否指定了「数据合并」或「数据清洗」环节，在进入「数据分析」或「可视化」之前，必须先对数据进行快速探查。**

探查内容包括：行数/列数、各字段类型、空值率、重复行数量、日期/金额等关键字段的格式一致性、**脏数据/测试数据特征（用户ID是否含测试标识、姓名是否为测试用户、联系方式是否为虚拟号段、ID编号序列是否存在孤立异常值等）**。

探查后按以下规则决策：

| 探查结果 | 处理方式 |
|---------|---------|
| 数据质量良好（无缺失、无重复、格式统一） | 直接进入分析/可视化，无需清洗 |
| 存在可自动修复的问题（格式不统一、含单位符号等） | 自动执行**轻量修复**（仅限无损操作：去除单位符号、统一日期格式、标准化空值变体等），并在输出中说明；**若涉及数据变更（删除行/填充值/去重等有损操作），则必须进入正式的「数据清洗」环节并按对应规则文件输出干净数据表+清洗报告** |
| 存在需确认的问题（高缺失率>30%、业务不合理值、去重歧义等） | 暂停并告知用户，给出建议方案供选择；用户确认后再继续 |

**若实际执行了数据清洗或数据合并操作，必须按照对应规则文件输出结果文件（干净数据表 + 清洗报告 / 汇总表 + 问题清单），不得省略。**

详细规则参见：[reference/data-aggregation-rules.md](reference/data-aggregation-rules.md)

### 环节二：数据清洗

详细规则参见：[reference/data-cleaning-rules.md](reference/data-cleaning-rules.md)

### 环节三：数据分析

根据用户指令判断分析方向：

- **用户指定了具体分析场景**（如用户画像、RFM、留存、分层打标等）→ 调用对应的场景 skill 执行分析
- **未指定分析场景** → 执行常规数据分析（模型自主探索数据并进行分析，不含可视化）

#### 可调用的场景 Skill

| 分析场景 | 触发关键词示例 | 对应 Skill |
|---------|--------------|-----------|
| 用户运营 | 用户画像、RFM、留存、流失、复购、生命周期、分群、分层、打标、触达名单、圈人 | `user-operations` |
| 流量分析 | 渠道UV、CTR/CPUV/CPM/CPC 计算、付费/免费流量拆分、渠道/平台汇总 | `traffic-analysis` |

> 场景 Skill 可扩展。当用户的分析需求属于某个已注册的场景时，调用该 Skill；不属于任何已注册场景时，使用「常规数据分析」作为默认方案。

### 环节四：可视化

详细规则参见：[reference/data-visualization-rules.md](reference/data-visualization-rules.md)

## 输出约束

**当用户指令对输出文件有明确要求时（如"只要汇总表"、"输出一个csv"、"给我图表"），仅交付用户要求的文件，不得附带任何用户未要求的过程文档（如清洗报告、问题清单、分析摘要等）。过程文档可在工作过程中生成并用于上下游环节传递，但最终交付前必须删除。**

**用户未对输出做明确要求时**，按各环节规则文件的默认输出约定执行。

## 输出路径

**用户指定了输出路径时，按用户指定的路径保存。未指定时，所有输出文件统一保存到当前目录下的 `结果/` 文件夹中。**
