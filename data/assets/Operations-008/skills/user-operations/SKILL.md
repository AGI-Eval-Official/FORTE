# 用户运营 Skill

## 概述

围绕**「拉新、促活、留存、转化」**四大核心指标构建用户运营分析体系。根据用户指令和数据特征，自动选择可执行的分析模块并输出结果。

## 与其他 Skill 的协作

本 Skill **专注于用户运营分析本身**，以下操作需调用 `operations-data-processing` skill：

| 操作 | 说明 |
|------|------|
| 数据合并/汇总 | 多表合并、数据汇总 |
| 数据清洗 | 缺失值处理、去重、格式统一、脏数据过滤 |
| 可视化/图表 | 图表生成、仪表盘、可视化报告 |

> 若数据探查结果显示输入数据已经过清洗且质量良好，可直接进入本 Skill 分析环节。

## 工作流程

### 1. 确定执行范围

- **用户指定了 `分析类型`** → 仅执行指定模块（如"只做 RFM"、"做留存+流失"），跳过其他模块
- **未指定 `分析类型`** → 进入自动检测模式，按顺序检测数据支持哪些模块并全部执行

### 2. 数据质量前置检查

**本步骤优先调用 `operations-data-processing` skill 执行数据探查和清洗。** 典型流程：原始数据 → `operations-data-processing`（探查+清洗）→ 干净数据 → 进入用户运营。

仅在以下情况时由本 Skill 自行执行轻量探查：
- 用户明确指定跳过清洗步骤
- 输入数据已明确来自 `operations-data-processing` 清洗输出

自行探查内容（行数/列数/字段类型/空值率/重复行/格式一致性/脏数据特征）及决策规则：

| 探查结果 | 处理方式 |
|---------|---------|
| 质量良好 | 直接进入分析 |
| 可自动修复 | 执行轻量修复（无损操作）并说明；涉及有损操作时暂停告知用户，建议调用 `operations-data-processing` 做正式清洗 |
| 需确认 | 暂停并给出建议方案供选择 |

### 3. 用户可指定的参数

| 参数 | 说明 | 默认行为 |
|------|------|---------|
| `用户ID字段` | 用户唯一标识字段 | 自动识别 |
| `分析类型` | `拉新` / `促活` / `留存` / `转化`，或细粒度：`画像` / `RFM` / `打标` / `分群` / `留存` / `流失` / `复购` / `生命周期` | 自动选择可执行的模块 |
| `时间范围` | 分析时间窗口 | 全量时间范围 |
| `行为事件` | 分析依据的用户行为 | 自动识别关键行为字段 |
| `分组维度` | 分群依据 | 按可用维度自动分组 |
| `字段名` | 输出列使用的字段名 | 与输入文件原始字段名保持一致 |
| `单元格格式` | 数值/日期/百分比显示格式 | 与输入文件保持一致（见下方默认值） |
| `输出格式` | `xlsx` / 嵌入报告 / 纯文本摘要 | 默认 xlsx 多工作表 |

#### 单元格格式默认值

- 人数/计数类：整数，无千分位
- 金额类：保留 2 位小数
- 百分比/比率/占比：1~2 位小数（留存率 1 位，其他 2 位），带 `%`
- 天数/时长：保留 1 位小数
- 统计量（均值/中位数）：保留 2 位小数
- 日期：`YYYY-MM-DD`
- 其他数值：保留 2 位小数

## 四大核心模块

按顺序检测数据是否支持该模块，支持则执行，不支持则跳过。详细规则见各 reference 文件。

| 序号 | 核心指标 | 规则文件 | 所需最低条件 |
|------|---------|---------|------------|
| 1 | **拉新 — 用户画像** | [Data_acquisition_rules.md](reference/data-acquisition-rules.md) | 有用户标识字段 |
| 2 | **促活 — RFM · 规则打标 · 分群** | [Data_activation_rules.md](reference/data-activation-rules.md) | 有 ≥1 个数值型字段可用于频次/价值计算 |
| 3 | **留存 — 留存 · 流失 · 生命周期** | [Data_retention_rules.md](reference/data-retention-rules.md) | 有时间戳字段 + 行为事件 |
| 4 | **转化 — 复购 · 价值评估** | [Data_conversion_rules.md](reference/data-conversion-rules.md) | 有时间戳字段 + 行为事件或金额字段 |

> 无论最终执行了哪些模块，必须在分析摘要中列出实际执行的模块清单和跳过的模块及原因。

## 公共规则

### 数据前提检查

进入各模块前仅检查该模块所需前提条件，不强制全部满足——缺少必要字段时跳过该模块并在摘要中说明。

| 前提 | 涉及模块 | 不满足时的处理 |
|------|---------|--------------|
| 用户标识字段 | 全部 | 必须确认候选字段；无法识别时建议切换至常规数据分析 |
| 时间戳/日期字段 | 留存、生命周期、流失、转化 | 必须确认时间字段；无则跳过时序类模块 |
| 金额/价值型字段 | RFM-M 维度、转化 | 给出降级方案供选择 |
| 行数 >100 独立用户 | 全部 | 执行但标注"样本量不足，结论仅供参考" |

### 校验与异常处理

字段校验规则、交叉一致性校验、异常默认处理、必须确认的场景等公共约束，详见 [Data_common_rules.md](reference/data-common-rules.md)。

## 输出约定

- 用户有明确要求时严格按指令执行
- 未指定时：输出 xlsx，按核心指标分工作表（拉新/促活/留存/转化），附分析摘要（含实际执行的模块清单和跳过的原因）

## 输出路径

用户指定路径时按指定保存，未指定时统一保存到当前目录下的 `结果/` 文件夹。
