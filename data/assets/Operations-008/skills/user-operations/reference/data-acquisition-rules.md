# 拉新：用户画像

## 定位

围绕「拉新」目标，对用户群体进行全面画像刻画，帮助理解**用户从哪里来、长什么样、具备什么特征**，为获客策略和渠道优化提供数据支撑。

---

## 1. 画像维度构建规则

用户画像**不预设固定维度**，而是根据数据中实际可用的字段动态构建。以下为各维度所需的最低字段及构建方式：

| 维度 | 所需字段（至少具备其一即可构建） | 缺失时的处理 |
|------|---------------------------|------------|
| **规模概览** | 用户标识字段（必有，否则前提检查已终止） | 必定可构建：总用户数、有行为记录的用户数 |
| **来源分布** | 渠道/来源/推广活动/首次访问平台等分类型字段 | 有哪些来源字段就展示哪些的取值分布；无来源字段时跳过并在摘要中说明 |
| **属性分布** | 地域/平台/性别/会员等级/年龄段等其他分类型字段 | 有哪些分类字段就展示哪些的取值分布；无分类字段时跳过 |
| **行为特征** | 任意数值型字段（浏览量/点击/花费/订单数等） | 有哪些数值字段就统计哪些，无数值字段时跳过该维度并在摘要中说明 |
| **时间特征** | 日期/时间戳字段（注册时间/首次访问时间等） | 有时间字段则计算：新增趋势、首次行为时间分布、活跃时段分布；无时间字段时跳过 |
| **价值分布** | 金额/价格/单价等数值字段 | 有金额字段则做分档（高/中/低）和频次分布；无金额字段时可改用频次作为价值代理指标，或跳过 |

> **构建原则**：数据中有多少字段就画多少维度的像，不臆测不编造。某维度因缺少必要字段无法构建时，**跳过该维度并在分析摘要中说明原因**，不使用默认值或插值填充。

### 1.1 新用户专项分析

当数据中可识别新用户时（如存在注册时间、首次访问时间等字段），额外输出以下内容：

| 分析项 | 说明 |
|--------|------|
| 新增趋势 | 按日/周/月的新增用户数走势（根据数据时间跨度自动选择粒度） |
| 新老用户占比 | 新用户 vs 老用户的数量及占比 |
| 各渠道新用户质量 | 不同来源的新用户在核心指标上的表现对比（如首日留存、首次转化率等，视数据可用性而定） |

---

## 2. 输出要求

- 输出为汇总级统计表格（非逐用户明细，除非用户要求）
- 数值型指标输出：均值/中位数/分位数（P25/P50/P75/P90）
- 分类型指标输出：各取值的用户数及占比
- **不生成臆测性描述**（如"该用户群体偏好高端消费"），只呈现数据事实
