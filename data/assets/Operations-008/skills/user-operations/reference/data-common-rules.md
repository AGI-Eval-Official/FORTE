# 公共规则：校验 · 异常处理 · 确认场景

本文件包含用户分析四大模块（拉新/促活/留存/转化）通用的约束规则，所有模块的输出均须遵守。

---

## 1. 字段-内容常识校验【强制】

**在生成任何输出之前，必须对计算结果进行以下校验。** 本节规则优先于各模块内部的格式约定——当本节的校验结果要求修正某值时，以本节为准。

### 1.1 字段类型-内容匹配校验

| 输出内容的业务含义 | 必须满足的约束 | 不满足时的处置 |
|-------------------|---------------|----------------|
| **人数/用户数/计数/频次** | **必须为整数**（不可出现小数）；且 ≥ 0 | 四舍五入取整；若为负数则标记为「⚠️ 计算异常」并保留原值 |
| **占比/比例/百分比** | 必须 ≥ 0 且 ≤ 100（或 0~1 小数形式）；各组占比之和应 = 100%（±1% 浮动容差） | 超出范围标记为「⚠️ 数值异常」；总和不等于100%时检查是否有 rounding error 并在备注中说明 |
| **金额/单价/价值** | 一般 ≥ 0（除非业务允许退款等负值场景）；精度不超过 2 位小数 | 负数时标注「含负值」；超精度时四舍五入到 2 位 |
| **天数/间隔/时长** | 一般 ≥ 0；未来日期间隔可为负数但需标注 | 负数标注含义（如"距今天数为负表示未来日期"） |
| **留存率/转化率** | 0% ~ 100%（或 0~1）；Cohort 首日留存率通常应接近 100% | 首日留存率明显偏离 100%（如 <80%）时标记「⚠️ 可能存在数据或定义问题」并提示用户复核 |
| **R/F/M 分数** | 必须为正整数（分箱档位 1-N） | 出现非整数或 ≤0 时回溯分箱逻辑 |
| **层级标签** | 必须为字符串/文本类型 | 若出现数值型标签则转为对应层级的名称文本 |

### 1.2 交叉一致性校验

| 校验项 | 规则 | 异常时处置 |
|--------|------|----------|
| 各层级/分组人数之和 vs 总人数 | 各组人数之和应等于有效总人数（±1 容差允许取整误差） | 不一致时以总人数为准，调整最后一组的数值使其平衡，并在问题清单中记录调整 |
| 占比之和 | 所有占比列之和应为 100%（±1% 容差） | 同上，调整最后一行 |
| RFM 各组人数 vs 总用户数 | RFM 交叉表中所有组合的人数之和 = 总用户数 | 不一致时标记为「⚠️ 分组覆盖不全或有重叠」 |
| 留存矩阵 Cohort 合计 | 每行（同一 Cohort）的留存人数递减或不增 | 若出现逆增长，标记「⚠️ 数据异常」但不阻断 |

> **执行时机**：所有计算完成、输出写入文件之前执行。校验不通过时不阻断流程，但在输出中添加醒目的 ⚠️ 标记及原因说明。

## 2. 异常情况默认处理

以下异常情况**不询问用户**，直接按此表处理：

| 异常场景 | 处理方式 |
|---------|---------|
| RFM 某分箱组用户数为 0 | 保留该组（显示为空），不在交叉表中删除，在摘要中备注 |
| 留存率计算中某 Cohort 分母为 0 | 显示为"—"，不做插值或跳过 |
| 聚类算法不收敛或轮廓系数极低（<0.3） | 回退至基于 RFM 的分群方案，并在摘要中说明原因 |
| 某用户分组仅含 1 个用户 | 保留该组但标注"样本量极低，统计结论不可靠" |
| 时间字段存在未来日期（如 2028 年） | 视为数据录入错误，剔除该行并在问题清单中记录 |

## 3. 必须确认的场景

以下情况**必须暂停并向用户确认**。确认时**必须同时给出建议的替代方案**：

| 场景 | 触发条件 | 确认时应给出的替代方案 | 用户可如何避免被询问 |
|------|---------|---------------------|---------------------|
| 用户标识不确定 | 数据中存在多个可能的标识字段 | 列出候选字段并说明推荐理由 | 提前指定 `用户ID字段` |
| 行为事件含义不明 | 无法确定哪个字段代表目标行为 | 列出候选事件字段及各自的数据特征，推荐最可能的一个 | 提前指定 `行为事件` |
| Cohort 起点不明 | 留存分析中无法确定应以哪个事件作为首触点 | 给出推荐并列出备选 | 指定 Cohort 起点事件 |
| 流失判定标准不明 | 无法确定"多少天无行为算流失" | 给出基于数据的建议值 + 行业常用值参考 | 指定流失天数阈值 |
| 数据不适合用户分析 | 数据无用户标识字段或无法建立用户级别视角 | 建议切换至 `operations-data-processing` 的常规数据处理 | 切换至常规数据处理 |
