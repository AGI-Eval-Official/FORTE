# 促活：RFM 分析 · 业务规则打标 · 用户分群

## 定位

围绕「促活」目标，通过 RFM 评分、业务规则打标、用户分群等方法**识别不同价值层次和活跃程度的用户群体**，为精细化运营和精准触达提供分层依据。

本文件包含三个相互关联的分析方法：

| 方法 | 适用场景 | 默认行为 |
|------|---------|---------|
| **RFM 分析** | 探索性分析，无先验规则时优先使用 | 基于数据分布自动分箱评分 |
| **业务规则打标法** | 有明确运营策略/阈值/分层标准时使用 | 按优先级逐条匹配打标签 |
| **用户分群** | 需要将用户划分为可运营的群体时使用 | 优先基于 RFM 或规则打标结果分群 |

---

## 1. RFM 分析

### 1.1 R/F/M 定义

| 维度 | 含义 | 默认计算方式（用户未指定时） |
|------|------|---------------------------|
| R（Recency） | 最近一次行为距今天数 | 取最近一次行为记录的日期距当前的天数 |
| F（Frequency） | 行为发生频次 | 统计时间窗口内的行为次数 |
| M（Monetary） | 行为金额/价值 | 取累计花费/累计订单金额等数值型字段的求和 |

> **用户可以自定义 R/F/M 的含义和计算方式**（如 R 改为"最近一次下单"、M 改为"访问时长总和"）。用户指定时严格按用户定义执行。

### 1.2 分箱规则

- **用户未指定分箱方式** → 使用五分位数（每维度分为 5 档：1-5 分，5 分最优）
- **用户指定了分箱方式**（如等宽/等频/自定义阈值）→ 严格按用户指定执行
- **用户指定了分箱数量** → 按用户指定的档位数执行
- **M 维度为可选**：有金额字段则做完整 RFM，无金额字段时降级为 RF 两维分析

### 1.3 输出

- 每个 RFM 组的用户数、占比、以及各组在各维度上的平均值
- RFM 组合交叉表（如 "高价值活跃用户" = R5F5M5 的用户群体）
- 各组与整体平均水平的对比

## 2. 业务规则打标法

### 2.1 方法概述

与 RFM 分箱评分法不同，业务规则打标法不依赖统计分箱，而是基于**业务经验预设的阈值条件**，按优先级从高到低逐条匹配，命中即停止并打上对应标签。

适用场景：用户分层、运营触达圈人、活动人群筛选等需要**可解释、可直接对接业务系统**的场景。

> **与 RFM 的选择原则**：当任务描述或上下文明确指定了分层规则/阈值时，使用本方法；未指定且无先验知识时，优先使用 RFM 探索性分析。

### 2.2 规则定义方式

分层规则由一组 **「优先级 → 条件 → 标签」** 三元组构成：

| 要素 | 说明 | 示例 |
|------|------|------|
| 优先级 | 规则的匹配顺序，数字越小越优先 | P1 > P2 > P3 ... |
| 条件 | 一个或多个字段的判定表达式（支持 AND / OR） | 近期金额 ≥ 阈值 且 购买次数 ≥ 阈值 |
| 标签 | 命中该规则时赋予用户的层级名称 | 重要价值用户 |

**规则来源（按优先级选取）：**

1. **用户明确指定** → 严格按用户提供规则执行
2. **任务文档 / Skill 内置规则** → 按文档中定义的规则执行
3. **均未提供** → Agent 基于数据特征自主推断合理阈值，并在输出中标注"规则为自动推断"

### 2.3 执行流程

```
对每位用户，按优先级从高到低逐条判断：
├── 命中 P1 规则 → 打标签 P1，结束该用户判断
├── 未命中 → 尝试 P2
│   ├── 命中 → 打标签 P2，结束
│   └── 未命中 → 尝试 P3
│       ...
└── 所有规则均未命中 → 归入兜底层级（最低优先级）
```

**关键约束：**
- 每位用户**有且仅有**一个层级标签
- 条件中的字段和阈值**不限定具体名称**，根据实际数据字段灵活适配
- 支持的条件运算符：`≥` `≤` `>` `<` `=` `!=` `在区间内` `包含` `为空` 等

### 2.4 输出

- 每位用户的层级标签（新增一列附加于原始数据）
- 各层级的统计汇总：人数、占比、各关键指标均值
- 若用于触达场景：按层级排序的名单（仅含标识字段 + 层级）

### 2.5 输出格式约束【强制】

当用户指令中对输出格式有明确要求时，严格按用户要求执行。未明确指定时，遵循以下默认规范：

#### 2.5.1 分层结果文件
- 内容：包含**全部原始字段** + 新增的「用户层级」列
- **不得擅自添加**原始数据中不存在的列（如「距基准日天数」「距离今天数」等计算中间量/辅助字段），除非用户明确要求
- **不得修改原始字段的列名**（如将「最近消费日期」改为「最近购买日期」），保持与输入数据完全一致
- 层级列命名：使用「用户层级」；值格式默认为不带前缀的纯中文（如「重要价值用户」），除非用户指定了带前缀格式（如P1/P2/P3/P4/P5）
- 排序：按层级优先级排列（P1在前，P5在后），同层级内默认按用户ID升序排列

#### 2.5.2 统计汇总文件
- 列名约定：「用户层级」、「人数」（或「用户数」）、「占比(%)」、「平均近30天消费金额(元)」（或类似含义的中文列名）
- 行顺序：按 P1→P2→P3→P4→P5 固定顺序排列
- 占比计算：**以有效数据总人数为分母**（即经过脏数据过滤后的总人数），所有层级占比之和应等于 100%
- 数值精度：**金额类保留2位小数，百分比固定保留2位小数（如16.00%）**

#### 2.5.3 触达名单文件
- 仅包含用户要求的列（通常为：用户ID、姓名、手机号、用户层级）
- **仅包含经过过滤后的有效用户**，不含已剔除的测试/脏数据
- 排序：按层级优先级排列（P1在前，P5在后），同层级内**默认按输出文件中第一列（通常为用户ID）升序排列；若输出不含用户ID则按第二列升序**

## 3. 用户分群

### 3.1 分群方法

| 方法 | 适用场景 | 默认行为 |
|------|---------|---------|
| 基于规则 | 用户指定了明确的分群条件（如"花费>1000的高价值用户"） | 严格按用户规则执行 |
| 基于 RFM | 无明确规则时 | 使用 §1 的 RFM 分箱结果作为分群基础 |
| 基于行为聚类 | 数据中有丰富的行为维度字段 | 使用 K-Means 等方法自动聚类，簇数由轮廓系数或肘部法则确定 |

### 3.2 分群结果输出

- 每个群体的用户数、占比
- 各群体在核心指标上的均值对比（群体 vs 整体）
- 各群体的典型特征描述（基于数据统计，不做主观推断）
