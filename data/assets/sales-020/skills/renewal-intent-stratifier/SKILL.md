# 续费意向分层与跟进话术生成技能

## 核心原则

续费沟通的本质是**帮助客户做决策**，而非推销。所有话术以服务进展为切入点，以价值感知为核心，以解决顾虑为落脚点。语气须专业、尊重、非施压。

---

## 职责边界

**本技能负责：**
- 意向分层的判断逻辑与语义定义（P0-P4）
- 各层沟通策略框架（渠道、目标、话术重点）
- 结构化话术的生成规则与质量标准
- 通用异议处理方法论
- 批量处理的执行流程

---

## 1. 输入规范

### 1.1 必要字段
- 服务对象姓名
- 服务类型（如科目、产品线、服务项目等）
- 历史续费意向（文本）
- 客户主要顾虑（文本）
- 最近一次服务反馈（文本）

### 1.2 可选字段
- 客户基本信息（如年级、职业、所在地等场景相关属性）
- 剩余服务量（如剩余课时、剩余次数、合同到期日等）
- 服务频次
- 最近联系状态
- 历史沟通记录

### 1.3 缺失字段处理
- 缺少"服务对象姓名/服务类型"：先输出缺失提示，再使用降级话术（以"您"/"这位客户"代替）
- 缺少"服务反馈"：先输出缺失提示，话术中以"近期服务状态"作为替代表述，并标注"[服务反馈缺失，请补充]"
- 缺少意向相关字段：默认进入 P2，给出探询型开场白

---

## 2. 意向分层定义（P0-P4）

| 层级 | 名称 | 核心特征 | 沟通目标 |
|------|------|----------|----------|
| P0 | 强烈意向 | 客户主动咨询续费方案、价格或购买方式，接近成交 | 缩短决策路径，快速成交 |
| P1 | 价格敏感 | 认可服务价值，但对费用有顾虑 | 先价值后价格，降低支付阻力 |
| P2 | 效果犹豫 | 对服务效果或配合度存疑 | 先共情解疑，再引导续费 |
| P3 | 沉默失联 | 不拒绝但低响应，多次联系无实质回复 | 恢复响应，不强压成交 |
| P4 | 明确拒绝 | 当前明确表示不续费 | 维护关系，保留未来召回机会 |

---

## 3. 意向判断规则

### 3.1 外部规则优先（重要）

如果用户指令中提供了明确的关键词映射表、优先级顺序或首命中即停规则，**必须严格按外部规则执行**，不使用本节默认规则。输出时须保留"命中字段"与"命中关键词"作为分类依据。

外部规则传入形式示例参考：
```
P4：历史续费意向包含"不续"、"拒绝"
P0：历史续费意向包含"主动"、"怎么买"
...（首个命中即停止）
```

**多信号冲突处理规则：**

- 若同一条客户数据同时命中多个外部规则层级，按外部规则中定义的优先级顺序取最高优先级，首个命中即停止，不叠加。
- 若外部规则只覆盖部分层级（如仅定义了 P0/P4），未覆盖层级（P1/P2/P3）仍使用第 3.2 节默认语义规则补充判断。
- 若外部规则与默认规则对同一条数据产生不同结论，**外部规则结论优先**，默认规则结论作为备注输出，不影响最终分类。
- 输出时须注明："命中来源：外部规则 / 默认规则"，以便人工复核。

### 3.2 默认语义判断规则（无外部规则时使用）

按以下顺序进行语义匹配，首个命中即停止：

1. **P4**：历史续费意向中出现明确拒绝续费的表达（如"不续了""不考虑了""拒绝"等）
2. **P0**：历史续费意向中出现主动咨询购买的表达（如"怎么续""怎么买""什么时候报名"等）
3. **P3**：历史续费意向中出现失联/低响应的表达（如"不接电话""未回复""再联系"等）
4. **P1**：客户主要顾虑中出现价格相关表达（如"太贵""经济压力""有没有优惠"等）
5. **P2**：客户主要顾虑中出现效果相关表达，或以上规则均未命中时的默认分类

### 3.3 分层结果输出要求

每条分层结果必须包含：
- 意向分类结果（P0-P4）
- 命中字段（"历史续费意向"或"客户主要顾虑"）
- 命中依据（关键词或语义描述）

## 4. 各层沟通策略

### P0（强烈意向）
- **建议渠道**：电话优先，微信/消息补充方案
- **话术重点**：近期服务进展 + 续费衔接价值 + 明确下一步付款动作
- **节奏**：当天响应，避免拖延导致意向冷却

### P1（价格敏感）
- **建议渠道**：电话解释价值，微信/消息发送方案对比
- **话术重点**：服务阶段窗口价值 → 单次性价比 → 分层方案（小套餐/分期）
- **节奏**：首次电话后 24 小时内跟进，给出具体方案

### P2（效果犹豫）
- **建议渠道**：电话深聊优先，不适合文字消息处理
- **话术重点**：共情 → 拆解问题来源（服务提供方/时间安排/方法/配合度）→ 给出可执行调整方案 → 短周期验证承诺
- **节奏**：给出调整方案后约定 1-2 周回访验证

### P3（沉默失联）
- **建议渠道**：文字消息优先，避免电话高压
- **话术重点**：高价值短内容（服务亮点）+ 低门槛互动（只需回复"好"）+ 轻提醒
- **节奏**：7 天内最多触达 2 次；首次发送后间隔 3 天无回复再跟进一次，第 2 次仍无回复则停止主动触达，等待客户自主响应

### P4（明确拒绝）
- **建议渠道**：文字消息，避免电话追单
- **话术重点**：尊重决定 → 收集真实原因 → 保留后续联系窗口
- **节奏**：不追单，仅做一次关系维护触达

## 5. 通用异议处理方法

处理顺序固定，不可跳步：

1. **先共情**：认可客户顾虑的合理性，不急于辩解
2. **再澄清**：确认顾虑的具体类型与场景
3. **给方案**：提供可执行的替代方案
4. **促下一步**：给出低压力的具体行动建议

常见异议处理要点：

| 异议类型 | 处理要点 |
|----------|----------|
| 太贵了 | 强调当前服务阶段的窗口价值与单次成本，不直接硬推价格；给出小套餐/分期选项 |
| 想先暂停 | 强调服务节奏连续性的重要性；给出降频方案作为过渡 |
| 等某件事之后再说 | 先保留衔接计划，约定具体回访时间，不催促 |
| 效果未达预期 | 拆解问题来源（服务提供方/时间安排/方法/配合度），给出短周期优化计划 |
| 最近手头紧 | 先帮锁定服务资源，约定付款时间，降低当下决策压力 |

具体话术示例见 `reference/renewal-objection-playbook.md`。

## 6. 话术生成框架

每条跟进话术按以下结构生成，顺序不可调换：

1. **开场称呼**：包含客户称呼、服务对象姓名、服务类型、本次联系目的
2. **进展同步**：引用"最近一次服务反馈"内容；若原文不超过100字，完整引用；若原文超过100字，可提炼核心摘要（不超过80字），但须保留关键事实与情感倾向，不得歪曲原意
3. **策略主句**：根据该客户的 P 层，说明建议的续费动作或解决方向
4. **异议应对**：预判该客户最可能提出的异议，给出应答
5. **收口动作**：给出明确且低压力的下一步（如"我先把方案发给您看看"）
6. **跟进建议**：建议下次跟进的时间与渠道

### 硬性质量要求

- 必须包含服务对象姓名与服务类型
- 必须包含"最近一次服务反馈"内容：原文≤100字时完整引用；原文>100字时可摘要（≤80字），须保留关键事实与情感倾向，不得歪曲原意
- 语气须专业、尊重、非施压
- 不得出现虚假承诺（如"保证达到预期效果""一定有效"）

### 红线禁止行为

以下行为在任何情况下均不得出现：

- **禁止施压催单**：不得使用"今天必须决定""名额只剩最后一个"等制造紧迫感的表述
- **禁止捏造数据**：不得编造服务次数、效果数据、优惠金额等客户信息中未提供的具体数字
- **禁止承诺退款或赔偿**：不得在话术中承诺退款、赔偿或任何超出服务范围的补偿
- **禁止贬低竞品**：不得在话术中直接或暗示性地贬低其他服务方或竞争产品
- **禁止忽略明确拒绝**：客户已明确表示 P4 时，不得继续生成促成话术，只能生成关系维护内容
- **禁止静默跳过异常**：遇到字段缺失、规则冲突或质量自检不通过时，必须显式输出提示，不得静默处理

## 7. 批量处理流程

处理多名客户时，按以下步骤逐条执行：

1. 读取并标准化字段（统一字段名，处理缺失值）
2. 检查是否有外部关键词规则传入；有则加载，无则使用默认语义规则
3. 逐条执行意向分层，记录命中依据
4. 逐条映射沟通策略与异议处理方向
5. 逐条生成结构化话术
6. 按优先级排序后输出

默认优先级顺序（无外部定义时）：P0 > P1 > P2 > P3 > P4

---

## 8. 质量自检清单

在输出每条结果前逐项确认：

- [ ] 是否包含服务对象姓名与服务类型
- [ ] 是否包含服务反馈内容（≤100字完整引用；>100字已摘要且未歪曲原意）
- [ ] 意向分类与沟通策略是否一致
- [ ] 是否给出了明确的下一步动作
- [ ] 是否给出了跟进时间与渠道建议
- [ ] 话术语气是否专业、尊重、非施压

若任一项不满足：先自动重生成一次；仍失败则输出"失败原因 + 当前最可用版本"，不得静默跳过。

---

## 9. 输出格式规范

### 9.1 单条输出结构

每条客户结果按以下顺序输出，字段间以空行分隔：

```
【客户姓名】XXX  【服务类型】XXX
【意向分层】PX — 层级名称
【命中来源】外部规则 / 默认规则
【命中依据】字段名：关键词或语义描述
【沟通策略】建议渠道 + 话术重点 + 跟进节奏（一句话概括）
【跟进话术】
（按第6节框架生成的完整话术正文）
【下次跟进】时间建议 + 渠道建议
```

### 9.2 批量输出结构

- 按优先级排序（P0 > P1 > P2 > P3 > P4）后逐条输出
- 每条之间用分隔线 `---` 隔开
- 批量输出开头须附一行汇总：`本次共处理 X 条，P0: X 条 / P1: X 条 / P2: X 条 / P3: X 条 / P4: X 条`

### 9.3 格式兜底规则

- 若用户指令中指定了输出格式（如 JSON、表格、纯文本），以用户指定格式为准，本节规范作为默认兜底
- 不得省略【意向分层】和【命中依据】字段，这两项为最低输出要求
