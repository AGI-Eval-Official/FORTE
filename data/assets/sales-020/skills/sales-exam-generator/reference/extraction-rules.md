# 文档读取与文本处理规则

本规则定义从培训资料中提取内容的全流程约束，包括格式处理策略、多文件处理、内容提取、长文本截断和异常处理。

---

## 1. 格式处理策略

### 1.1 核心原则：尽力读取

对用户提供的所有资料文件，Agent 应**尽力读取并提取可用内容**，不得因格式不常见而拒绝处理。销售培训资料的来源多样，可能包含文档、表格、演示文稿等，均应尝试提取其中的有效信息用于出题。

### 1.2 格式与提取方式

| 格式 | 扩展名 | 提取方式 |
|------|--------|---------|
| PDF | `.pdf` | 逐页提取文字内容 |
| Word 文档 | `.docx` | 提取段落文字 + 表格内容 |
| 旧版 Word | `.doc` | 尝试读取；若失败则自动跳过该文件并记录“建议转换为 .docx”的处理备注 |
| Excel 表格 | `.xlsx` / `.xls` / `.csv` | 提取各工作表的文本和数值内容，表格以 `|` 分隔输出 |
| PowerPoint | `.pptx` / `.ppt` | 逐页提取幻灯片文字 + 备注 |
| 纯文本 | `.txt` / `.md` | 直接读取文本内容 |
| 其他格式 | 任意 | 尝试以文本方式读取；若无法提取有效内容，记录失败原因并跳过 |

### 1.3 读取失败时的处理

- **尽力尝试，失败不阻塞**：某个文件读取失败时，记录失败原因并继续处理其余文件，不得因一个文件失败而中断整个流程。
- **自动降级处理**：对于无法提取有效内容的文件，写入“替代方案建议（如转为文本版）”到执行结果，不进入反问环节。

---

## 2. 多文件处理

### 2.1 合并规则

当用户提供多份培训资料时：

1. **依次读取**每个文件，分别提取文本。
2. 在每个文件提取的文本开头插入来源标记：`[来源：{文件名}]`，便于出题时覆盖各文件内容，避免题目集中在一份资料。
3. **按用户提供的文件顺序**合并为一份完整文本，文件之间用两个换行符分隔。
4. 合并后检查是否有**重复段落**（完全相同的连续文字），如有则去重保留一份。

### 2.2 示例

用户提供文件 A（公司介绍）和文件 B（产品说明），合并后：

```
[来源：文件A.pdf]
[文件A的全部文字]

[来源：文件B.docx]
[文件B的全部文字]
```

---

## 3. 内容提取规则

### 3.1 文字内容

- 提取文档中所有段落文字，保留段落结构（空行分隔段落）。
- 页眉页脚处理规则：默认保留页眉页脚文字内容；仅在明确识别为页码（纯数字）、版权声明（含"版权""©""Copyright"等关键词）或公司 logo 文字时忽略。

### 3.2 表格内容

- 读取文档时，**必须同时提取表格内容**，不得遗漏。
- 表格内容以 `|` 分隔各列，每行一条记录。
- 表格前后各空一行，与正文分隔。

示例：

```
| 产品名称 | 型号 | 价格 |
| 泵A | X-100 | 5000元 |
| 泵B | X-200 | 8000元 |
```

### 3.3 Excel 特殊处理

- 每个工作表的内容单独提取，工作表之间以 `[工作表：{名称}]` 标记分隔。
- 数值保留原始精度，不进行四舍五入或格式转换。
- 空单元格以空字符串表示，不以 "NULL" 或 "0" 填充。

---

## 4. 长文本处理

### 4.1 截断阈值

- 默认截断阈值为 **15000 字符**。
- **用户指定优先**：用户明确指定阈值时，以用户要求为准。

### 4.2 截断规则

文本超过阈值时，**按段落边界截断**，不得在句子或表格中间截断：

1. 定位阈值位置前的**最后一个换行符**。
2. 在该换行符处截断。
3. 截断后在执行结果中写入说明：`文本较长（共 X 字符），已截取前 Y 字符出题；可通过更大阈值或分章节策略覆盖更多内容。`

### 4.3 分章节处理（推荐）

当文本超过阈值时，自动采用以下兜底策略之一：

- 按章节拆分文档，分批生成考题后合并。
- 在上下文窗口允许范围内提升截断阈值。

---

## 5. 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 文件路径不存在 | 明确报错：`文件不存在：{路径}，请检查文件路径是否正确。` |
| 文件读取报错 | 显示具体错误信息，继续处理其余文件；若所有文件均失败则终止。 |
| 某个文件提取结果为空 | 记录该文件无有效内容并继续处理其余文件。 |
| 格式未知或不常见 | 尝试以文本方式读取；若无法提取有效内容，记录失败原因并跳过。 |

---

## 6. 提取后检查

文本提取完成后，在进入出题环节前，需进行以下检查：

1. **内容完整性**：确认提取的文字涵盖文档的核心内容（非仅页眉页脚、版权声明等）。
2. **编码正确性**：确认无乱码（如 `ï¿½` 等替换字符）。
3. **长度判断**：如文本少于 500 字符，记录“内容可能不足、出题质量受限”的风险提示。

---

## 7. 兜底策略（用户输入优先原则）

在所有提取和处理环节中，**用户输入为第一优先级**：

| 场景 | 处理规则 |
|------|---------|
| 用户指定截断阈值 | 直接使用用户指定的阈值，覆盖默认 15000 字符 |
| 用户指定文件读取顺序 | 按用户指定的顺序处理文件，不按字母序或其他默认顺序 |
| 用户指定编码格式 | 按用户指定的编码读取文本文件，覆盖默认自动检测 |
| 用户指定提取范围 | 仅提取用户指定的页码范围或章节，不提取全文 |
| 用户提供纯文本 | 跳过提取步骤，直接使用用户提供的文本内容 |

> **核心原则**：用户明确指定的任何提取参数、格式规则、处理顺序均以用户输入为准，本规则仅作为用户未指定时的默认示例。

### 7.1 规则兜底与冲突消解

- 规则冲突优先级：用户最新明确要求 > 用户历史明确要求 > `SKILL.md` 主流程约束 > 本规则默认示例。
- 除输入文件全部不可用等阻断条件外，执行阶段不进入反问，统一采用自动兜底并继续流程。
- 无法自动修复的异常以结构化问题清单输出，不追加询问语句。
