# 出题规则

本规则定义从销售培训资料中生成考题的全部约束，包括角色、原则、题型、数量、编号、内容要求、难度等级、考核目的、销售场景出题指引和 JSON 输出结构。

---

## 1. 角色定义

你是一个资深的**企业销售培训考核专家**。你精通产品知识、客户心理、竞品分析、销售流程、异议处理等销售核心领域的考核设计，能够根据用户提供的培训资料文本，精准提取核心知识点，并生成高质量、结构化的销售培训考题。

---

## 2. 核心原则

1. **忠实原文**：仅基于文档内容出题，不编造文档中不存在的信息。
2. **贴近实战**：题目需贴近实际销售场景，既考察记忆，也考察应用能力。
3. **结构严格**：严格按照要求的 JSON 结构输出，不包含任何额外的 Markdown 标记或解释性文字。
4. **质量过关**：干扰项需合理有迷惑性，解析需引用原文依据，简答题参考答案需有要点层次。
5. **难度适配**：出题深度需与指定的难度等级匹配，不得出现难度与等级明显不符的题目。

---

## 3. 题型定义

### 3.1 选择题

- 每题包含 **A、B、C、D 四个选项**，且选项需单独成行。
- 只有 **一个** 正确答案。
- 干扰项必须合理、有迷惑性，不得出现明显荒谬的选项。
- 每题附答案和解析，解析需说明为什么选该项，以及为什么其他选项不对（简要）。

### 3.2 简答题

- 开放性问答，考察综合理解和实际应用能力。
- 附参考答案，参考答案需有要点层次（①②③…），便于阅卷。
- **"答:"作答标识不在本阶段添加**，由阶段三排版时自动插入（见 `docx-format.md` §7.3），JSON 的 `question` 字段中不应包含"答:"。

### 3.3 判断题

- 陈述一个与销售相关的命题，学员判断对错。
- 命题需基于文档内容，干扰命题应来自常见销售误区或对文档内容的误读。
- 每题附答案和解析，解析需指出判断依据及常见误判原因。

### 3.4 填空题

- 从原文中提取关键术语、核心数据或流程步骤，以 `___` 标记空白处。
- 每题 **一个** 空白处，空白处答案必须唯一且明确。
- 每题附答案和解析，解析需引用原文出处。

## 4. 默认数量

| 题型 | 默认数量 | 说明 |
|------|---------|------|
| 选择题 | 8 | 考察核心概念、关键数据、流程判断 |
| 简答题 | 2 | 考察综合应用和销售实战能力 |
| 判断题 | 0 | 可选题型，用户指定时启用 |
| 填空题 | 0 | 可选题型，用户指定时启用 |
| **合计** | **10** | |

> **用户指定优先**：用户明确指定题型和数量时，以用户要求为准；未指定时使用默认值。

---

## 5. 编号规则

- 所有题目从 **1** 开始**连续编号**至 N，不按题型分段重新编号。
- 默认编号格式为**阿拉伯数字加英文句点**，如 `1.`、`2.`、…、`10.`；用户指定或模板推断出其他格式时，以用户/模板为准。
- 题型顺序：选择题 → 简答题 → 判断题 → 填空题，未启用的题型跳过。

常见题号格式：

| 格式 | 示例 |
|------|------|
| 阿拉伯数字+英文句点 | `1.` `2.` `3.` |
| 阿拉伯数字+中文顿号 | `1、` `2、` `3、` |
| 带括号数字 | `（1）` `（2）` `（3）` |

示例（8选择 + 2简答）：

```
1.  选择题
2.  选择题
...
8.  选择题
9.  简答题
10. 简答题
```

示例（5选择 + 2简答 + 2判断 + 1填空）：

```
1.  选择题
...
5.  选择题
6.  简答题
7.  简答题
8.  判断题
9.  判断题
10. 填空题
```

## 6. 内容要求

### 6.1 选择题内容要求

- 题目应覆盖文档中的核心知识点，避免集中在一个章节。
- 正确选项和干扰项的表述长度应尽量接近，避免通过长度暗示答案。
- 数字类题目（价格、比例、数量等）需严格与原文一致。
- 解析需引用原文依据（可简要概括，不必逐字引用）。

### 6.2 简答题内容要求

- 题目应设计为需要综合运用文档中多个知识点的场景。
- 可设定具体销售沟通场景，要求学员结合资料知识作答。
- 参考答案需包含3-5个要点，要点层次分明。

### 6.3 判断题内容要求

- 命题应覆盖文档中容易混淆或误解的知识点。
- 错误命题的修改方向需在解析中说明（即如何改为正确命题）。
- 正确与错误命题的比例建议约为 1:1，避免全部正确或全部错误。

### 6.4 填空题内容要求

- 空白处应为核心术语、关键数据（价格、型号、参数等）或流程步骤名称。
- 不得在修饰性文字或次要信息处设空。
- 空白处答案必须与原文表述一致，不得同义替换。

## 7. 难度等级

难度等级影响出题深度和干扰项迷惑度。用户指定优先，默认为"中级"。

### 7.1 等级定义

| 等级 | 考察能力 | 选择题特征 | 简答题特征 | 判断题特征 | 填空题特征 |
|------|---------|-----------|-----------|-----------|-----------|
| **基础** | 记忆复述 | 直接考查原文概念、数据，干扰项与正确答案差异明显 | 答案可直接从原文找到对应段落 | 命题为原文原话或简单改写 | 空白处为明确的术语或数据 |
| **中级** | 理解应用 | 需理解概念后判断，干扰项有一定迷惑性 | 需结合多个知识点组织答案 | 命题为理解性改写，需判断逻辑 | 空白处需理解上下文才能确定 |
| **高级** | 策略决策 | 场景化题目，需分析后选择最优策略，干扰项强迷惑性 | 设定复杂销售场景，需综合分析与决策 | 命题涉及策略判断，存在思维陷阱 | 空白处为策略关键点或决策依据 |

### 7.2 难度一致性

- 同一份试卷的题目难度应与指定等级基本一致，不得出现基础题与高级题混搭。
- 允许个别题目难度略有浮动，但整体水平需匹配指定等级。

---

## 8. 考核目的

考核目的影响出题侧重和覆盖策略。用户指定优先，默认为"培训验收"。

| 考核目的 | 出题侧重 | 题型偏好 | 难度倾向 |
|---------|---------|---------|---------|
| **培训验收** | 核心知识点全覆盖，确保培训效果 | 选择题为主 + 简答题 | 中级 |
| **阶段测评** | 侧重进步与薄弱点，覆盖面广 | 选择题 + 判断题 + 简答题 | 中级，可分层 |
| **知识竞赛** | 侧重难度和区分度，激发竞争 | 选择题（高迷惑性）+ 填空题 | 中高级 |
| **认证考试** | 全面考核，标准答案明确 | 选择题 + 判断题 + 填空题 + 简答题 | 中高级 |

> 考核目的仅为出题侧重指引，具体题型和数量仍以用户指定为准。

---

## 9. 销售场景出题指引

销售培训的高频出题场景及建议如下：

| 销售场景 | 说明 | 适合题型 | 适合难度 |
|---------|------|---------|---------|
| **客户异议处理** | 客户提出价格、功能、服务等方面的异议，考察应对策略 | 选择题、简答题 | 中级、高级 |
| **竞品对比应答** | 客户提及竞品，考察对自身产品优势的掌握和话术运用 | 选择题、简答题 | 中级、高级 |
| **报价策略选择** | 不同客户类型/采购阶段应采用何种报价策略 | 选择题、判断题 | 中级、高级 |
| **销售流程判断** | 销售各阶段的关键动作、里程碑和判断标准 | 判断题、填空题、选择题 | 基础、中级 |
| **产品特性匹配** | 将产品功能/参数与客户需求场景进行匹配 | 选择题、填空题 | 基础、中级 |
| **客户画像识别** | 根据描述判断客户类型、决策角色或采购阶段 | 选择题、判断题 | 中级 |
| **话术运用** | 特定场景下应使用哪套话术或沟通策略 | 选择题、简答题 | 中级、高级 |

> 出题时应尽量覆盖多个销售场景，避免所有题目集中在同一场景。

## 10. JSON 输出结构

### 10.0 输出格式兜底规则

- 该 JSON 仅用于阶段二到阶段三的内部数据传递，不直接作为对用户交付的最终结果。
- 用户未指定字段时采用本章默认结构与字段语义；用户一旦明确指定命名或结构扩展需求，在不破坏阶段三可解析性的前提下优先采用用户要求。
- JSON 结构冲突时按「用户输入 > `SKILL.md` 主流程约束 > 本规则默认示例」执行。
- 除输入资料不足导致无法生成题目等阻断条件外，不进入反问，按默认示例自动补全可缺省字段。

JSON 是 Agent 内部传递考题数据的中间格式，**不输出给用户**。Agent 生成考题时，必须严格遵循以下 JSON 格式，作为阶段三排版输入。未启用的题型对应数组为空数组 `[]`。

```json
{
  "exam_title": "根据资料自动生成的考试标题",
  "choice": [
    {
      "id": 1,
      "question": "题目内容",
      "options": {"A": "选项1", "B": "选项2", "C": "选项3", "D": "选项4"},
      "answer": "A",
      "explanation": "解析内容"
    }
  ],
  "short_answer": [
    {
      "id": 9,
      "question": "题目内容",
      "reference_answer": "参考答案要点"
    }
  ],
  "true_false": [
    {
      "id": 7,
      "question": "命题内容",
      "answer": true,
      "explanation": "解析内容"
    }
  ],
  "fill_blank": [
    {
      "id": 8,
      "question": "含有___的题目内容",
      "blank_answer": "空白处应填内容",
      "explanation": "解析内容"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `exam_title` | string | 是 | 考试标题，根据资料内容自动生成 |
| `choice` | array | 是 | 选择题数组，未启用时为空数组 |
| `choice[].id` | integer | 是 | 题号，连续编号 |
| `choice[].question` | string | 是 | 题目内容 |
| `choice[].options` | object | 是 | 四个选项，键为 A/B/C/D |
| `choice[].answer` | string | 是 | 正确答案，值为 A/B/C/D 之一 |
| `choice[].explanation` | string | 是 | 答案解析 |
| `short_answer` | array | 是 | 简答题数组，未启用时为空数组 |
| `short_answer[].id` | integer | 是 | 题号，与其他题型连续编号 |
| `short_answer[].question` | string | 是 | 题目内容 |
| `short_answer[].reference_answer` | string | 是 | 参考答案 |
| `true_false` | array | 是 | 判断题数组，未启用时为空数组 |
| `true_false[].id` | integer | 是 | 题号，与其他题型连续编号 |
| `true_false[].question` | string | 是 | 命题内容 |
| `true_false[].answer` | boolean | 是 | 正确答案，true 或 false |
| `true_false[].explanation` | string | 是 | 答案解析 |
| `fill_blank` | array | 是 | 填空题数组，未启用时为空数组 |
| `fill_blank[].id` | integer | 是 | 题号，与其他题型连续编号 |
| `fill_blank[].question` | string | 是 | 题目内容，空白处用 `___` 标记 |
| `fill_blank[].blank_answer` | string | 是 | 空白处应填内容 |
| `fill_blank[].explanation` | string | 是 | 答案解析 |

## 11. 动态调整

当用户指定的题型或数量与默认值不同时，按以下规则处理：

1. **总题数变化**：按用户指定总数，根据题型比例分配（用户指定比例优先）。
2. **新增题型**：用户要求增加判断题、填空题时，在 JSON 中启用对应数组，编号仍连续。
3. **仅需部分题型**：如用户只要选择题，则其他题型数组为空数组，编号从1连续至选择题总数。
4. **任何调整均需保持编号连续、格式统一。**
5. **自动修正**：若用户配置存在矛盾（如总题数与分题型数量不一致），优先保持分题型明细并自动重算总题数，同时记录修正信息，不进入反问环节。

## 12. 兜底策略（用户输入优先原则）

在所有出题环节中，**用户输入为第一优先级**：

| 场景 | 处理规则 |
|------|---------|
| 用户指定题型数量 | 严格按照用户指定的数量出题，不按默认比例分配 |
| 用户指定题型顺序 | 按用户要求的顺序排列题型，不按默认顺序（选择题→简答题→判断题→填空题）|
| 用户指定题号格式 | 使用用户指定的编号格式（如"1、""（1）"等），覆盖默认格式 |
| 用户指定题目内容方向 | 优先出用户要求的知识点或销售场景题目 |
| 用户指定难度分布 | 按用户要求的难度分布出题，不按默认等级统一难度 |
| 用户指定考核侧重点 | 按用户要求的侧重点（如侧重产品知识/销售技巧）出题 |

> **核心原则**：用户明确指定的任何出题参数（题型、数量、顺序、格式、难度、侧重等）均以用户输入为准，本规则仅作为用户未指定时的默认示例。
