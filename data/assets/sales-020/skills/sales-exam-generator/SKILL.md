# 销售培训考题生成器

读取销售培训资料（支持 PDF、Word、Excel、PPT 等格式），提取核心知识点，生成结构化考题，输出排版规范的 Word（.docx）文档。

---

## 参数解析

在执行任务前，**必须先从用户输入中解析以下参数**。解析规则：**用户指定优先，默认方案兜底**。默认值仅作为示例，不覆盖用户明确要求；除输入文件缺失这类无法继续执行的场景外，不设置反问环节。

| 参数 | 解析来源 | 默认值 | 说明 |
|------|---------|--------|------|
| 输入文件 | 用户明确指定 | 无（必填） | 一个或多个资料文件路径；支持 PDF、DOCX、XLSX、PPTX 等格式 |
| 总题数 | 用户明确指定 | 10 | 考题总数 |
| 题型配置 | 用户明确指定 | 选择题 8 + 简答题 2 | 各题型的数量；支持选择题、简答题、判断题、填空题自由组合 |
| 难度等级 | 用户明确指定 | 中级 | 可选：基础 / 中级 / 高级 |
| 考核目的 | 用户明确指定 | 培训验收 | 可选：培训验收 / 阶段测评 / 知识竞赛 / 认证考试 |
| 截断阈值 | 用户明确指定 | 15000 | 单次处理的最大字符数；一般无需指定，仅在需要处理超长文档时调整 |
| 排版模板 | 用户明确指定 | 无（使用默认排版） | .docx 模板文件路径；提供时从模板提取页面设置、字体、间距等样式，覆盖默认排版 |
| 题号编号格式 | 用户明确指定 | 阿拉伯数字+英文句点（1. 2. …） | 自定义题号编号格式；常见格式见 prompt-rules.md §5 |
| 章节编号格式 | 用户明确指定 | 中文序号（一、二、…） | 自定义章节编号格式；常见格式见 docx-format.md §5.2 |
| 输出文件名 | 用户明确指定 | 培训验收.docx | 输出 .docx 文件名 |
| 输出路径 | 用户明确指定 | 输入文件所在目录 | 输出文件的保存目录 |

**兜底策略（用户输入优先原则）**：
- 用户明确指定的任何参数（包括格式、样式、数量、顺序等）均以用户输入为第一优先级
- 用户指定的题型数量、题型顺序、编号格式、字体样式等规则直接采用，不套用默认值
- 用户提供排版模板时，模板中定义的页面设置、字体、间距、颜色、编号格式等样式优先于默认规则
- 仅当用户未指定时，才使用上述默认值兜底

### 解析示例

**示例一：知识竞赛场景**

用户输入：`用这份资料出15道竞赛题，10道选择题3道判断题2道填空题，高级难度`

解析结果：

- 输入文件：沿用当前任务中已提供的资料文件；若未提供则标记为缺失
- 总题数：15（用户指定）
- 题型配置：选择题 10 + 判断题 3 + 填空题 2（用户指定）
- 难度等级：高级（用户指定）
- 考核目的：知识竞赛（由"竞赛"推断）
- 排版模板：无（默认）
- 题号编号格式：阿拉伯数字+英文句点（默认）
- 章节编号格式：中文序号（默认）
- 输出文件名：培训验收.docx（默认）
- 输出路径：输入文件所在目录（默认）

## 任务规划

参数解析完成后，先执行参数校验，再按以下 4 个阶段**顺序执行**，每个阶段的输出是下一阶段的输入。

### 参数校验

参数解析完成后，执行以下校验，不通过时优先自动兜底修正，并做标记

| 校验项 | 规则 | 不通过时的处理 |
|--------|------|---------------|
| 输入文件必填 | 至少有一个有效的输入文件路径 | 无有效输入文件时直接终止，返回缺失文件清单 |
| 题型数量之和 | 各题型数量之和等于总题数 | 不一致时以题型数量之和重算总题数，并记录自动修正 |
| 数量非负 | 每种题型数量 ≥ 0 | 小于 0 的数量按 0 处理，并记录自动修正 |
| 总题数下限 | 总题数 ≥ 1 | 小于 1 时兜底为 1，并将题型配置重置为选择题 1 |
| 数量为0的含义 | 某题型数量为0等同于未启用该题型 | 正常处理 |

---

### 阶段一：文档读取与文本提取

- **规则文档**：reference/extraction-rules.md
- **输入**：用户指定的文件路径
- **输出**：合并后的完整文本（含来源标记）
- **关键动作**：
  1. 按文件顺序依次读取，尽力提取各格式文件中的文字、表格内容
  2. 每个文件文本开头插入来源标记 `[来源：{文件名}]`
  3. 合并多文件文本，文件间以两个换行分隔
  4. 检查重复段落并去重
  5. 检查提取结果的完整性和编码正确性
  6. 超过截断阈值时按段落边界截断，并在执行结果中记录截断说明

### 阶段二：考题生成

- **规则文档**：reference/prompt-rules.md
- **输入**：阶段一输出的文本 + 解析后的参数
- **输出**：考题数据（Agent 内部 JSON 格式，不输出给用户）
- **关键动作**：
  1. 以"企业销售培训考核专家"角色，基于文本生成考题
  2. 按题型配置和难度等级出题
  3. 按考核目的调整出题侧重
  4. 参考销售场景出题指引，覆盖多个销售场景
  5. 所有题目从1连续编号至总题数
  6. 选择题必须含ABCD四选项；简答题的"答:"作答标识由阶段三排版时自动插入，本阶段不在 JSON 中添加
  7. 考题数据严格符合 JSON 结构定义，作为阶段三的内部输入

### 阶段三：文档排版与输出

- **规则文档**：reference/docx-format.md
- **输入**：阶段二生成的考题数据（JSON）
- **输出**：排版规范的 .docx 文件（交付用户的最终产物）
- **关键动作**：
  1. 若用户提供了排版模板，解析模板提取样式（页面设置、字体、间距、编号格式等）；未提供时使用默认排版
  2. 按排版规范（模板优先，默认兜底）生成 Word 文档
  3. 按启用题型动态生成章节，编号格式按用户指定或默认
  4. 题目中不显示答案和解析
  5. 文档末尾附答案速查表
  6. 保存到用户指定路径或默认路径

### 阶段四：输出质量检查

- **输入**：阶段三生成的 .docx 文件 + 阶段二的考题数据（用于核对内容）
- **输出**：检查通过 / 检查不通过（需修正后重新执行对应阶段）
- **关键动作**：见下方"输出质量检查"章节
- **回退分级**：
  - 排版类失败（#1-#25）：修正后从**阶段三**重新执行
  - 内容类失败（#26-#31）：修正后从**阶段二**重新执行，因为题目内容本身有问题，仅重新排版无法修复
- **最大重试**：总计最多重新执行 3 次，超过后输出当前最佳版本并附带未通过项清单，告知用户需手动调整

## 规则路由

| 阶段 | 适用规则文档 | 何时引用 |
|------|-------------|---------|
| 文档读取与文本提取 | `reference/extraction-rules.md` | 需要读取、提取、合并、截断文本时 |
| 考题生成 | `reference/prompt-rules.md` | 需要出题、定义题型、构建 JSON、设定难度/目的时 |
| 文档排版与输出 | `reference/docx-format.md` | 需要生成 .docx 文件、设置字体/间距/表格、动态章节编号时 |

**路由原则**：

1. 每个阶段仅引用对应规则文档，不交叉引用。
2. 当规则文档内容与用户指令冲突时，**用户指令优先**。
3. 阶段二内部按题型配置动态引用 `prompt-rules.md` 中对应题型的规则段落（§3.1-§3.4）。
4. **兜底策略**：用户提供任何具体参数（格式、样式、数量、顺序等）时，以用户输入为第一优先级，规则文档仅作为未指定时的默认示例。
5. **冲突消解**：当同一参数出现多条冲突约束时，优先级为「用户最新明确要求 > 用户历史明确要求 > 对应 reference 规则 > 本文件默认示例」。

---

## 输出格式兜底规则

1. 最终交付格式固定为 `.docx`；阶段二 JSON 仅作为内部中间数据，不直接对外输出。
2. 用户未指定输出文件名时，使用默认示例 `培训验收.docx`；用户一旦指定则完全覆盖默认示例。
3. 用户未指定输出路径时，默认保存到输入文件所在目录；当该目录不可写时，自动回退到当前工作区目录并记录回退信息。
4. 输出格式相关规则冲突时，按「用户输入 > `reference/docx-format.md` > 本文件默认示例」执行。

## 输出质量检查

文档生成后，**必须逐项检查以下清单**。任一项不通过则按回退分级规则修正后重新执行：

### 结构检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 总题数 | 等于用户指定或默认的总题数 |
| 2 | 各题型数量 | 各题型数量等于用户指定或默认值 |
| 3 | 题号连续性 | 从1到总题数连续编号，无跳号、无重号 |
| 4 | 题号格式 | 符合用户指定或默认的编号格式（默认：阿拉伯数字+英文句点） |
| 5 | 题型顺序 | 选择题 → 简答题 → 判断题 → 填空题，未启用题型跳过 |

### 选择题检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 6 | 选项完整性 | 每题均含 A、B、C、D 四个选项 |
| 7 | 选项独立成行 | 每个选项独占一行 |
| 8 | 选项格式 | 选项独立成行，格式为缩进+字母+英文句点+内容（默认：`    A. 选项内容`） |
| 9 | 答案有效性 | answer 值为 A/B/C/D 之一 |
| 10 | 答案隐藏 | 题目中不显示答案和解析 |

### 简答题检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 11 | "答:"标识 | 题干末尾另起一行包含"答:" |
| 12 | 参考答案隐藏 | 题目中不显示参考答案 |

### 判断题检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 13 | 命题完整性 | 每题有明确的命题陈述 |
| 14 | 答案有效性 | answer 值为 true 或 false |
| 15 | 答案隐藏 | 题目中不显示答案和解析 |

### 填空题检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 16 | 空白标记 | 题目中包含 `___` 标记 |
| 17 | 空白答案唯一 | blank_answer 为唯一明确答案 |
| 18 | 答案隐藏 | 题目中不显示答案和解析 |

### 文档排版检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 19 | 文档标题 | 居中，包含考试标题 |
| 20 | 章节标题 | 按启用题型动态编号，格式符合用户指定或默认（默认：一、二、三…） |
| 21 | 章节标题内容 | 与实际题型名称匹配（选择题/简答题/判断题/填空题） |
| 22 | 答案速查表 | 文档末尾存在，4列（题型/题号/正确答案/解析摘要） |
| 23 | 表头样式 | 深蓝底白字（模板指定时以模板为准） |
| 24 | 页面边距 | 符合用户指定或默认值（默认：上下左右均为 2.5cm） |
| 25 | 模板样式一致性 | 提供模板时，页面设置、字体、间距与模板一致 |

### 内容质量检查

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 26 | 题目来源 | 所有题目内容均可在原文中找到依据，无编造信息 |
| 27 | 干扰项合理性 | 选择题干扰项有一定迷惑性，非明显荒谬 |
| 28 | 解析有效性 | 各题型解析说明了正确答案的依据 |
| 29 | 难度一致性 | 题目整体难度与指定难度等级匹配 |
| 30 | 销售场景覆盖 | 题目覆盖多个销售场景（客户异议、竞品对比、产品匹配等），未集中在单一场景 |
| 31 | 资料覆盖均衡 | 题目涉及各来源文件的内容，未集中在单份资料 |
