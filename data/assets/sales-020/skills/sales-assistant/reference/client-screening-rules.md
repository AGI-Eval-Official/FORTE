# 客户筛选规则

本文件定义任务 A（客户筛选）的输入要求、评分规则和输出模板。

---

## 规则优先级（兜底）

客户筛选任务按以下优先级执行（从高到低）：

1. 用户明确提出的筛选规则（维度、权重、分级阈值、输出字段、排序方式）
2. 用户提供的规则文档/样例/模板
3. 本文件默认规则（仅补全用户未给出的部分）

执行要求：
- 若用户给了自定义维度或权重，必须使用用户规则
- 仅对用户未定义的维度/阈值/字段启用默认规则
- 用户规则与默认规则冲突时，以用户规则为准

---

## 输入要求

优先使用用户已提供信息；信息不足时按本文件缺失信息处理规则执行。

1. **产品信息**（至少包含以下之一）：
   - 产品名称和简介
   - 产品功能特性
   - 适用行业或场景
   - 价格区间或定价模式

2. **客户列表**（至少包含以下之一）：
   - 企业名称
   - 行业
   - 其他客户相关信息（规模、地区、预算、现有痛点等）

---

## 默认评分规则（仅在用户未自定义时生效）

如无用户特殊要求，对每个客户按以下 5 个维度逐一评估并打分（每维度 1-5 分），最终加权计算总分。

### 维度 1：行业匹配度（权重 25%）

评估客户所在行业与产品目标行业的契合程度。

| 分值 | 标准 |
|------|------|
| 5 | 客户所在行业是产品核心目标行业，产品专为该行业设计 |
| 4 | 客户行业属于产品主要适用行业，有明确应用场景 |
| 3 | 客户行业可以使用该产品，但非核心目标行业 |
| 2 | 客户行业与产品有边缘关联，需要较多定制才能适配 |
| 1 | 客户行业与产品几乎无关联 |

### 维度 2：需求匹配度（权重 30%）

评估客户已知需求或痛点与产品功能的匹配程度。

| 分值 | 标准 |
|------|------|
| 5 | 客户核心需求与产品核心功能直接对应 |
| 4 | 客户多数需求可被产品满足 |
| 3 | 客户部分需求与产品功能匹配 |
| 2 | 客户需求与产品仅少量交集 |
| 1 | 无法识别客户需求与产品匹配点 |

### 维度 3：预算适配性（权重 20%）

评估客户预算能力与产品定价的匹配程度。

| 分值 | 标准 |
|------|------|
| 5 | 客户预算充足且与产品定价高度吻合 |
| 4 | 客户预算范围涵盖产品价格 |
| 3 | 预算不明确，但有支付能力信号 |
| 2 | 预算偏低或可能只适配入门方案 |
| 1 | 预算明显不足或无预算信号 |

### 维度 4：决策就绪度（权重 15%）

评估客户当前阶段是否接近购买决策。

| 分值 | 标准 |
|------|------|
| 5 | 已评估同类产品或已发采购需求 |
| 4 | 明确表达需求，处于需求确认阶段 |
| 3 | 有潜在需求但未进入正式评估 |
| 2 | 需求偏推测，缺乏明确信号 |
| 1 | 无法判断客户决策阶段 |

### 维度 5：合作可行性（权重 10%）

评估合作落地可行性（关系、地域、渠道、障碍）。

| 分值 | 标准 |
|------|------|
| 5 | 有明显有利因素且无合作障碍 |
| 4 | 至少有一个有利因素，无明显障碍 |
| 3 | 无明显有利或不利因素 |
| 2 | 存在一个不利因素 |
| 1 | 存在多个不利因素，合作难度高 |

---

## 默认分级规则（仅在用户未自定义时生效）

加权总分 = 行业匹配度 × 0.25 + 需求匹配度 × 0.30 + 预算适配性 × 0.20 + 决策就绪度 × 0.15 + 合作可行性 × 0.10

| 等级 | 分数区间 | 含义 |
|------|---------|------|
| A 级（优先跟进） | 4.0 - 5.0 | 高度匹配，优先投入销售资源 |
| B 级（值得培育） | 2.5 - 3.9 | 有一定匹配度，建议持续跟进 |
| C 级（暂缓关注） | 1.0 - 2.4 | 匹配度较低，建议低优先级处理 |

---

## 信息缺失处理（兜底）

当客户信息缺失时：

- 缺失维度默认给 3 分，并标注“[信息不足，默认评分]”
- 可基于间接线索合理推断，但标注“[推断]”
- 若同一客户超过 3 个维度信息缺失，标注“信息严重不足，建议补充后重评”

---

## 默认输出模板（仅在用户未指定格式时生效）

用户指定输出格式时，优先按用户格式输出；无指定时按以下模板输出并按总分降序排列。

```

- 客户总数：[N]

---

- 行业与属性：[行业 + 规模/属性]

  - 需求匹配：[X]/5 — [理由]
  - 预算适配：[X]/5 — [理由]
  - 决策就绪：[X]/5 — [理由]
  - 合作可行：[X]/5 — [理由]
```
