# 销售话术生成规则

本文件定义任务 B（销售话术生成）的输入要求、生成逻辑和输出模板。

---

## 规则优先级（兜底）

销售话术任务按以下优先级执行（从高到低）：

1. 用户明确提出的要求（场景、对象、语气、结构、篇幅、禁用词、输出格式）
2. 用户提供的样例/模板/企业规范
3. 本文件默认规则（仅补全用户未定义部分）

执行要求：
- 不得用默认模板覆盖用户已明确要求
- 当用户只给了部分要求，只补全缺失项
- 用户要求与默认规则冲突时，以用户要求为准

---

## 输入要求

优先使用用户已提供信息；信息不足时按默认逻辑做缺失兜底。

1. **产品信息**（至少包含以下之一）：
   - 产品名称和简介
   - 产品功能特性
   - 技术规格或案例
   - 价格信息

2. **目标客户信息**（至少包含以下之一）：
   - 客户名称或客户类型
   - 核心需求/痛点
   - 决策人角色（如 CTO、采购经理）

---

## 默认生成逻辑（仅在用户未自定义时生效）

如无特殊要求，话术必须针对具体客户定制，不生成通用稿。定制化至少包含：

- 引用客户行业、已知痛点、业务目标
- 结合产品能力给出“痛点 -> 方案 -> 收益”的说服链路
- 信息不足时，用“[信息不足，按通用场景生成]”显式标注

---

## 默认输出模板（仅在用户未指定格式时生效）

用户指定输出方式时，优先按用户方式；无指定时使用以下模板：

```
## 目标客户：[企业名称]

# 销售话术

## 话术使用说明
- 目标客户：[企业名称]
- 建议沟通对象：[角色]
- 沟通目标：[如获取演示机会]

## 场景一：初次触达
开场白：
> [30 秒内，包含客户信息 + 痛点 + 一句价值]

引导问题：
> [2-3 个开放式问题]

价值陈述：
> [行业场景 + 常见痛点 + 解决方案 + 预期收益]

## 场景二：需求确认与方案讲解
需求确认：
> [3-4 个深挖问题]

方案呈现：
> [客户痛点 -> 我方方案 -> 可见改变]

竞品应对：
> [2-3 条差异化表达，不贬低竞品]

## 场景三：异议处理
价格异议：
> [成本拆解 -> 价值对比 -> 可选方案]

决策拖延：
> [确认顾虑 -> 提供证据 -> 推进下一步]

需求不确定：
> [引导发现痛点 -> 提示风险 -> 低门槛试用]
```

---

## 输出质量检查（兜底）

输出前至少确认：

1. 优先满足用户自定义要求
2. 话术对客户信息有明确引用（至少 3 处）
3. 结构完整且可直接用于沟通
4. 表达自然，不是书面化空话
5. 未遗漏用户要求的字段/格式/禁用项
