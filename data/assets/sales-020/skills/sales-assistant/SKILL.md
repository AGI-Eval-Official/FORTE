# 销售客户筛选与沟通内容生成助手

本 skill 包含三类任务：
1. 客户分析筛选
2. 销售话术生成
3. 开发信生成

---

## 使用流程

### 第一步：识别任务类型

- **任务 A：客户筛选** — 用户提供产品信息和客户列表，输出筛选后的目标客户及分析
- **任务 B：销售话术生成** — 用户提供产品信息、目标客户信息和场景要求，输出定制化销售话术
- **任务 C：开发信生成** — 用户提供产品信息和目标客户列表（含联系人信息），输出定制化开发信

如果用户同时提供了产品信息和客户列表，且未明确任务类型，默认先执行任务 A（客户筛选），再继续任务 B 或任务 C。

### 第二步：统一优先级与兜底策略

所有任务都遵循以下优先级（从高到低）：

1. **用户明确提出的要求**（目标、规则、字段、格式、语气、篇幅、场景等）
2. **用户提供的规则文档/样例/模板**
3. **本 skill 的 reference 默认规则**（仅用于补全用户未说明部分）
4. **本 skill 的输出质量要求**（用于最终校验）

执行要求：
- 不得用默认规则覆盖用户已明确给出的规则
- 当用户只给了部分规则时，仅对缺失项启用默认规则补充
- 若用户要求与默认模板冲突，按用户要求执行

## 任务 A：客户筛选

### 规则

输入要求、评分规则、输出模板，详见 `reference/client-screening-rules.md`

### 输出质量要求

完成任务 A 后，输出必须满足以下所有条件：

1. **完整性**：客户列表中的每一个客户都被评估和输出，无遗漏
2. **评分合理性**：每个维度评分有一句话理由，且引用客户具体信息
3. **计算准确**：加权总分计算正确，等级划分与分数区间一致
4. **排序正确**：输出按总分从高到低排序
5. **信息标注**：缺失信息有明确标注（如"[信息不足，默认评分]"或"[推断]"）
6. **分析具体**：明确说明产品功能与客户需求的对应关系
7. **建议可执行**：跟进建议包含可落地的切入点和沟通重点
8. **概览一致**：筛选概览统计与明细一致
9. **格式合规**：优先遵循用户指定格式；无指定时使用默认模板

## 任务 B：销售话术生成

### 规则

输入要求、生成逻辑、输出模板，详见 `reference/sales-script-rules.md`

### 输出质量要求

完成任务 B 后，输出必须满足以下所有条件：

1. **客户定制化**：至少 3 处引用目标客户信息（企业、行业、痛点、角色等）
2. **场景匹配**：优先按用户指定场景生成；无指定时使用默认场景
3. **表达自然**：话术口语化、可直接用于真实沟通
4. **逻辑完整**：关键环节（开场、需求确认、异议处理）逻辑闭环
5. **格式合规**：优先遵循用户指定格式和结构；无指定时使用默认模板

---

## 任务 C：开发信生成

### 规则

输入要求、生成逻辑、输出模板，详见 `reference/outreach-email-rules.md`

### 输出质量要求

完成任务 C 后，输出必须满足以下所有条件：

1. **客户定制化**：至少 2 处引用目标客户信息（称呼、企业、行业、需求等）
2. **问题-方案对应**：明确写出客户问题与对应解决方案
3. **行动指引明确**：包含清晰的下一步行动引导（预约沟通/回复邮件/预约演示等）
4. **格式合规**：优先遵循用户指定格式、语言和篇幅；无指定时使用默认模板
