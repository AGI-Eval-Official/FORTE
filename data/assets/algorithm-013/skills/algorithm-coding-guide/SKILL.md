# 算法模型开发规范指南

本技能提供推荐系统模型开发的编码规范和最佳实践。

## 开发环境要求

- Python 3.8+
- PyTorch >= 1.10.0
- NumPy, Pandas, scikit-learn 等常用科学计算库

## 代码规范

### 模型代码规范

1. **模块化设计**：模型各组件（Embedding层、特征交互层、预测层等）应独立为单独的类
2. **类型注解**：forward方法的参数和返回值应有清晰的shape注释
3. **配置管理**：超参数通过配置类统一管理，不在代码中硬编码
4. **可复现性**：设置随机种子，确保实验可复现

### 训练代码规范

1. **训练/评估分离**：train和evaluate函数独立实现
2. **指标计算**：CTR任务必须计算AUC和LogLoss
3. **模型保存**：保存最优模型checkpoint，包含epoch、指标、配置信息
4. **早停机制**：实现early stopping防止过拟合
5. **日志记录**：每个epoch打印训练/验证指标

### 数据处理规范

1. **Dataset类**：继承torch.utils.data.Dataset
2. **变长序列**：使用padding + mask处理变长用户行为序列
3. **负采样**：需要负采样时，在数据生成阶段完成

## 推荐模型升级检查清单

将模型从A升级为B时，确保：

- [ ] 新模型包含所有B模型的核心组件
- [ ] 保留原有业务特征体系，不遗漏
- [ ] 数据pipeline适配新模型的输入需求
- [ ] 训练流程适配新模型的损失函数
- [ ] 代码可正常运行，无语法/运行时错误
