# 美团餐饮市场分析 Skill

本 skill 用于对美团餐饮市场相关数据进行多维度分析，覆盖外卖和到餐两大业务、五个餐饮品类的GTV规模、同比/环比变化、结构占比等核心场景，输出标准化的结构化分析结果。

## 餐饮业务背景知识

### 业务定义

- **外卖业务**：以消费GTV衡量，指用户在美团外卖平台的实际消费金额
- **到餐业务**：以实付GTV衡量，指用户在美团到店餐饮的实际支付金额
- **整体规模**：外卖消费GTV + 到餐实付GTV

### 餐饮品类分类

| 品类编号 | 品类名称 | 说明 |
|---------|---------|------|
| 1 | 正餐 | 中式正餐、西式正餐等 |
| 2 | 中式快餐 | 中式快餐连锁、单店等 |
| 3 | 西式快餐 | 麦当劳、肯德基等 |
| 4 | 奶咖 | 奶茶、咖啡等饮品 |
| 5 | 蛋糕甜点/小吃/零食零售 | 烘焙甜点、休闲零食等 |

### 时间口径说明

- **周期汇总**：指定周期内的所有数据汇总
- **日均**：周期汇总 / 周期天数
- **同比（YoY）**：与去年同期相比的变化率，公式：(本期值 - 同期值) / 同期值 × 100%
- **环比（QoQ）**：与上期相比的变化率，公式：(本期值 - 上期值) / 上期值 × 100%
- **春节周期**：1月和2月受春节影响，通常需要合并呈现YoY

### 城市等级分类

| 城市等级 | 说明 |
|---------|------|
| 一线城市 | 北京、上海、广州、深圳 |
| 新一线城市 | 成都、杭州、重庆等 |
| 二线城市 | 省会城市、计划单列市等 |
| 三线及以下 | 其他地级市及县域 |

## 分析模块与输出字段规范

### 模块一：基础GTV规模分析

分析MT餐饮整体及各品类的GTV规模。

**核心公式**：
- `整体GTV = 外卖消费GTV + 到餐实付GTV`（单位统一为万元，再转换为亿）
- `日均GTV = 周期汇总GTV / 周期天数`
- `品类GTV占比 = 某品类GTV / 所有品类GTV之和`

**必须输出的字段**（字段名不可更改）：

| 字段名 | 说明 |
|--------|------|
| `整体GTV_亿` | 指定月份/周期MT餐饮整体GTV，单位亿，保留1位小数 |
| `日均GTV_亿` | 日均GTV，单位亿，保留1位小数 |
| `最大品类` | GTV规模最大的品类名称 |
| `最小品类` | GTV规模最小的品类名称 |
| `最大品类占比` | 最大品类GTV占整体比例，格式 `XX.X%`，保留1位小数 |
| `最小品类占比` | 最小品类GTV占整体比例，格式 `XX.X%`，保留1位小数 |

### 模块二：同比（YoY）分析

分析MT餐饮GTV同比变化趋势。

**核心公式**：
- `YoY = (本期值 - 同期值) / 同期值 × 100%`
- 1月和2月合并计算YoY：合并后值 = 1月值 + 2月值

**必须输出的字段**（字段名不可更改）：

| 字段名 | 说明 |
|--------|------|
| `YoY最高月份` | YoY同比增长率最高的月份名称，如"3月" |
| `YoY最低月份` | YoY同比增长率最低（可能是负值最大）的月份名称 |
| `YoY最高值` | YoY最高值，格式 `+XX.X%` 或 `-XX.X%`，保留1位小数 |
| `YoY最低值` | YoY最低值，格式同上，保留1位小数 |
| `12月YoY` | 12月YoY值，格式同上，保留1位小数 |
| `累计YoY` | 上半年（1-6月）整体YoY值，格式同上，保留1位小数 |
| `1月2月合并YoY` | 1月2月合并后的YoY值，格式同上，保留1位小数 |
| `正增长月份数` | YoY为正的月份数量，取整数 |
| `负增长月份数` | YoY为负的月份数量，取整数 |

### 模块三：品类维度YoY分析

## 分析执行规范

1. **GTV合并计算**：整体GTV = 外卖消费GTV + 到餐实付GTV，需确保单位统一
2. **YoY计算**：
   - YoY = (本期值 - 同期值) / 同期值 × 100%
   - 1月和2月合并计算：合并值 = 1月值 + 2月值
3. **环比计算**：QoQ = (本期值 - 上期值) / 上期值 × 100%
4. **精度要求**：
   - 亿单位保留1位小数（如"12.3亿"）
   - 百分比保留1位小数（如"+5.6%"、"-2.3%"）
   - 元单位保留1位小数
   - 用户数取整数
5. **正负号处理**：增长用"+"表示，下降用"-"表示
6. **月份格式**：统一使用"1月"、"2月"等中文格式

## 输出格式

所有分析结果以 **JSON** 格式输出，字段名称必须与本 skill 规定的字段名完全一致。

```json
{
    "整体GTV_亿": "12.3",
    "日均GTV_亿": "0.4",
    "YoY最高月份": "5月",
    "YoY最低月份": "2月",
    "YoY最高值": "+15.6%",
    "YoY最低值": "-3.2%"
}
```

## 注意事项

- 当 instruction 与本 skill 要求冲突时，优先执行 instruction 要求
- 月份数据注意区分：1-2月合并呈现YoY是特殊规则
- 城市等级分类需严格按给定标准划分
- 人均GTV计算：整体GTV / 整体交易用户数（注意单位换算）
- 若某品类/城市等级数据缺失，需排除后再计算占比
