# 合同模板自动填充

## 能力定位

本 Skill 用于把填写信息写入合同模板，输出可交付的合同文档。

它解决的是"怎么填模板"，不是"合同条款是否合理"。

## 触发条件与边界

### 应触发场景

当用户出现以下需求时触发：

- 合同模板填写、合同自动生成、签约合同生成
- 把一段信息（无论是文本描述、表格、JSON 还是其他格式）填入 docx 合同模板
- 占位符替换、勾选框勾选、建议文案清理
- 根据条款批量填充合同

### 不处理场景

以下需求不在本 Skill 职责内：

- 法律审阅、合规判断、风险评级
- 条款改写、法律措辞优化、谈判意见
- 新建完全不同结构的合同模板

若任务同时包含"模板填充 + 法律审阅"，本 Skill 仅完成模板填充部分，并在输出中明确边界。

## 输入与输出

**输入**：合同模板文件（.docx）+ 填写信息。填写信息格式不限，例如一段自然语言描述、JSON / Excel / CSV 等结构化文件、或经过整理的表单文本。如有字段映射说明或格式规则（日期、金额、勾选项等），一并提供。

**输出**：填充后的合同文档（.docx），以及填充报告，至少包含已填充项（`filled_fields`）、未填充项（`unresolved_fields`）、自动决策记录（`assumptions`）和风险说明（`warnings`）。

## 核心流程

### 第一步：输入预检

执行前必须检查：

1. 是否找到模板文件
2. 是否存在可读取的填写信息
3. 关键字段是否缺失
4. 关键锚点是否可在模板中定位

预检失败时，不进入填充阶段，直接返回问题清单。

### 第二步：构建字段映射

按优先级建立映射关系：

1. 用户或任务明确提供的映射规则
2. 输入信息中已有"条款类别/条款项目/取值"结构（如来自 Excel）
3. 按锚点文本与字段语义自动匹配（兜底，包括从自然语言输入中提取）

映射必须包含三要素：

- 数据来源（输入字段或文本段落）
- 模板目标（条款或锚点）
- 填写规则（直填 / 拆分 / 勾选 / 格式化）

### 第三步：数据标准化

填充前统一处理：

- 去首尾空格、全半角归一
- 支持多分隔符拆分：`、` `,` `，` `;` `；` `/`
- 日期解析为结构化年月日
- 数值统一为可格式化数字

### 第四步：模板填充

按映射逐项填充：

1. 文本占位符替换
   - **XML run 合并规则**：docx 内部 XML 中，同一逻辑占位符（如 `乙方`、`____`）可能被拆分到多个相邻 `<w:t>` 节点中。填充前必须先合并同一 `<w:r>` 或相邻 `<w:r>` 内的文本节点，确保替换后的值写入单个连续的 `<w:t>` 节点，避免出现字符被拆散导致替换失败或输出文本断裂的问题。
2. 表格单元格写入
3. 勾选框状态更新
4. "其他"补充项写入
5. 建议文案清理

### 第五步：异常处理

按异常类型执行固定策略：

- **缺失型**：字段不存在，记录到 `unresolved_fields`
- **冲突型**：多来源值不一致，优先高置信来源并记录 `assumptions`
- **定位型**：模板锚点缺失，记录 `warnings`，保留原文不盲改

### 第六步：结果校验

至少完成以下校验：

1. 必填项是否全部处理
2. 占位下划线是否残留在"应填位置"
3. 勾选项是否与输入一致
4. 金额/日期/百分比格式是否符合规则
5. 不应修改区域是否保持原样

### 第七步：输出交付

- 生成填充后的合同文档
- 同步输出填充报告，确保可追溯

若用户未指定输出文件名，默认使用：`<模板名>-filled.docx`。

## 通用填写规则

### 占位符规则

- 用实际值替换占位下划线（如 `____`、`___`）
- 不得在下划线后追加内容
- 不应填写的位置保持原样
- **无数据来源时保留原文**：若某处占位下划线在所有输入信息中均无对应字段（即字段映射中找不到该位置的映射），则视为"无数据来源的空白区"，必须保留原始下划线，既不填值，也不清空。仅当字段映射明确指向该位置时，才允许替换或清空。

### 勾选框规则

- 选中项：`（□）` -> `（☑）`
- 未选中项：保留 `（□）`
- 若存在模板外选项，写入"其他"
- **"其他：____"下划线清空规则**：当某勾选组中"其他"选项未被选中时，其后的补充下划线（如 `其他：____`）必须清空为 `其他：`，不得保留原始下划线占位符

### 日期规则

- 优先输出 `YYYY年MM月DD日`
- 月、日补零

### 数值规则

- 金额加千位分隔符（如 `150000` -> `150,000`）
- 百分比按 `数字 + %`
- 纯数量字段保留数字

### 建议文案清理规则

当模板中出现以下提示文字时，填值后应删除整段提示：

- `（建议填写：...）`
- `（建议：...）`

**空白字符保留规则**：删除提示文字时，只移除提示括号本身（含括号内全部内容），不得改动提示括号前后原有的空格、标点或换行。例如模板为 `合作，____日内（建议填写：30日）内无法恢复`，填入 `30` 并清除提示后，结果应为 `合作，30日内无法恢复`，逗号后若原有空格须原样保留。

## 决策与冲突处理规则

当出现多个候选值时，按优先级决策：

1. 用户在本轮任务中明确指定的值
2. 同一来源内字段值
3. 历史或推断值（仅兜底）

每次自动决策都要写入 `assumptions`，避免黑盒行为。

## 交付质量检查清单

交付前确认：

- 模板结构未被破坏（段落、表格、编号连续）
- 所有已映射字段均已落位
- 无多余占位下划线残留在应填区域
- 勾选结果与输入一致
- 填充报告完整可追溯

## 失败兜底

当以下任一情况发生时，返回"未完成交付 + 问题清单"：

- 模板无法解析
- 关键锚点大面积缺失
- 关键必填字段缺失且无法推断
- 输入文件损坏或不可读取

问题清单需明确：问题类型、影响范围、建议补充信息。
