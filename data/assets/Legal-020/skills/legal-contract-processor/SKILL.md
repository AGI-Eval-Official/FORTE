# 通用法务合同处理技能

## 概述

本技能是一个**多功能法务合同处理工具箱**，根据用户意图自动路由到对应的处理能力。支持多种合同类型，每种类型使用独立的字段配置进行精确匹配。

各能力的详细规则（处理逻辑、执行步骤、兜底机制等）定义在 `references/` 目录下的子文档中，本文档为总控入口，负责意图路由和通用规范。

## 文件结构

```
legal-contract-processor/
├── SKILL.md                              ← 本文件（总控：路由 + 通用规范）
└── references/
    ├── capability-1-field-extraction.md  ← 能力一：字段提取（含字段配置）
    ├── capability-2-comparison.md        ← 能力二：合同比对
    ├── capability-3-summary.md           ← 能力三：摘要生成
    └── capability-4-structured-export.md ← 能力四：结构化导出
```

## 支持的合同类型

| 类型标识 | 说明 | 适用场景 |
|----------|------|----------|
| `system-integration` | 系统集成/开发合同字段配置（内嵌于 capability-1） | 系统集成/软件开发/技术服务合同 |
| `general` | 通用商业合同字段配置（内嵌于 capability-1） | 采购、租赁、劳务、保密、合作等通用商业合同 |

> **合同类型选择规则**：
> - 用户明确指定类型 → 使用指定配置
> - 未指定 → 默认使用 `system-integration`
> - 若检测到合同内容与默认类型明显不匹配 → 自动切换为 `general`
> - 后续扩展新合同类型只需在 `capability-1-field-extraction.md` 中新增字段配置表即可

## 四大能力一览

| # | 能力名称 | 一句话说明 | 默认输入 | 默认输出 | 详细规则 |
|---|----------|-----------|----------|----------|----------|
| 1 | **字段提取 & 填写清单** | 识别合同占位符，生成分类填写清单 | .docx | .xlsx | `capability-1-field-extraction.md` |
| 2 | **两版合同差异比对** | 对比两份合同，输出差异报告 |  .docx | .xlsx | `capability-2-comparison.md` |
| 3 | **合同信息摘要** | 提取关键信息，生成摘要报告 | .docx | .docx | `capability-3-summary.md` |
| 4 | **数据结构化导出** | 导出为结构化数据 | .docx | .xlsx 或 .json | `capability-4-structured-export.md` |

> **输入输出以用户实际要求为准。** 上表仅为默认示例——用户可在 input 中指定具体的输入文件格式、输出格式、Sheet 结构、列定义、报告模板等，skill 将按用户指定执行。

## 意图路由规则

当用户请求触发本 skill 时，按以下优先级判断用户意图并路由到对应能力：

| 优先级 | 用户意图特征 | 路由到 | 示例关键词 |
|--------|-------------|--------|-----------|
| 1 | 提取字段、填写清单、空白处、占位符 | 能力一 | "提取填写清单"、"合同字段"、"空白处"、"__________" |
| 2 | 对比、差异、比对、修订 | 能力二 | "对比两份合同"、"有什么不同"、"版本比对"、"修订了什么" |
| 3 | 摘要、总结、概要、快速了解 | 能力三 | "合同摘要"、"总结一下"、"快速了解"、"概括" |
| 4 | 导出、结构化、JSON、数据提取 | 能力四 | "导出数据"、"转 JSON"、"结构化提取" |

> 若用户意图同时匹配多个能力 → 按优先级从高到低选择第一个匹配的能力

## 执行流程（通用）

无论路由到哪个能力，均遵循以下通用执行流程：

```
1. 意图识别 → 匹配上述路由表，确定目标能力
2. 解析用户 input → 提取用户指定的输入输出格式要求（若有）
3. 合同类型检测 → 确定使用哪种字段配置
4. 加载能力规则 → 读取 references/ 下对应的 capability 子文档
5. 读取输入文件 → 解析合同内容
6. 执行具体能力逻辑 → 以用户 input 中的格式要求优先，无特殊指定时采用能力子文档中的默认方案
7. 生成输出文件 → 保存到用户指定目录
```

## 兜底总则（适用于所有能力）

以下兜底策略在各能力子文档中有更详细的场景定义，此处为通用原则：

1. **配置降级链**：专用配置匹配失败 → 回退到通用合同配置 → 仍失败则进入无配置/自由提取模式，**绝不因配置问题中断执行**
2. **输出不空原则**：即使合同内容异常（过短、无占位符、零差异等），也必须输出内容而非空文件——至少包含一条说明性记录告知用户当前状况
3. **不编造原则**：提取不到的字段值标记为 `null` / `【未提及】` / `【待填写】` 等，**绝不编造原文中不存在的内容**
4. **明确标注原则**：任何触发了兜底机制的情况，必须在输出结果中用 `⚠️` 前缀显式标注，让用户一目了然
5. **渐进降级**：优先尝试精确模式（逐条款/逐字段），精确模式不可用时自动降级为模糊模式（段落级/语义理解），而非直接报错放弃

## 错误处理（不可恢复的致命错误）

以下情况应终止执行并向用户报错：

1. **输入文件不存在或不支持** → 明确提示用户并提供正确的格式要求
2. **合同文件严重损坏且无法修复** → 尝试诊断原因并反馈
3. **能力二（比对）仅提供一份文件** → 明确提示需要两份合同文件
4. **能力子文档缺失** → 提示 references/ 目录下缺少对应的能力定义文件
