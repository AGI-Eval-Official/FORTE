# 能力三：合同信息摘要生成

## 功能说明

快速阅读合同全文，提取关键信息，生成结构化的合同摘要报告，便于快速掌握合同核心内容。

## 输入输出（默认）

> **用户可在 input 中指定具体的输入文件、输出格式、摘要模板/结构、详略程度等。以下为无特殊指定时的默认行为。**

| 类型 | 格式 | 说明 |
|------|------|------|
| 输入 | .docx | 合同文件 |
| 输出 | .docx | 摘要报告 |

### 默认摘要结构（示例）

当用户未指定摘要模板时，默认按以下结构生成 Word 报告：

```
# 合同摘要报告

一、基本信息        （合同名称、编号、日期、类型、期限）
二、当事方信息      （甲乙方名称、地址、法定代表人、联系方式）
三、交易标的        （标的描述、数量/规模、质量标准）
四、核心商务条款    （总价、支付方式、付款节点、发票）
五、履行安排        （履行期限、交付/验收、里程碑）
六、主要权利义务    （各方核心权利义务概括）
七、违约责任        （违约金/赔偿约定）
八、重要特别条款    （知识产权、保密、争议解决等）
九、附件清单
十、总体评价与关注要点
```

> 若用户在 input 中指定了不同的摘要模板、板块设置、输出格式或详略要求，则以用户指定为准。

## 提取规则

1. 摘要应简洁明了，突出重点，避免大段原文堆砌
2. 涉及金额、日期、期限等关键数据需精确摘录
3. "总体评价与关注要点"应给出有价值的法务视角分析
4. 报告排版整洁，层级分明

## 兜底机制

### 场景一：关键信息大面积缺失
- 缺失字段标注 `【未提及】`，不编造内容；在评价部分首条指出缺失情况

### 场景二：合同为纯模板（大量未填充占位符）
- 占位符字段标注 `【待填写】`；在评价中说明本合同为待填写模板版本

### 场景三：合同内容过长
- 对各板块采取**要点提炼**而非原文摘录；在头部注明为提炼版本

## 执行步骤

1. 解析用户 input → 提取用户指定的输出格式/摘要模板/详略要求（若有），否则采用默认方案
2. 读取合同文件（解析失败则报错终止）
3. 全文解析合同结构和内容
4. 按用户指定或默认的摘要模板提取各板块信息
   - 大面积缺失 → 触发兜底场景一
   - 大量未填充占位符 → 触发兜底场景二
   - 篇幅过长 → 触发兜底场景三
5. 生成总体评价（根据实际情况调整措辞重点）
6. 按用户指定的或默认的格式生成报告并保存
