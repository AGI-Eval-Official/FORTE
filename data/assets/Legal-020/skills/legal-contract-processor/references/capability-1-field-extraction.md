# 能力一：合同字段提取与填写清单生成

## 功能说明

读取合同模板（.docx），识别合同中所有需要填写的字段（以 `__________` 等占位符标记），根据条款位置和上下文语义判断每个字段的含义，从字段配置中匹配填写对象及格式要求，生成结构化的填写清单。

## 输入输出（默认）

> **用户可在 input 中指定具体的输入文件、输出格式、Sheet 结构、列定义等。以下为无特殊指定时的默认行为。**

| 类型 | 格式 | 说明 |
|------|------|------|
| 输入 | .docx | 合同模板文件 |
| 输出 | .xlsx | 填写清单 |

### 默认输出结构（示例）

当用户未指定输出格式时，默认按以下结构生成 Excel：

- 按**填写对象**分类为多个 Sheet（如：汇总表 / 甲方 / 乙方 / 双方 / 系统生成）
- 每行对应一个占位符字段，包含：序号、条款类别、填写内容/约定值、格式或内容要求、模板位置、备注等列
- 汇总表包含所有行的合集，额外增加"填写对象"列

> 若用户在 input 中指定了不同的 Sheet 数量、列定义、分类维度或输出格式，则以用户指定为准。

## 字段提取规则

### 字段配置

根据合同类型使用对应的字段配置进行匹配。每个字段类型包含：

| 字段 | 说明 |
|------|------|
| category | 条款类别的固定名称 |
| format | 格式或内容要求的固定值 |
| remark | 备注的固定值 |
| assignee | 填写对象（甲方/乙方/双方/系统生成），直接使用该值，不得修改 |

提取字段时，根据占位符的上下文语义，从配置中匹配最合适的 `category`，然后使用该 category 对应的 `format` 和 `remark` 作为输出值。`assignee` 作为填写对象，直接使用该值。匹配不到 category 时，填写对象统一为"双方"。

### 合同结构扫描方法

按以下顺序扫描合同全文，确保不遗漏字段：

1. **合同首部**：合同编号、签订日期、甲乙双方基本信息
2. **标的与范围**：合同标的、名称、数量/规模、质量标准
3. **履行与交付**：履行期限、交付时间、里程碑、验收
4. **费用与支付**：合同总价、支付方式、付款节点、发票
5. **权利与知识产权**：知识产权归属、第三方组件
6. **服务与支持**：维护期、响应时间、培训、保修（如有）
7. **保密条款**：保密期限、保密范围
8. **违约责任**：违约金、赔偿上限
9. **其他**：争议解决、合同份数、生效条件、通知方式、附件、不可抗力、终止

### 占位符识别规则

1. **占位符格式**：`__________`（通常 8 个以上下划线）、`( )`、`（ ）`、`___` 等常见空白占位形式
2. **签署页跳过**：签署页中的占位符不提取（签署页通常位于正文之后）
3. **提取硬约束**：除签署页外，所有占位符均输出为一行，不得遗漏
4. **关联判断**：需结合前后文语义判断含义，不能仅凭位置
5. **不合并原则**：每个占位符单独一行，不得合并
6. **区分规则**：同名占位符需根据上下文区分归属（甲/乙方）
7. **条款归类**：同一条款内的占位符统一归入该条款主题对应的 category

## 兜底机制

### 场景一：当前配置中匹配不到 category
- 先尝试跨配置回退到通用合同配置再匹配；仍失败则条款类别和格式填 `/`，备注填原句片段，归入双方

### 场景二：合同中无可识别占位符
- 正常生成输出，但仅含说明性提示行，告知用户未发现占位符

### 场景三：合同文件异常
- 不生成空输出，向用户报错并给出诊断信息

## 执行步骤

1. 解析用户 input → 提取用户指定的输出格式要求（若有），否则采用默认方案
2. 根据合同类型加载对应字段配置（默认系统集成，检测到不匹配则切换通用配置）
3. 读取合同模板（.docx）（解析失败则触发兜底场景三）
4. 遍历全文识别占位符（无占位符则触发兜底场景二）
5. 逐个匹配 category（匹配失败则触发兜底场景一）
6. 按用户指定的或默认的输出结构组织数据
7. 生成输出文件并保存
