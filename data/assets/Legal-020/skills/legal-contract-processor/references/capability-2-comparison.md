# 能力二：两版合同差异比对

## 功能说明

对比两份合同文档（如修订版 vs 原版、我方版 vs 对方版），识别所有差异点，输出结构化比对报告。

## 输入输出（默认）

> **用户可在 input 中指定具体的输入文件、输出格式、报告结构、比对粒度等。以下为无特殊指定时的默认行为。**

| 类型 | 格式 | 说明 |
|------|------|------|
| 输入 A | .docx | 合同文件 A（基准版） |
| 输入 B | .docx | 合同文件 B（对比版） |
| 输出 | .xlsx | 比对报告 |

> 两份文件需放在同一输入目录下，通过文件名区分。

### 默认输出结构（示例）

当用户未指定输出格式时，默认生成包含以下内容的 Excel：

- **差异明细**：每条差异记录差异类型（新增/删除/修改/移动）、所在条款、原文对照、变更影响分析、风险提示、建议措施等
- **变更统计汇总**：按差异类型和条款类型的数量统计

> 若用户在 input 中指定了不同的列定义、报告结构、比对侧重点或输出格式，则以用户指定为准。

## 比对规则

### 比对粒度

1. **条款级**：新增/删除整个条款
2. **段落级**：条款内段落变化
3. **语句级**：句子变化
4. **字词级**：关键词语变化（金额、日期、数值等需特别标注）

### 比对关注点

1. 差异点按在合同中出现的先后顺序排列
2. 数值变化（金额、日期、比例、天数等）需特别标注
3. 变更影响分析从法务角度评估对各方权利义务的影响
4. 实质性变更（涉及金额、责任、期限等核心条款）需给出明确警示

## 兜底机制

### 场景一：两份合同结构差异过大
- 放弃逐条款精确对标，降级为**段落级模糊比对**，在报告中插入模式切换说明

### 场景二：其中一份文件无法解析
- 不生成残缺报告，直接报错并指出问题文件

### 场景三：两份合同完全一致（零差异）
- 输出一致性确认记录，统计项均为 0 并附文字说明

### 场景四：多文件无法确定比对对象
- 自动选取最新修改的两个文件，在报告中注明选取依据

## 执行步骤

1. 解析用户 input → 提取用户指定的输出格式/比对要求（若有），否则采用默认方案
2. 扫描输入目录确定比对文件（文件不足或多文件歧义则触发兜底场景四）
3. 读取两份合同（任一失败则触发兜底场景二）
4. 解析合同结构（结构差异大则触发兜底场景一）
5. 逐条比对识别差异（无差异则触发兜底场景三）
6. 分类标记差异类型，分析影响和风险
7. 生成统计汇总
8. 按用户指定的或默认的输出结构生成报告并保存
