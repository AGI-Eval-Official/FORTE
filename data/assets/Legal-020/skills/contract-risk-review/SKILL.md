# 互联网行业合同高风险条款审查

## 概述

本技能用于审查互联网行业合同中的高风险条款，涵盖八类核心风险点，帮助用户快速识别合同中的不利条款。

**核心特性**：
- 支持多种合同类型，不同类型自动调整审查重点和权重
- 支持甲方/乙方/中立三种审查视角，同一条款在不同视角下风险判定不同
- 支持用户自定义审查规则，用户定义的规则优先于默认规则

## 参数定义

| 参数 | 说明 | 可选值 | 默认值 |
|------|------|--------|--------|
| `contract_type` | 合同类型 | `软件开发` / `SaaS订阅` / `采购` / `技术许可` / `通用` | `通用` |
| `perspective` | 审查视角 | `甲方` / `乙方` / `中立` / `全部` | `全部` |
| `custom_rules` | 用户自定义审查规则 | 文件路径 / 规则文本 / 用户指令中的零散规则描述 | 无 |

**参数获取优先级**：
1. 用户在指令中明确指定
2. 从合同文本内容自动推断（仅限 `contract_type`）
3. 使用默认值

**用户自定义规则优先级**：
- 当用户提供了 `custom_rules`（文件或指令中的规则文本）时，其中定义的判定标准、关键词、风险等级等**覆盖** reference 中对应的默认规则
- 用户未定义的部分，继续使用默认规则
- 即：用户规则 > 默认规则，未定义的沿用默认

## 审查流程

### Step 1：确定参数

1. 读取用户指令，提取 `contract_type`、`perspective`、`custom_rules` 参数
2. 若用户未指定 `contract_type`，根据合同标题和正文内容自动识别（识别规则见 `reference/contract-type-guide.md` 中的"类型识别规则"）
3. 若用户未指定 `perspective`，默认为`全部`，即甲方、乙方、中立三个视角都进行审查；若用户指定了某个视角，仅从该视角审查
4. 若用户提供了 `custom_rules`，加载该规则，后续审查中用户规则优先。`custom_rules` 的输入形式包括：
   - 规则文件路径（如 `input/审查规则.md`）
   - 用户指令中引用的规则文本
   - 用户指令中零散描述的判定标准（如"违约金总额超过合同金额的30%即为高风险"），需从中提取判定条件和关键词，合并到对应风险类别的审查逻辑中

### Step 2：确定审查重点

根据 `contract_type`，从 `reference/contract-type-guide.md` 中获取：
- 重点审查项（高权重，缺项视为审查不完整）
- 一般审查项（标准权重）
- 低频审查项（低权重，可简要审查）
- 特有风险点（该合同类型专属的风险项）

### Step 3：逐项审查

按重点→一般→低频的顺序，对八类风险逐项审查：

1. **知识产权归属** → 读取 `reference/ip-ownership.md` + 用户自定义规则（如有）
2. **验收标准** → 读取 `reference/acceptance-criteria.md` + 用户自定义规则（如有）
3. **付款节奏** → 读取 `reference/payment-schedule.md` + 用户自定义规则（如有）
4. **单方变更权** → 读取 `reference/unilateral-modification.md` + 用户自定义规则（如有）
5. **自动续约** → 读取 `reference/auto-renewal.md` + 用户自定义规则（如有）
6. **责任上限** → 读取 `reference/liability-cap.md` + 用户自定义规则（如有）
7. **管辖法院** → 读取 `reference/jurisdiction.md` + 用户自定义规则（如有）
8. **违约金** → 读取 `reference/penalty.md` + 用户自定义规则（如有）

每项审查流程：
1. 使用该类的关键词库搜索合同文本中的相关条款
2. 根据 `perspective` 选取对应的判定标准（用户自定义优先，其次默认规则）
3. 对匹配到的条款进行风险等级评定（高/中/低）
4. 记录风险类型、原文引用、风险说明

**条款不存在时的处理**：
- 重点审查项：标记为"未约定"，提示为风险（缺失重要条款本身即为风险）
- 一般/低频审查项：标记为"未约定"，视为无风险

### Step 4：生成报告

**输出格式与内容优先级**：
1. 用户指令中对输出格式、输出内容有明确要求的 → **严格按照用户指令执行**，即使与预设格式不同
2. 用户指令中未对输出做要求的 → 按照本文件「输出规范」中的预设格式生成

按上述优先级生成审查报告。

## 输出规范

> **优先级原则**：当用户指令对输出格式（如报告结构、字段、排版）或输出内容（如风险等级定义、是否输出建议、输出语言等）有明确要求时，以用户指令为准，预设的报告结构可全部替换，不必保留未要求的字段；用户未做要求的部分，按本节预设格式输出。

### 报告结构

```
# 《合同名称》合同风险审查报告

## 审查概要
- 合同类型：{contract_type}
- 审查视角：{perspective}
- 规则来源：{默认规则 / 用户自定义规则 + 默认规则}
- 整体风险评级：高 / 中 / 低
- 风险条款总数：N 项（高 X / 中 Y / 低 Z）
- 重点关注：列出所有高风险项的一句话摘要

## 风险条款详情

### 1. {风险类型名称}
- **风险等级**：高 / 中 / 低
- **原文引用**："{合同中的原文内容}"
- **风险说明**：{基于当前视角，说明为什么构成风险}
- **建议**：{基于当前视角的修改建议}

### 2. {风险类型名称}
...

## 未约定条款
- {风险类型名称}：未在合同中发现相关约定{（重点项，建议补充）}

## 审查范围
已审查以下八类风险条款：知识产权归属、验收标准、付款节奏、单方变更权、自动续约、责任上限、管辖法院、违约金
```

### 全部无风险时的输出

当八类风险条款均未发现风险时，输出：
```
《合同名称》合同风险审查报告

审查概要：合同类型 {contract_type}，审查视角 {perspective}，整体风险评级：低
风险条款详情：无
未约定条款：{列出未约定的条款，若无则注明"全部八类风险条款均有约定"}
```

### 风险等级说明

| 等级 | 甲方/乙方视角含义 | 中立视角含义 |
|------|-------------------|--------------|
| 高 | 条款严重不利于当前审查视角方，建议必须修改 | 条款严重缺失或严重不对等，极易产生争议，建议必须修改 |
| 中 | 条款存在不利因素，建议协商调整 | 条款不够完善或不够对等，建议协商完善 |
| 低 | 条款略有不利，可接受但需知悉 | 条款略有不足但基本合理，可接受但需知悉 |

### 整体风险评级规则

- 存在任一高风险项 → 整体评级"高"
- 无高风险项，但存在中风险项 → 整体评级"中"
- 仅存在低风险项或无风险 → 整体评级"低"

## 用户自定义规则格式

用户可通过 `custom_rules` 参数提供自定义规则文件，格式如下：

```markdown
# 自定义审查规则

## 知识产权归属

### 判定标准（甲方视角）
- 高：{用户自定义的高风险判定条件}
- 中：{用户自定义的中风险判定条件}
- 低：{用户自定义的低风险判定条件}

### 判定标准（乙方视角）
- 高：...
- 中：...
- 低：...

### 判定标准（中立视角）
- 高：...
- 中：...
- 低：...

### 额外关键词
- {用户自定义的搜索关键词}

## 验收标准

### 判定标准（甲方视角）
...

### 额外关键词
...

（按需定义其他风险类别，未定义的类别沿用默认规则）
```

**规则合并逻辑**：
- 用户定义了某类风险的判定标准 → 完全替换该类该视角的默认判定标准
- 用户定义了额外关键词 → 追加到默认关键词列表中，追加的关键词与默认关键词同等优先，任一关键词匹配到相关条款均视为命中
- 用户未定义某类风险 → 该类风险沿用 `reference/` 中的默认规则
- 用户以自然语言零散描述了判定条件 → 将描述中的判定条件提取并归入对应风险类别和视角，等同于格式化的自定义规则

## 注意事项

1. **输出格式与内容用户指令优先**：当用户指令对输出格式或内容有明确要求时，必须严格按照用户指令输出，预设格式中的报告结构、字段、风险等级定义等均可被用户指令覆盖；用户未做要求的部分，按预设格式输出
2. 原文引用必须准确，保持原文内容，不得改写
3. 风险类型默认从八类中选择：知识产权归属、验收标准、付款节奏、单方变更权、自动续约、责任上限、管辖法院、违约金。若用户自定义规则中提出了不属于八类的新风险类型，应按用户规则审查并输出，同时在报告中标注为"用户自定义类型"
4. 风险判定必须基于当前审查视角，不同视角的判定结论可能完全相反
5. 用户自定义规则优先于默认规则，未自定义的部分沿用默认规则；用户以自然语言零散描述的规则同样优先于默认规则
6. 当合同类型无法自动识别时，使用"通用"类型，按标准权重审查全部八类风险
7. 若用户指令中指定了输出文件路径和/或文件命名，审查结果应输出至指定路径，按指定命名保存；若用户未指定，则将结果直接输出到对话中
