# 单方变更权

## 风险逻辑

单方变更权允许一方不经对方同意即可修改合同内容。不受约束的变更权可能导致：
- 合同核心条款（价格、范围、标准）被单方修改，对方被动接受
- 合同的确定性被破坏，无法预测实际履约条件
- 变更权成为一方规避违约责任的工具

## 法律依据

- 《中华人民共和国民法典》第五百四十三条：当事人协商一致，可以变更合同。
- 《中华人民共和国民法典》第五百四十四条：当事人对合同变更的内容约定不明确的，推定为未变更。
- 原则上合同变更应经双方协商一致，单方变更权属于例外情形，需有明确约定。

## 判定标准

### 甲方视角（委托方/采购方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 乙方有权单方修改合同核心条款（价格、服务范围、服务等级）且无需甲方同意 |
| 高 | 乙方有权随时变更合同且甲方仅能选择接受或终止（终止仍有损失） |
| 中 | 乙方有权单方变更但需提前通知甲方（通知期短于30天） |
| 中 | 乙方有权单方变更非核心条款（如联系方式、办公地址） |
| 低 | 变更需双方协商一致，任何一方无单方变更权 |

### 乙方视角（开发方/服务方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 甲方有权单方变更需求/范围且不调整合同价格和工期 |
| 高 | 甲方有权无限次变更，乙方必须接受且无额外费用补偿 |
| 中 | 甲方有权变更但需补偿乙方（补偿标准不明确或偏低） |
| 中 | 甲方有权变更但变更流程不清晰（如无书面变更确认机制） |
| 低 | 变更需双方协商一致，变更对价格/工期的影响有明确调整机制 |

### 中立视角

| 等级 | 判定条件 |
|------|---------|
| 高 | 一方有权单方变更合同核心条款且对方无救济途径 |
| 高 | 变更权范围和边界不明确 |
| 中 | 单方变更权存在但附有限制条件（如提前通知、补偿机制） |
| 中 | 变更仅限于非核心条款 |
| 低 | 变更需双方协商一致，有明确的变更管理流程 |

## 合同类型侧重

| 合同类型 | 权重 | 特有风险点 |
|---------|------|-----------|
| 软件开发 | 一般 | 甲方需求变更的管理机制、变更对工期和费用的影响 |
| SaaS订阅 | 重点 | 服务方变更服务内容/价格的权利、变更通知机制、用户退出权 |
| 采购 | 低频 | 采购合同变更通常需双方协商 |
| 技术许可 | 重点 | 许可方变更授权范围/许可费的权利、被许可方的应对措施 |
| 通用 | 标准 | — |

## 关键词与同义模式

### 核心关键词
- 变更、修改、调整、有权

### 扩展关键词
- 单方、自行、无需同意、不经同意
- 修订、更新、升级、迭代
- 需求变更、范围变更、价格调整
- 通知、提前告知、书面确认
- 补偿、调整合同价格、顺延工期

### 同义模式
- `有权.*单方.*变更/修改/调整`
- `无需.*同意.*变更/修改`
- `自行.*调整.*价格/范围/内容`
- `变更.*不调整.*价格/工期`
- `提前.*天通知.*后生效`
