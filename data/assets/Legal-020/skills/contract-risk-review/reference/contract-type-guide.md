# 合同类型审查指南

## 概述

本文件定义不同合同类型的审查重点、输出要求及特有风险项。SKILL.md 根据此文件确定审查优先级和必须输出的内容。

## 合同类型一览

| 类型编码 | 类型名称 | 典型场景 |
|---------|---------|---------|
| DEV     | 软件开发 | 定制开发、系统建设、应用开发 |
| SAAS    | SaaS订阅 | 云服务订阅、平台使用、在线服务 |
| PROC    | 采购     | 硬件/服务采购、供应商合作 |
| LIC     | 技术许可 | 专利/商标/软件授权、License |
| GEN     | 通用     | 无法归类时的默认模式 |

> **编码映射**：DEV → `软件开发`，SAAS → `SaaS订阅`，PROC → `采购`，LIC → `技术许可`，GEN → `通用`。与 SKILL.md 中 `contract_type` 参数的中文值一一对应。

## DEV — 软件开发

### 重点审查项（高权重，缺项即视为审查不完整）

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 知识产权归属 | 高 | 开发成果的权属是软件合同的核心争议点 |
| 验收标准 | 高 | 软件交付的质量判定标准直接影响付款 |
| 付款节奏 | 高 | 付款是否与里程碑/验收节点绑定 |

### 一般审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 单方变更权 | 标准 | 需求变更的控制权归属 |
| 违约金 | 标准 | 逾期交付/逾期付款的罚则 |
| 责任上限 | 标准 | 软件缺陷导致的损失赔偿限制 |
| 管辖法院 | 标准 | 争议解决方式 |

### 低频审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 自动续约 | 低 | 软件开发合同通常为一次性项目，较少涉及续约 |

### 特有风险点

- **源码交付与持有方式**：是否交付源代码、源码托管方式（如托管至第三方）
- **二次开发权**：是否允许在交付成果基础上进行二次开发
- **技术文档交付范围**：设计文档、接口文档、部署文档等是否明确交付清单
- **第三方组件/开源协议风险**：是否声明使用的第三方组件及开源协议，是否规避传染性开源协议

### 输出必须包含

- 知识产权条款的完整分析（含源码交付判定）
- 验收流程是否含"沉默视为同意"机制
- 付款是否与验收/里程碑节点绑定
- 是否存在特有风险点中的任一项

## SAAS — SaaS订阅

### 重点审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 自动续约 | 高 | SaaS 合同的典型风险，到期自动续费容易导致被动支出 |
| 单方变更权 | 高 | 服务方单方调整服务内容、价格的权利 |
| 责任上限 | 高 | 服务中断/数据丢失的赔偿限制 |

### 一般审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 知识产权归属 | 标准 | 数据归属和平台内容的权利 |
| 付款节奏 | 标准 | 订阅费用的支付周期和方式 |
| 管辖法院 | 标准 | 争议解决方式 |

### 低频审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 验收标准 | 低 | SaaS 通常不涉及验收环节 |
| 违约金 | 低 | 较少约定违约金，多为服务等级承诺 |

### 特有风险点

- **SLA 服务等级承诺及违约后果**：是否有可用性承诺，未达标时的补偿方式
- **数据归属与数据退出机制**：合同终止后数据如何导出、格式是否通用
- **服务中止/终止后的数据处理**：数据保留期限、销毁方式
- **数据安全与合规义务**：数据存储地、安全认证、合规责任

### 输出必须包含

- 自动续约的触发条件与退出方式
- 变更权是否覆盖价格/服务范围的变更
- 数据退出方案是否明确
- 是否存在特有风险点中的任一项

## PROC — 采购

### 重点审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 验收标准 | 高 | 采购品的质量判定标准 |
| 付款节奏 | 高 | 付款是否与交付/验收绑定 |
| 违约金 | 高 | 逾期交付/质量不合格的罚则 |

### 一般审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 责任上限 | 标准 | 质量问题的赔偿限制 |
| 管辖法院 | 标准 | 争议解决方式 |
| 知识产权归属 | 标准 | 采购品涉及的知识产权归属 |

### 低频审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 自动续约 | 低 | 一次性采购通常不涉及续约 |
| 单方变更权 | 低 | 采购合同变更通常需双方协商 |

### 特有风险点

- **质保期及质保责任**：质保期限、质保范围、质保期内的响应要求
- **退换货条件与流程**：什么条件下可退货/换货，流程和费用承担
- **供应商资质与合规**：是否要求供应商具备特定资质或许可

### 输出必须包含

- 验收不合格的处理路径
- 付款是否与验收/质保挂钩
- 违约金是否覆盖逾期交付和质量不合格
- 是否存在特有风险点中的任一项

## LIC — 技术许可

### 重点审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 知识产权归属 | 高 | 许可范围和权利边界是核心 |
| 付款节奏 | 高 | 许可费的支付方式和周期 |
| 单方变更权 | 高 | 许可方是否可单方调整授权范围或费用 |

### 一般审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 违约金 | 标准 | 超范围使用的罚则 |
| 管辖法院 | 标准 | 争议解决方式 |
| 责任上限 | 标准 | 许可技术缺陷的赔偿限制 |

### 低频审查项

| 风险类别 | 权重 | 说明 |
|---------|------|------|
| 验收标准 | 低 | 技术许可通常不涉及验收 |
| 自动续约 | 低 | 部分许可涉及续期，但非普遍 |

### 特有风险点

- **许可范围边界**：地域范围、使用期限、用途限制、授权人数/设备数
- **再许可与转授权限制**：是否允许再许可或转授权给第三方
- **许可方权利终止条件**：在什么条件下许可方可终止授权
- **衍生成果归属**：基于许可技术开发的新成果归谁所有

### 输出必须包含

- 许可范围的边界是否清晰（地域/期限/用途/人数）
- 许可终止后的处置方式（停止使用、数据删除等）
- 付款是否与授权范围变更联动
- 是否存在特有风险点中的任一项

---

## GEN — 通用

### 审查策略

八类风险全部按标准权重审查，无重点/一般/低频之分，无特有风险点。

### 输出要求

按标准输出规范生成报告，不额外要求特定内容。

## 类型识别规则

根据合同标题和正文内容自动识别合同类型，按以下规则匹配：

| 匹配关键词 | 推荐类型 | 置信度 |
|-----------|---------|--------|
| 开发、定制、系统建设、应用开发、软件研发 | DEV | 高 |
| SaaS、订阅、云服务、平台使用、在线服务、会员 | SAAS | 高 |
| 采购、供货、供应商、购买、采购订单 | PROC | 高 |
| 许可、专利授权、商标许可、License授权、软件授权 | LIC | 高 |
| 无法匹配以上任一类型 | GEN | — |

**识别规则**：
1. 优先匹配合同标题中的关键词
2. 若标题无法识别，扫描正文前 500 字
3. 若多种类型均有匹配，选择匹配关键词最多的类型
4. 若仍无法判断，使用 GEN（通用）
5. 自动识别结果仅供参考，用户明确指定的类型优先
