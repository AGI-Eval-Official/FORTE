# 知识产权归属

## 风险逻辑

知识产权归属决定了合同标的物（软件、技术、内容等）的权利归属。权属不明或归属对方，可能导致：
- 委托方无法自由使用、修改、二次开发成果
- 开发方无法将成果复用于其他项目
- 双方共有但行权规则不明，导致实际无法行使权利

## 法律依据

- 《中华人民共和国著作权法》第十九条：受委托创作的作品，著作权的归属由委托人和受托人通过合同约定。合同未作明确约定或者没有订立合同的，著作权属于受托人。
- 《计算机软件保护条例》第十一条：接受他人委托开发的软件，其著作权的归属由委托人与受托人签订书面合同约定；无书面合同或者合同未作明确约定的，其著作权由受托人享有。
- 《中华人民共和国专利法》第八条：两个以上单位或者个人合作完成的发明创造，除另有协议外，申请专利的权利属于共同完成的单位或者个人。

## 判定标准

### 甲方视角（委托方/采购方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 知识产权完全归乙方所有，甲方仅获得使用权 |
| 高 | 未作知识产权归属约定（法律默认归开发方/受托方） |
| 高 | 约定双方共有但未明确行权规则（如谁可许可第三方、收益如何分配） |
| 中 | 甲方享有所有权但乙方保留部分权利（如乙方享有底层框架/平台的知识产权） |
| 中 | 甲方享有所有权但乙方享有免费/优先的商业化使用权 |
| 低 | 甲方享有完全的知识产权，乙方仅在项目范围内有权使用 |

### 乙方视角（开发方/服务方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 知识产权完全归甲方所有，乙方无法复用任何成果（含底层框架和通用模块） |
| 高 | 要求乙方将已有技术/框架的知识产权也转让给甲方 |
| 中 | 甲方享有所有权但乙方保留底层框架/通用模块的权利（合理但有争议空间） |
| 中 | 约定双方共有但行权规则不利于乙方（如乙方不可自行商业化） |
| 低 | 乙方保留知识产权，甲方获得使用许可 |
| 低 | 未作约定（法律默认归乙方） |

### 中立视角

| 等级 | 判定条件 |
|------|---------|
| 高 | 未作知识产权归属约定（权属不明，易生争议） |
| 高 | 约定双方共有但未明确行权规则 |
| 中 | 归属明确但行权规则不完善 |
| 低 | 归属明确且行权规则完善 |

## 合同类型侧重

| 合同类型 | 权重 | 特有风险点 |
|---------|------|-----------|
| 软件开发 | 重点 | 源码交付与持有方式、二次开发权、第三方组件/开源协议风险 |
| SaaS订阅 | 一般 | 数据归属、平台内容权利、用户生成内容的知识产权 |
| 采购 | 标准 | 采购品涉及的知识产权（如预装软件的授权） |
| 技术许可 | 重点 | 许可范围边界、再许可与转授权限制、衍生成果归属 |
| 通用 | 标准 | — |

## 关键词与同义模式

### 核心关键词
- 知识产权、著作权、版权、源代码、所有权、专利、商标

### 扩展关键词
- 商业秘密、专有技术、许可使用、授权范围、转让、归属
- 复用、二次开发、底层框架、通用模块、组件
- 开源、开源协议、第三方组件、GPL

### 同义模式
- `权利.*归属`、`知识产权.*所有`、`著作权.*享有`
- `源码.*交付`、`源代码.*提供`
- `未.*约定.*归.*所有`（识别默认规则触发场景）
- `双方共有`、`共同享有`、`联合所有`
