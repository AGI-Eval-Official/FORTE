# 付款节奏

## 风险逻辑

付款节奏决定了资金的安全性和杠杆效应。不合理的付款安排可能导致：
- 委托方在验收前支付全部款项，丧失付款杠杆
- 开发方/供应商在交付后无法及时收款，资金压力大
- 付款与交付/验收节点脱钩，无法约束对方履约

## 法律依据

- 《中华人民共和国民法典》第五百零九条：当事人应当按照约定全面履行自己的义务。
- 《中华人民共和国民法典》第五百二十六条：当事人互负债务，有先后履行顺序，应当先履行债务一方未履行的，后履行一方有权拒绝其履行请求。
- 《中华人民共和国民法典》第五百二十七条：应当先履行债务的当事人，有确切证据证明对方有丧失或者可能丧失履行债务能力的情形，可以中止履行。

## 判定标准

### 甲方视角（付款方/委托方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 验收合格前需付清全部款项 |
| 高 | 预付款比例过高（超过合同总额的50%）且无退款保障 |
| 高 | 付款节点与交付/验收里程碑完全不挂钩 |
| 中 | 尾款比例过低（低于合同总额的10%），验收后付款的约束力不足 |
| 中 | 付款条件仅以时间节点触发，不与实际交付进度关联 |
| 中 | 无逾期付款的宽限期或违约责任约定 |
| 低 | 付款节奏合理（如按预付款30%-进度款40%-验收款20%-质保金10%分期），与里程碑/验收绑定 |

### 乙方视角（收款方/开发方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 付款严重滞后于交付（如交付后90天以上才付款） |
| 高 | 尾款比例过高且付款条件依赖甲方主观判断 |
| 高 | 甲方可以"未满意"等模糊理由拒绝付款且无时限 |
| 中 | 预付款比例过低（低于合同总额的20%） |
| 中 | 付款周期过长，无明确付款时限 |
| 中 | 缺乏甲方逾期付款的违约金条款 |
| 低 | 付款节奏合理，节点清晰，付款时限明确 |

### 中立视角

| 等级 | 判定条件 |
|------|---------|
| 高 | 付款与交付/验收完全脱钩 |
| 高 | 付款条件由单方完全控制 |
| 中 | 付款节奏对一方明显不利但尚可接受 |
| 中 | 缺乏逾期付款/逾期交付的对应罚则 |
| 低 | 付款节点清晰，与里程碑对齐，双方义务对等 |

## 合同类型侧重

| 合同类型 | 权重 | 特有风险点 |
|---------|------|-----------|
| 软件开发 | 重点 | 里程碑付款（需求确认/设计确认/开发完成/验收通过）、预付款与尾款比例 |
| SaaS订阅 | 标准 | 订阅周期（月付/年付）、自动扣款、提前终止的退费规则 |
| 采购 | 重点 | 到货款/验收款/质保金的比例、质保金返还条件 |
| 技术许可 | 重点 | 入门费/许可费/分成费的支付条件、授权范围变更时的费用调整 |
| 通用 | 标准 | — |

## 关键词与同义模式

### 核心关键词
- 付款、支付、尾款、预付款

### 扩展关键词
- 结算、分期、进度款、里程碑、阶段款
- 首付款、定金、保证金、质保金
- 到货款、验收款、余款
- 扣款、抵扣、发票

### 同义模式
- `验收.*前.*付清/全部支付/结清`
- `付款.*不与.*验收.*挂钩/关联`
- `预付.*比例.*超过.*50%`
- `尾款.*低于.*10%`
- `收到发票后.*天内付款`（识别付款周期）
