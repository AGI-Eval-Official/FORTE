# 验收标准

## 风险逻辑

验收标准决定了合同标的物是否达到交付要求，直接影响付款条件是否触发。不合理或缺失的验收标准可能导致：
- 委托方被迫接受不符合要求的交付成果
- 开发方交付后无法获得验收确认和付款
- 验收流程缺乏客观标准，双方产生争议

## 法律依据

- 《中华人民共和国民法典》第五百零九条：当事人应当按照约定全面履行自己的义务。
- 《中华人民共和国民法典》第六百一十五条：出卖人应当按照约定的质量要求交付标的物。
- 《中华人民共和国民法典》第六百二十条：买受人收到标的物时应当在约定的检验期限内检验。

## 判定标准

### 甲方视角（委托方/采购方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 存在"沉默视为同意"条款（如甲方逾期未异议则视为验收通过） |
| 高 | 验收标准完全由乙方单方定义或解释 |
| 高 | 无验收标准或验收流程约定 |
| 中 | 验收标准模糊（如仅写"符合行业惯例"而无具体标准） |
| 中 | 验收期限过短，甲方难以充分检验 |
| 中 | 乙方有权在甲方未验收时直接确认验收 |
| 低 | 验收标准明确但略有弹性（如"以甲方合理判断为准"） |

### 乙方视角（开发方/服务方）

| 等级 | 判定条件 |
|------|---------|
| 高 | 甲方验收标准主观且无限制（如"甲方满意为止"） |
| 高 | 甲方可无限次拒绝验收且无需理由 |
| 高 | 无验收时限约定，甲方可无限期拖延验收 |
| 中 | 验收流程繁琐，需经多轮审批但时限不明确 |
| 中 | 验收标准过于严苛或超出行业惯例 |
| 低 | 甲方有权在合理期限内提出异议 |
| 低 | 存在"沉默视为同意"条款（对乙方有利） |

### 中立视角

| 等级 | 判定条件 |
|------|---------|
| 高 | 无验收标准或验收流程约定 |
| 高 | 验收标准完全由单方定义，对方无参与权 |
| 中 | 存在"沉默视为同意"条款（需关注时限是否合理） |
| 中 | 验收标准模糊但双方有协商机制 |
| 低 | 验收标准明确，流程清晰，双方权利义务对等 |

## 合同类型侧重

| 合同类型 | 权重 | 特有风险点 |
|---------|------|-----------|
| 软件开发 | 重点 | 分阶段验收（单元测试→集成测试→系统测试）、试运行期、缺陷修复后的重新验收 |
| SaaS订阅 | 低频 | SaaS 通常不涉及传统验收，关注服务开通确认 |
| 采购 | 重点 | 到货验收与安装验收分离、质保期起始与验收的关系 |
| 技术许可 | 低频 | 技术许可通常不涉及验收，关注授权生效条件 |
| 通用 | 标准 | — |

## 关键词与同义模式

### 核心关键词
- 验收、交付、异议、视为、合格

### 扩展关键词
- 检验、测试、试运行、确认、签收、接收
- 验收申请、验收报告、验收标准
- 缺陷、整改、修复、返工
- 里程碑、阶段性交付

### 同义模式
- `逾期.*未.*异议.*视为.*通过/合格/接受`
- `未.*提出.*视为.*同意/认可/合格`
- `沉默.*视为.*同意`
- `单方.*确认.*验收`
- `满意.*为止`（识别甲方主观验收标准）
