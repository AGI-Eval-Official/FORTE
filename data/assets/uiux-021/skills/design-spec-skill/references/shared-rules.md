# 通用规则

## 目标

输出简洁、可执行、可复用的设计规范文档。

## 写作方式

- 用直接、可落地的表述，不写空泛描述。
- token 命名全篇统一，推荐小写 kebab-case。
- 每条规则尽量可量化、可核对。

## 用户可配置参数

用户可显式指定以下参数，且应优先生效：
- 色彩：主色、功能色、中性色、状态色、暗黑模式色
- 字体：字号、字重、行高、最小正文字号
- 图标：尺寸、线宽、圆角风格
- 间距：基础步长与间距刻度
- 圆角与阴影：层级和具体数值
- 组件：按钮、输入框、卡片等尺寸与状态
- 单位：`dp`、`sp`、`pt`、`rpx`、`px`、`rem`
- 输出格式：`md`、`html`、`docx`、`json`
- 输出内容：章节范围、包含项、排除项、是否输出参数来源说明

## 参数优先级

默认参数仅在用户未指定时生效，优先级如下：
1. 用户显式指定值
2. 用户约束条件（如最小/最大值）
3. 平台规则默认值
4. 通用规则兜底值

## 全局默认值（未指定时使用）

### 输出格式
- 默认输出格式：`md`

### 输出内容
- 默认输出内容：完整章节（适用范围、色彩、字体、图标、间距、圆角与阴影、组件示例、token 映射）

### 颜色
- `color-primary-500`: `#1677FF`
- `color-success-500`: `#16A34A`
- `color-warning-500`: `#F59E0B`
- `color-error-500`: `#DC2626`
- `color-text-primary`: `#1F2937`
- `color-text-secondary`: `#6B7280`
- `color-bg-primary`: `#FFFFFF`
- `color-bg-secondary`: `#F8FAFC`

### 字体
- 字重：`400` / `500` / `600`
- 默认行高系数：`1.5`
- 最小正文字号：`12`（按平台单位解释）

### 图标
- 默认图标尺寸：`16` / `20` / `24`
- 默认图标线宽：`1.5`

### 间距
- 默认间距刻度：`4` / `8` / `12` / `16` / `24` / `32`

### 圆角
- 默认圆角：`4` / `8` / `12`

### 阴影
- `shadow-1`: `0 1 2 0 rgba(0,0,0,0.08)`
- `shadow-2`: `0 4 12 0 rgba(0,0,0,0.12)`

## 输出内容默认流程

若用户未指定输出内容，默认输出完整章节：
1. 适用范围与目标平台
2. 色彩系统
3. 字体系统
4. 图标规范
5. 间距系统
6. 圆角与阴影规范
7. 组件示例
8. 研发 token 映射

若用户指定输出内容范围，以用户指定为准。

## 命名规范

采用如下命名模式：
- 颜色：`color-{group}-{level}`，例如 `color-primary-500`
- 字号：`font-size-{name}`，例如 `font-size-body-m`
- 间距：`space-{step}`，例如 `space-16`
- 圆角：`radius-{name}`，例如 `radius-m`
- 阴影：`shadow-{level}`，例如 `shadow-2`

## 质检清单

- 命名是否前后一致
- 尺度是否成体系且可复用
- 正文字号是否清晰可读
- 交互控件是否定义状态样式
- 色彩语义是否清晰
- 单位是否与目标平台一致
- 用户指定参数是否被完整保留
- 未指定字段是否正确使用默认值
- 输出格式是否符合用户指定或默认 `md`
- 输出内容范围是否符合用户指定或默认完整流程
