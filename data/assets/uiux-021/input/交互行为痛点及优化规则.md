# 交互行为痛点及优化规则

## 一、痛点严重程度识别

| 痛点类型 | 示例 | 严重程度 |
| --- | --- | --- |
| 操作卡顿 | · 首页加载时间超过5秒，Banner图片和功能入口均未响应<br>· 展开章节目录时列表渲染卡顿，滚动不流畅<br>· 输入笔记内容时键盘响应延迟，文字出现约500ms后才显示<br>· 切换视图时动画卡顿，等待约2秒才完成切换 | 严重 |
| 渲染失败 | · 翻页后新页面白屏，内存不足导致渲染失败<br>· 书架页加载时内存溢出，大封面图片导致App崩溃<br>· 搜索结果书籍封面图片全部加载失败<br>· 日历热力图加载后颜色全部显示为灰色，数据未渲染 | 严重 |
| 反馈迟钝 | · 切换开关后状态变化动画延迟，用户重复操作确认<br>· 点击保存按钮后Toast提示"已保存"出现延迟超过500ms<br>· 数据加载时骨架屏出现延迟超过300ms，页面短暂白屏<br>· 翻页手势结束后页面跟随动画延迟，非实时响应导致体验割裂 | 严重 |
| 操作失败 | · 字体大小调整后未生效，页面仍显示原字号<br>· 点击书签按钮后书签未保存，重新进入页面书签消失<br>· 笔记内容为空时点击保存无任何提示，操作无反馈<br>· 输入关键词后点击搜索无结果返回，但实际有匹配内容 | 严重 |
| 跳转失败 | · 点击书籍封面后跳转到书籍详情页而非阅读页<br>· 点击章节目录条目后App无响应，页面未跳转<br>· 点击笔记按钮后跳转到空白笔记列表而非编辑页<br>· 保存成功后未自动返回笔记列表，停留在编辑页 | 严重 |
| 排版错乱 | · 章节正文在部分机型上段落缩进错乱，首行缩进与正文混排<br>· 笔记卡片摘录文字在多行时出现行间距不均匀<br>· 统计数字在不同字号设置下出现数值与单位错位<br>· 个人信息区昵称过长时文字溢出头像区域边界 | 中等 |
| 视觉抖动 | · 快速翻页时页面内容出现短暂位置跳变，约3px偏移<br>· 首页滚动时Banner区域出现轻微抖动，FPS降至45帧以下<br>· 键盘弹出时页面上移动画出现抖动，内容区域位置不稳定<br>· 图表渲染完成时出现短暂抖动，数据点位置跳变 | 中等 |
| 文案截断 | · 工具栏按钮文字标签在部分机型上被截断，功能名称不完整<br>· 书名超过2行时省略号位置错误，出现在行中而非行尾<br>· 设置项说明文字在小屏设备上被截断，用户无法了解完整功能说明<br>· 首页功能入口图标下方文字标签仅显示前2个字 | 中等 |
| 动效缺失 | · 菜单项点击后跳转缺少转场动画，页面切换生硬<br>· 书籍简介展开/收起时缺少高度变化动画，内容突然出现<br>· 标签选择器底部Sheet弹出时缺少上滑动画，直接出现<br>· 开关Toggle切换时缺少滑动动画，状态突然变化 | 轻微 |

## 二、场景频次分档定义

> 场景频次 = 某"页面 × 操作动作"组合在全量日志中的出现次数 ÷ 总操作次数。高/中/低频档位依据以下定义表确定。

| 场景档位 | 占比阈值 | 计算方式 |
| --- | --- | --- | 
| 高频场景 | 单场景操作占比 ≥ 5% | 统计"页面×操作动作"组合在全量日志中的出现次数，除以总操作次数，结果 ≥ 5% 即为高频 |
| 中频场景 | 3% ≤ 单场景操作占比 < 5% | 同上，结果在 (3%, 5%) 区间内即为中频 | 
| 低频场景 | 单场景操作占比 < 3% | 同上，结果 < 3% 即为低频 | 

---

## 三、优化优先级矩阵

> 场景频次 = 该"页面 × 操作动作"组合在全量日志中的出现次数 ÷ 总操作次数。严重程度取痛点识别表中对应类型的定级。

| 场景频次 \ 严重程度 | 严重 | 中等 | 轻微 |
| --- | --- | --- | --- |
| **高频场景**（占比 ≥ 5%） | **P0** 当前版本立即修复，阻塞发布 | **P1** 本迭代内修复，不可延期 | **P2** 下迭代修复，需排入计划 |
| **中频场景**（3% ≤ 占比 < 5%） | **P1** 本迭代内修复，不可延期 | **P2** 下迭代修复，需排入计划 | **P3** 排期修复，可延期一个版本 |
| **低频场景**（占比 < 3%） | **P2** 下迭代修复，需排入计划 | **P3** 排期修复，可延期一个版本 | **P4** 积压处理，视资源决定 |
