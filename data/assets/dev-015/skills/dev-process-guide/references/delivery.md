# 交付上线规范

## 一、上线前检查

### 1.1 代码检查

- [ ] 所有代码已通过 Code Review 并合入目标分支
- [ ] 无未解决的 Review 意见（Must Fix 级别）
- [ ] 单元测试全部通过
- [ ] 代码中无调试信息和临时代码
- [ ] 依赖版本已锁定

### 1.2 测试检查

- [ ] 自测用例全部通过
- [ ] 核心功能路径验证通过
- [ ] 边界场景和异常场景验证通过
- [ ] 回归测试通过（确认未影响现有功能）

### 1.3 文档检查

- [ ] 接口文档已更新
- [ ] 配置变更已记录
- [ ] 数据库变更脚本已准备
- [ ] 上线步骤已明确

## 二、上线流程

### 2.1 上线步骤

1. 确认上线分支和版本号
2. 执行数据库变更（如有）
3. 部署应用到目标环境
4. 执行冒烟测试（核心功能快速验证）
5. 监控系统指标（错误率、响应时间、资源使用）
6. 确认上线成功

### 2.2 回滚方案

每次上线必须准备回滚方案：

- 明确回滚触发条件
- 准备回滚操作步骤
- 确认数据库变更是否可回滚
- 预估回滚所需时间

## 三、交付报告

上线完成后应产出交付报告，包含：

- 需求概述和交付范围
- 技术方案摘要
- 测试覆盖情况（单测覆盖率、自测通过率）
- Code Review 结论
- 上线时间和验证结果
- 遗留问题和后续计划
