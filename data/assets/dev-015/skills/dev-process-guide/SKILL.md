## 研发流程总览

完整的研发流程包含以下阶段，每个阶段都有明确的准入条件和交付产物：

1. **需求分析** → 产出需求理解文档，明确功能边界和验收标准
2. **技术方案设计** → 产出技术方案文档，包含架构设计、接口定义、数据模型
3. **编码开发** → 按照编码规范完成功能实现，遵循分支管理策略
4. **单元测试** → 编写单测用例，真实执行并确保全部通过
5. **Code Review** → 代码评审，发现问题并修复
6. **自测验证** → 本地启动服务，通过真实 HTTP 请求逐项验证功能正确性
7. **交付上线** → 完成上线检查清单，生成交付报告

## 通用原则

**必须遵守：**
- 每个阶段必须有明确的交付产物
- 代码提交前必须通过本地单测
- PR/MR 必须经过至少一人 Code Review
- 自测必须覆盖所有核心功能路径和边界场景
- 上线前必须完成回归验证

**禁止操作：**
- 禁止跳过 Code Review 直接合入主分支
- 禁止在没有单测的情况下提交核心业务逻辑
- 禁止未经自测直接提交上线
- 禁止在主分支上直接开发

## 复杂度分级

| 等级 | 说明 | 必要产物 |
|------|------|---------|
| L（轻量级） | 简单 bug 修复、配置变更、文案修改 | 需求分析 + 编码 + 单测 + 自测 |
| M（标准级） | 常规功能开发、接口新增/修改 | L级 + 技术方案 + 接口文档 + Code Review 报告 |
| H（完整级） | 复杂功能、架构变更、跨模块改动 | M级 + 详细设计文档 + 任务拆解 + 性能评估 |

> 所有操作前请先理解需求全貌，开发过程中保持代码整洁和可维护性。
