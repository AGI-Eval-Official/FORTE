# 评测平台产品需求文档（PRD）

## 1. 项目概述

### 1.1 背景

随着团队 AI 模型数量的增长和应用场景的多样化，需要一个统一的评测平台来管理模型、评测数据集和评测实验，以系统化地评估模型效果，支撑模型选型和迭代优化决策。

### 1.2 目标

构建一个评测平台，提供以下核心能力：

- **模型管理**：统一管理待评测的 AI 模型，支持模型信息的录入、查看和维护
- **评测集管理**：管理评测数据集，支持评测集的创建、导入、查看和版本管理
- **评测实验**：支持发起评测实验（将模型与评测集组合执行评测），查看评测结果和历史记录

### 1.3 用户角色

- **平台管理员**：管理平台配置、用户权限
- **评测工程师**：创建和管理模型、评测集，发起和查看评测实验
- **观察者**：查看评测结果和报告，无编辑权限

## 2. 功能需求

### 2.1 模型管理

#### 2.1.1 模型列表

- 以表格形式展示所有已注册的模型
- 表格列：模型名称、模型类型、版本号、描述、创建人、创建时间、状态
- 支持按模型名称搜索
- 支持按模型类型、状态筛选
- 支持按创建时间排序
- 分页展示，每页默认 20 条

#### 2.1.2 创建模型

- 必填字段：模型名称（唯一，最长 100 字符）、模型类型（下拉选择：LLM / 分类模型 / 回归模型 / 其他）
- 选填字段：版本号（格式 vX.Y.Z）、描述（最长 500 字符）、模型端点 URL、API Key（加密存储）
- 创建成功后跳转到模型详情页
- 模型名称不可与已有模型重复，重复时提示"模型名称已存在"

#### 2.1.3 模型详情

- 展示模型的完整信息
- 展示该模型关联的评测实验历史（最近 10 条）
- 支持编辑模型信息（名称不可修改）
- 支持删除模型（如有关联的进行中评测实验，禁止删除，提示"该模型有进行中的评测实验，无法删除"）

#### 2.1.4 编辑模型

- 除模型名称外，其他字段均可编辑
- 编辑后保存，更新修改时间和修改人

#### 2.1.5 删除模型

- 删除前弹出确认对话框
- 如有关联的进行中评测实验，禁止删除
- 删除为软删除，数据保留但不再展示

### 2.2 评测集管理

#### 2.2.1 评测集列表

- 以表格形式展示所有评测集
- 表格列：评测集名称、数据量（条数）、版本号、描述、创建人、创建时间
- 支持按名称搜索
- 支持按创建时间排序
- 分页展示，每页默认 20 条

#### 2.2.2 创建评测集

- 必填字段：评测集名称（唯一，最长 100 字符）
- 选填字段：版本号（格式 vX.Y.Z）、描述（最长 500 字符）
- 数据导入方式：上传 JSON 文件（单文件最大 50MB）
- JSON 文件格式要求：数组格式，每个元素包含 `input`（字符串，必填）和 `expected_output`（字符串，选填）字段
- 上传时进行格式校验，校验失败提示具体错误（如"第 N 行缺少 input 字段"）
- 创建成功后跳转到评测集详情页

#### 2.2.3 评测集详情

- 展示评测集基本信息
- 以表格形式展示评测数据（分页，每页 50 条）
- 表格列：序号、input、expected_output
- 支持预览单条数据的完整内容（点击展开）
- 展示该评测集关联的评测实验历史

#### 2.2.4 编辑评测集

- 支持修改评测集名称、版本号、描述
- 支持追加导入数据（上传新的 JSON 文件，数据追加到已有数据后）
- 支持清空并重新导入数据

#### 2.2.5 删除评测集

- 删除前弹出确认对话框
- 如有关联的进行中评测实验，禁止删除
- 删除为软删除

### 2.3 评测实验

#### 2.3.1 评测实验列表

- 以表格形式展示所有评测实验
- 表格列：实验名称、关联模型、关联评测集、状态（待执行/执行中/已完成/失败）、创建人、创建时间、完成时间
- 支持按实验名称搜索
- 支持按状态筛选
- 支持按创建时间排序
- 分页展示，每页默认 20 条

#### 2.3.2 发起评测实验

- 必填字段：实验名称（最长 100 字符）、选择模型（下拉选择已注册的模型）、选择评测集（下拉选择已有的评测集）
- 选填字段：描述（最长 500 字符）、并发数（默认 5，范围 1-20）、超时时间（默认 30 秒，范围 5-300 秒）
- 点击"发起评测"后，实验状态变为"执行中"
- 评测过程：逐条取出评测集中的数据，将 input 发送给模型端点，记录模型返回的 output 和响应时间
- 如模型端点不可达或返回错误，该条标记为"失败"并记录错误信息，继续执行下一条
- 所有数据执行完毕后，实验状态变为"已完成"

#### 2.3.3 评测实验详情

- 展示实验基本信息（名称、模型、评测集、状态、时间等）
- 展示评测结果统计：总条数、成功条数、失败条数、平均响应时间、P50/P90/P99 响应时间
- 以表格形式展示逐条评测结果（分页，每页 50 条）
- 表格列：序号、input、expected_output、actual_output、响应时间（ms）、状态（成功/失败）
- 支持按状态筛选（只看成功/只看失败）
- 支持导出评测结果为 JSON 文件

#### 2.3.4 取消评测实验

- 仅"执行中"状态的实验可取消
- 取消后状态变为"已取消"，已执行的结果保留

## 3. 非功能性需求

### 3.1 性能要求

- 列表页加载时间 < 2 秒
- 评测实验支持最大 10000 条数据的评测集
- 评测结果导出支持最大 50MB 文件

### 3.2 数据安全

- API Key 加密存储，接口返回时脱敏显示
- 操作日志记录（创建、修改、删除操作）
- 软删除机制，数据可恢复

### 3.3 可用性

- 界面响应式设计，支持 1280px 及以上分辨率
- 操作反馈及时（加载状态、成功/失败提示）
- 表单校验实时提示
