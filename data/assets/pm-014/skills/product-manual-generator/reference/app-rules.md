# APP 平台补充规则

> 本文件定义了 APP（iOS / Android）类产品的功能说明书中，需要特别说明的平台特性和注意事项。
> 与通用模板 `product_manual_rules.md` 配合使用。

---

## 一、需要在说明书中额外覆盖的内容

### 1. 设备与系统要求

在「产品环境要求」中重点说明：

- **操作系统及版本**：
  | 平台 | 最低版本 | 推荐版本 | 备注 |
  |------|----------|----------|------|
  | iOS | iOS 14.0+ | 最新稳定版 | 需注明是否支持 iPad |
  | Android | Android 8.0+（API 26） | Android 12+ | 需注明是否适配平板 |

- **硬件要求**：
  - 存储空间（安装包大小 + 运行时缓存空间）
  - 内存建议（RAM）
  - 特殊硬件依赖（如：需要摄像头/NFC/蓝牙/GPS/指纹/面容 ID）

- **屏幕适配**：
  - 支持的设备类型（手机 / 平板 / 折叠屏）
  - 横屏 / 竖屏支持情况
  - 异形屏（刘海/药丸/挖孔）的适配说明（如有 UI 影响）

### 2. 安装与更新

| 项目 | 说明要点 |
|------|----------|
| 下载渠道 | App Store / 应用宝 / 各品牌官方市场 / APK 直装 / TestFlight 等 |
| 安装步骤 | 从各渠道下载到安装完成的简要流程 |
| 安装权限 | 首次安装时需要授予的系统权限列表及说明 |
| 更新方式 | 自动更新 / 手动检查更新 / 强制更新的策略 |
| 更新大小 | 一般增量更新的大小范围，帮助用户判断是否在流量环境下更新 |
| 卸载后重装 | 数据是否能恢复、如何恢复 |

### 3. 权限说明（重要！APP 必须写）

APP 涉及大量系统权限，必须在说明书中逐一说明。建议以表格形式呈现：

| 权限 | 用途 | 是否必须 | 拒绝后的影响 |
|------|------|----------|--------------|
| 相机 | 扫码、拍照上传头像 | 可选（扫码/拍照时才需要） | 无法使用扫码和拍照功能 |
| 相册 | 选择图片上传、保存图片 | 可选 | 无法选择相册图片 |
| 定位 | 获取附近门店/打卡 | 视功能而定 | 无法使用基于位置的服务 |
| 通知 | 接收消息推送 | 推荐 | 收不到推送通知 |
| 麦克风 | 语音输入/语音通话 | 可选 | 无法使用语音功能 |
| 通讯录 | 邀请好友/关联联系人 | 可选 | 无法快捷邀请联系人 |
| 蓝牙 | 连接外部设备 | 视功能而定 | 无法连接外设 |
| NFC | 刷卡/标签感应 | 视功能而定 | 无法使用 NFC 功能 |
| 悬浮窗 | 显示悬浮球/全局快捷入口 | 可选 | 无法使用悬浮窗功能 |

**权限说明的写作原则：**
- 用用户能理解的语言解释"为什么要这个权限"
- 明确区分"必须授权"和"可选授权"
- 告知用户拒绝授权后的实际影响（而不是笼统地说"可能影响使用"）

### 4. 账号与登录

| 项目 | 说明要点 |
|------|----------|
| 注册方式 | 手机号 / 邮箱 / 第三方（微信/Apple/Google） |
| 登录方式 | 密码 / 验证码 / 第三方 / 生物识别（指纹/面容） |
| 设备绑定 | 账号是否绑定设备、换机后如何迁移 |
| 多设备登录 | 同账号在多台设备上同时登录的行为和数据同步 |
| 退出登录 | 退出后数据的本地保留情况（消息记录/缓存等） |
| 注销账号 | 账号注销的入口、条件和后果（合规要求） |

### 5. 推送通知

- 推送消息的类型分类（交易类 / 营销类 / 系统类）
- 如何管理推送偏好（全局关闭 / 分类关闭）
- 推送不收到的排查方向（系统设置中检查通知权限）

### 6. 存储与缓存

- APP 占用的存储空间构成（安装包 / 缓存 / 用户数据 / 其他）
- 如何清理缓存
- 离线可用功能及离线数据的范围
- 存储空间不足时的表现和解决方法

## 二、APP 特有的交互方式说明

### 手势操作

如果 APP 使用了手势操作，必须在附录或对应功能中列出：

| 手势 | 触发区域 | 功能 | 适用页面 |
|------|----------|------|----------|
| 左滑 | 列表项 | 删除/归档/更多操作 | 消息列表/订单列表等 |
| 右滑 | 列表项 | 标记已读/回复等 | 消息列表 |
| 下拉 | 页面顶部 | 刷新数据 | 大多数列表页 |
| 长按 | 图标/文字/图片 | 进入编辑模式/弹出菜单 | 桌面图标/聊天消息/图片 |
| 双指缩放 | 图片/地图/文档 | 放大缩小 | 图片预览/地图/文档查看 |
| 侧滑返回 | 屏幕左边缘 | 返回上一页 | 所有页面（iOS） |

### 系统手势冲突说明

- APP 内手势是否可能与系统手势冲突
- 如何区分 APP 手势和系统手势（如：iOS 左边缘滑动返回 vs APP 内侧滑菜单）

### 键盘与输入

| 场景 | 说明要点 |
|------|----------|
| 自定义键盘 | 是否有自定义键盘布局（如金额输入/验证码输入） |
| 键盘类型 | 数字键盘 / 带小数点的数字键盘 / 邮箱键盘 / URL 键盘等自动切换 |
| 键盘遮挡 | 键盘弹出时页面的自适应行为（是否自动上推内容） |
| 输入法切换 | 中英文切换、 emoji 输入对表单的影响 |

## 三、iOS vs Android 差异说明（如产品同时有两个版本）

如果产品同时有 iOS 和 Android 版本，且两个版本存在功能或体验差异，需在第 4 章中说明：

| 差异维度 | iOS | Android |
|----------|-----|---------|
| 生物识别 | Face ID / Touch ID | 指纹识别 |
| 推送通道 | APNs | FCM / 各厂商通道 |
| 分享渠道 | 系统分享面板 | 系统分享面板 + 微信/QQ/微博 SDK |
| 支付方式 | Apple Pay / 微信支付 / 支付宝 | 微信支付 / 支付宝 / 银联 |
| 文件管理 | 沙盒隔离 | 可访问公共存储目录 |
| 通知分组 | 按应用分组 | 按通知渠道分组 |
| 深链 Universal Link | Universal Links | App Links / Intent Filter |
| 后台运行 | 严格限制 | 相对宽松但仍有省电策略 |

**注意**：只列出用户能感知到的差异，不需要罗列底层技术差异。

- ❌ 热修复 / 热更新方案
- ❌ App Store 审核规则
- ❌ 签名证书 / 包名
- ❌ 原型图 / 设计稿标注（Figma/Sketch 文件）

## 五、截图与示例建议

| 截图类型 | 使用场景 | 注意事项 |
|----------|----------|----------|
| 手机实机截图 | 主要功能界面展示 | 使用真机截图而非模拟器，状态栏时间真实 |
| 操作步骤截图 | 分步教程 | 每步配一张，关键操作区域高亮 |
| 手势示意图 | 手势操作说明 | 用箭头/动画轨迹示意滑动方向 |
| 状态对比截图 | 不同状态的 UI 变化 | 如：未读vs已读、展开vs收起 |
| 多机型截图 | 不同设备的适配效果 | iPhone / Android / iPad 各一张（如适用） |

**截图规范：**
- 使用标准机型截图（如 iPhone 15 / 华为 Mate 60 / 小米 14 等主流机型）
- 状态栏显示正常的时间、电量、信号
- 隐藏敏感信息（个人数据、聊天内容、隐私信息）
- 如果是暗色模式适配的产品，提供明暗两套截图
