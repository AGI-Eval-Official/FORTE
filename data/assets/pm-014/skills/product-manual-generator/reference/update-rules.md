# 产品说明书更新规则

本文档定义产品功能说明书的三种更新模式及其执行规范。

> **前置依赖**：执行本文档规则前，需先读取 `product_manual_rules.md` 获取通用章节结构、编写规范和自检清单。

---

## 更新模式概览

| 模式 | 触发场景 | 输入 | 输出 |
|------|----------|------|------|
| 模式一：PRD→手册更新 | 有 MEP 需求链接或 PRD 文本 | PRD 内容 + 现有手册 | 更新后的手册（diff 草稿确认后写入） |
| 模式二：功能 Diff 诊断 | 手册可能过时，需诊断差异 | 产品现状信息 + 现有手册 | 诊断报告（过时/缺失/建议优化） |
| 模式三：用户指令更新 | 用户明确指定更新内容 | 用户指令 + 现有手册 | 按指令更新后的手册 |

---

## 模式一：PRD→手册更新

### 适用场景
- 产品迭代有 MEP 需求链接或 PRD 文档
- 需要将产品变更同步到用户手册
- 希望自动分析影响范围，减少人工梳理

### 输入资料

**必填**：

| 资料类型 | 说明 | 示例 |
|----------|------|------|
| PRD 来源 | MEP 链接或 PRD 文本内容 | MEP-12345 链接 / PRD 文档 |
| 现有手册 | 已有说明书文件路径 | 订单系统_功能说明书_V1.2.docx |

**常见补充资料**：

| 优先级 | 资料类型 | 内容说明 | 用途 |
|--------|----------|----------|------|
| ⭐⭐⭐ | 产品 Demo / 测试环境链接 | 新功能可操作环境 | 验证 PRD 变更、录制操作 |
| ⭐⭐⭐ | 界面截图 / 设计稿 | 新功能界面 | 确认界面细节 |
| ⭐⭐ | 旧版 PRD / 需求记录 | 历史需求背景 | 理解变更原因 |
| ⭐⭐ | 测试用例 / 验收标准 | 功能验收内容 | 确认边界条件 |
| ⭐ | 用户反馈 | 关于新功能的用户声音 | 补充注意事项 |

### 执行流程

#### Step 1：解析 PRD 内容
从 PRD 中提取关键信息：
- **新增功能**：功能名称、功能描述、涉及模块、用户操作流程
- **功能修改**：修改前后的对比、影响范围、用户可见变化
- **功能下线**：下线功能名称、替代方案（如有）
- **配置/权限变更**：新增配置项、权限调整、入口变化

输出 PRD 变更摘要表：

| 变更类型 | 功能名称 | 变更描述 | 影响手册章节 |
|----------|----------|----------|--------------|
| 新增 | 数据导出 | 支持导出 Excel/CSV | 第 3.5 章 |
| 修改 | 用户登录 | 新增短信验证码登录方式 | 第 2.1 章 |
| 下线 | 批量导入 | 已下线，建议使用 API 接口 | 第 3.3 章 |

#### Step 2：匹配手册章节
根据 PRD 变更内容，定位需更新的手册章节：
- 按功能名称匹配章节标题
- 按模块名称匹配章节层级
- 按操作入口匹配功能说明章节

输出章节匹配结果：

| PRD 变更项 | 匹配的手册章节 | 匹配置信度 |
|------------|----------------|------------|
| 新增「数据导出」功能 | 第 3.5 章 数据管理 | 高（新增章节） |
| 登录方式新增短信验证码 | 第 2.1.2 章 登录方式 | 高（修改现有章节） |
| 批量导入功能下线 | 第 3.3 章 批量操作 | 中（需删除或标注下线） |

#### Step 3：生成 Diff 草稿
为每个需更新的章节生成 diff 草稿：

**新增章节格式**：
```markdown
【新增】第 3.5 章 数据导出

## 3.5 数据导出

### 功能简介
支持将系统数据导出为 Excel 或 CSV 格式，便于本地分析和归档。

### 功能入口
- 路径：数据管理 → 数据导出
- 权限：需「数据导出」权限

### 操作步骤
1. 进入「数据管理」模块
2. 点击「数据导出」按钮
3. 选择导出数据范围（全部/筛选结果）
4. 选择导出格式（Excel/CSV）
5. 点击「确认导出」，等待文件生成
6. 下载导出文件

### 注意事项
- 单次导出上限 10 万条
- 导出文件保留 7 天
```

**修改章节格式**：
```markdown
【修改】第 2.1.2 章 登录方式

--- 原内容 ---
系统支持账号密码登录和第三方登录（微信、企业微信）。

+++ 新内容 +++
系统支持以下登录方式：
- 账号密码登录
- 短信验证码登录（新增）
- 第三方登录（微信、企业微信）

### 短信验证码登录
1. 在登录页点击「短信验证码登录」
2. 输入手机号
3. 点击「获取验证码」
4. 输入收到的验证码
5. 点击「登录」
```

**删除/下线章节格式**：
```markdown
【下线】第 3.3 章 批量导入

--- 原内容 ---
（保留原文）

+++ 处理建议 +++
该功能已下线，建议：
1. 本章标注「已下线」状态
2. 添加替代方案说明：「请使用 API 接口进行批量数据导入，详见第 6 章接口文档」
3. 或直接删除本章（需确认）
```

#### Step 4：用户确认
将 diff 草稿呈现给用户，逐项确认：
- ✅ 采纳：按草稿更新
- ✏️ 修改：用户编辑后采纳
- ❌ 跳过：不更新该章节
- 🔄 调整：重新生成该章节的 diff

确认后记录最终更新决策。

#### Step 5：写回文档
根据确认结果更新手册：
- 按版本管理规则处理版本号和修订记录
- 应用所有确认的更新
- 执行自检清单
- 输出新版本文档

## 模式二：功能 Diff 诊断

### 适用场景
- 手册可能过时，但不确定具体差异
- 产品经历多次迭代，手册未同步
- 需要评估手册更新工作量

### 输入资料

**必填**：

| 资料类型 | 说明 |
|----------|------|
| 现有手册 | 已有说明书文件路径 |
| 产品现状信息 | 以下至少一种 |

**产品现状信息来源**：

| 优先级 | 资料类型 | 内容说明 | 适用场景 |
|--------|----------|----------|----------|
| ⭐⭐⭐ | 产品 Demo / 测试环境链接 | 可操作的产品环境 | 最直接，可验证所有功能 |
| ⭐⭐⭐ | 界面截图 / 操作录屏 | 当前产品界面 | 无法访问环境时使用 |
| ⭐⭐ | 功能列表 / 菜单结构 | 产品模块和功能点清单 | 快速对比结构差异 |
| ⭐⭐ | 近期版本更新日志 | 版本变更记录 | 了解迭代历史 |
| ⭐ | 用户口述功能 | 用户描述产品功能 | 无法获取其他资料时 |

**选填**：
- 产品版本信息
- 近期更新日志

### 执行流程

#### Step 1：解析手册内容
提取手册现有功能清单：
- 章节结构（目录）
- 已记录的功能列表
- 已记录的操作流程
- 版本信息和修订记录

输出手册功能清单：

| 章节 | 功能名称 | 记录的操作流程 |
|------|----------|----------------|
| 2.1 | 用户登录 | 账号密码登录、第三方登录 |
| 3.1 | 数据查询 | 筛选、排序、分页 |
| 3.2 | 数据录入 | 表单填写、批量导入 |
| ... | ... | ... |

#### Step 2：获取产品现状
根据用户提供的产品信息，整理当前产品功能：
- 梳理产品模块结构
- 列出各模块下的功能点
- 识别关键操作流程
- 记录界面元素和交互方式

输出产品功能现状：

| 模块 | 功能名称 | 当前状态 | 关键操作 |
|------|----------|----------|----------|
| 账户 | 用户登录 | 正常 | 账号密码/短信验证码/第三方登录 |
| 数据 | 数据查询 | 正常 | 筛选/排序/分页/导出 |
| 数据 | 数据录入 | 正常 | 表单填写 |
| 数据 | 批量导入 | 已下线 | - |
| ... | ... | ... | ... |

#### Step 3：对比诊断
对比手册与产品现状，识别差异：

**差异类型定义**：

| 差异类型 | 定义 | 示例 |
|----------|------|------|
| 过时 | 手册记录的功能在产品中已不存在或已变更 | 手册写「批量导入」，产品已下线 |
| 缺失 | 产品有此功能，手册未记录 | 产品有「数据导出」，手册无此章节 |
| 不一致 | 手册描述与产品实际不符 | 手册写「仅支持 Excel」，产品已支持 CSV |
| 建议优化 | 内容正确但可改进 | 操作步骤过于简略、缺少截图、术语不统一 |

#### Step 4：输出诊断报告

```markdown
# 产品说明书诊断报告

## 概览
- 手册版本：V1.2
- 诊断日期：2024-03-20
- 差异项总数：8 项

## 差异明细

### 🔴 过时内容（2 项）
| 章节 | 问题描述 | 建议处理 |
|------|----------|----------|
| 3.2 批量导入 | 功能已下线 | 标注下线或删除章节 |
| 4.1 支付方式 | 已下线「货到付款」 | 删除该支付方式描述 |

### 🟡 缺失内容（3 项）
| 功能名称 | 所属模块 | 建议章节位置 |
|----------|----------|--------------|
| 数据导出 | 数据管理 | 新增 3.5 章 |
| 短信验证码登录 | 账户管理 | 补充到 2.1.2 章 |
| 操作日志 | 系统设置 | 新增 5.3 章 |

### 🟠 不一致内容（2 项）
| 章节 | 手册描述 | 产品实际 | 建议修改 |
|------|----------|----------|----------|
| 3.1 数据查询 | 仅支持 Excel 导出 | 支持 Excel/CSV 导出 | 更新格式描述 |
| 2.1 用户登录 | 登录入口在顶部导航 | 登录入口已移至右上角头像菜单 | 更新入口位置 |

### 🔵 建议优化（1 项）
| 章节 | 当前问题 | 优化建议 |
|------|----------|----------|
| 3.3 数据录入 | 操作步骤仅有文字描述 | 建议添加表单字段说明和截图 |

## 后续建议
1. 优先处理过时内容，避免误导用户
2. 补充缺失功能说明，保证功能覆盖完整
3. 修正不一致内容，确保描述准确
4. 根据优先级逐步优化内容质量
```

#### Step 5：后续处理
诊断报告输出后，用户可选择：
- **一键修复**：自动生成所有差异项的 diff 草稿，进入模式一的确认流程
- **选择性修复**：指定部分差异项生成 diff 草稿
- **仅输出报告**：仅保留诊断报告，不执行更新

## 模式三：用户指令更新

### 适用场景
- 用户明确知道要更新什么内容
- 更新内容具体、可直接执行
- 无需 PRD 分析或差异诊断

### 输入资料

**必填**：

| 资料类型 | 说明 |
|----------|------|
| 现有手册 | 已有说明书文件路径 |
| 用户更新指令 | 明确的更新要求 |

**常见补充资料**：

| 优先级 | 资料类型 | 内容说明 | 用途 |
|--------|----------|----------|------|
| ⭐⭐⭐ | 产品 Demo / 测试环境链接 | 相关功能可操作环境 | 验证更新内容、确认操作步骤 |
| ⭐⭐ | 界面截图 / 设计稿 | 相关功能界面 | 确认界面细节 |
| ⭐⭐ | 功能演示 / 操作录屏 | 相关功能操作演示 | 验证操作流程 |
| ⭐ | 参考文档 | 其他相关文档 | 保持一致性 |

### 指令类型

#### 类型 A：内容新增
用户指令示例：
- "在数据管理章节新增「数据导出」功能说明"
- "补充「短信验证码登录」的操作步骤"

处理方式：
1. 定位插入位置（指定章节或自动匹配）
2. 按创建文档的内容规范生成新内容
3. 呈现草稿供用户确认
4. 确认后写入文档

#### 类型 B：内容修改
用户指令示例：
- "把「批量导入」改为支持 10 万条数据"
- "更新登录入口的位置描述"

处理方式：
1. 定位需修改的章节
2. 按用户指令修改内容
3. 呈现修改前后对比供确认
4. 确认后写入文档

#### 类型 C：内容删除
用户指令示例：
- "删除「货到付款」的支付方式描述"
- "把「批量导入」章节标注为已下线"

处理方式：
1. 定位需删除的内容
2. 呈现删除预览（标注删除线）
3. 确认后执行删除或标注下线

#### 类型 D：结构调整
用户指令示例：
- "把「数据查询」和「数据录入」合并成一个章节"
- "调整章节顺序，把系统设置放到最后"

处理方式：
1. 按用户指令调整章节结构
2. 呈现调整后的目录结构
3. 确认后执行调整

#### 类型 E：格式修正
用户指令示例：
- "统一所有章节的标题格式"
- "把所有「点击」改为「单击」"

处理方式：
1. 按用户指令批量修改格式
2. 呈现修改范围和示例
3. 确认后执行批量修改

### 执行流程

```
用户指令 → 解析指令类型 → 定位目标内容 → 生成预览 → 用户确认 → 写入文档
```

---

## 更新模式选择指南

当用户触发更新操作时，按以下规则选择模式：

1. **用户提供 PRD/MEP 链接或文本** → 模式一：PRD→手册更新
2. **用户要求诊断手册与产品的差异** → 模式二：功能 Diff 诊断
3. **用户给出明确的更新指令** → 模式三：用户指令更新
4. **用户未明确指定** → 询问用户选择模式

---

## 输出格式

根据用户需求，支持多种输出格式：

### 主要输出格式

| 格式 | 适用场景 | 说明 |
|------|----------|------|
| **Word (.docx)** | 正式交付、打印、存档 | 默认格式 |
| **Markdown (.md)** | 内部协作、版本管理 | 轻量级，易于 diff |
| **HTML** | 在线帮助中心更新 | 直接部署 |
| **PDF** | 正式交付 | 不可修改 |

### 中间输出

**Diff 草稿**（更新预览）：

所有模式的 diff 草稿统一使用以下格式，方便用户审阅：

```markdown
## 更新预览

### 章节：[章节编号] [章节名称]
**更新类型**：新增 / 修改 / 删除 / 调整

--- 原内容 ---
（原文档内容，新增时为空）

+++ 新内容 +++
（更新后内容，删除时为空）

### 变更说明
（简述本次变更的原因和影响）
```

**诊断报告**（模式二输出）：

```markdown
# 产品说明书诊断报告

## 概览
- 手册版本：V1.2
- 诊断日期：2024-03-20
- 差异项总数：8 项

## 差异明细

### 🔴 过时内容（2 项）
| 章节 | 问题描述 | 建议处理 |
|------|----------|----------|
| ... | ... | ... |

### 🟡 缺失内容（3 项）
| 功能名称 | 所属模块 | 建议章节位置 |
|----------|----------|--------------|
| ... | ... | ... |

### 🟠 不一致内容（2 项）
| 章节 | 手册描述 | 产品实际 | 建议修改 |
|------|----------|----------|----------|
| ... | ... | ... | ... |

### 🔵 建议优化（1 项）
| 章节 | 当前问题 | 优化建议 |
|------|----------|----------|
| ... | ... | ... |
```

### 确认交互

每项更新需用户确认后才写入文档，确认选项：
- ✅ 采纳
- ✏️ 修改后采纳
- ❌ 跳过
- 🔄 重新生成

---

## 质量检查

更新完成后执行以下检查：

### 内容完整性
- [ ] PRD 中的所有变更是否都已同步
- [ ] 诊断报告中的差异是否都已处理
- [ ] 用户指令是否都已执行

### 内容准确性
- [ ] 新增内容是否与产品实际一致
- [ ] 修改内容是否准确反映变更
- [ ] 删除内容是否已确认无误

### 格式一致性
- [ ] 新增章节格式是否与现有章节一致
- [ ] 术语使用是否统一
- [ ] 标题层级是否正确

### 版本管理
- [ ] 版本号是否已按要求升级
- [ ] 修订记录是否已更新
- [ ] 文件名是否符合规范
