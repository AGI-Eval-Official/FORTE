# 网站平台补充规则

> 本文件定义了网站/Web 应用类产品的功能说明书中，需要特别说明的平台特性和注意事项。
> 与通用模板 `product_manual_rules.md` 配合使用。

---

## 一、需要在说明书中额外覆盖的内容

### 1. 浏览器兼容性

在「产品环境要求」或独立的小节中说明：

- **支持的浏览器及最低版本**：
  | 浏览器 | 最低版本 | 推荐版本 | 备注 |
  |--------|----------|----------|------|
  | Chrome | 最近 2 个主版本 | 最新版 | 主力支持 |
  | Firefox | 最近 2 个主版本 | 最新版 | 主力支持 |
  | Safari | 最近 2 个主版本 | 最新版 | macOS / iOS |
  | Edge | 最近 2 个主版本 | 最新版 | 基于 Chromium |
  | IE | 不支持 / IE11+（视情况） | - | 注明是否兼容 |

- **不支持或不推荐使用的浏览器**及其原因
- **浏览器设置要求**：是否需要启用 Cookie / JavaScript / 弹窗等

### 2. 页面响应式适配说明

如果产品支持多设备访问（PC / 平板 / 手机），需说明：

- 不同屏幕尺寸下的布局差异
- 移动端浏览器中的特殊表现（与原生 APP 的区别）
- 横屏 / 屏幕模式的支持情况

### 3. 网络与性能

- **网络要求**：是否需要持续联网、支持的最低网速
- **离线能力**：是否支持离线使用、离线时可用的功能范围
- **加载时间预期**：首次加载 / 页面切换的正常等待时间
- **数据量限制**：单次上传/下载的文件大小限制

### 4. 安全相关（用户视角）

- **登录与会话**：
  - 会话超时时间（如：无操作 30 分钟后自动退出）
  - 是否支持"记住登录状态"
  - 同账号多设备/多标签同时登录的行为
- **数据安全**：
  - 数据传输加密说明（通俗描述即可，如"所有数据通过安全连接传输"）
  - 敏感信息的展示规则（如：银行卡号中间隐藏）

### 5. 搜索引擎优化（SEO）（仅面向公开产品时需要）

如果产品是面向公众的网站（非后台系统），需说明：
- 产品/内容的公开可见性
- 是否可被搜索引擎收录
- 如何分享页面链接

## 二、网站产品功能描述的特殊要点

### 表单交互类功能

网站表单相比其他平台有一些特殊场景需要说明：

| 场景 | 说明要点 |
|------|----------|
| 页面刷新/关闭 | 未提交的表单数据是否会保存？是否有"离开页面的提醒"？ |
| 浏览器后退 | 提交后按后退按钮的行为（避免重复提交的说明） |
| 多标签操作 | 同一功能在多个标签页打开时的数据同步/冲突处理 |
| 输入法兼容 | 中文输入法在搜索/筛选等实时响应场景的表现 |
| 快捷键支持 | 是否支持键盘快捷键（Enter 提交、Esc 取消等） |

### 数据导出/导入类功能

| 场景 | 说明要点 |
|------|----------|
| 导出格式 | 支持的文件格式（Excel/CSV/PDF/Word）及适用场景 |
| 数据量限制 | 单次导出的最大行数/文件大小 |
| 导入模板 | 是否提供标准模板、模板下载位置 |
| 导入错误处理 | 导入失败时的错误提示方式（行级错误定位） |
| 浏览器下载行为 | 文件下载到哪个目录、自动弹出还是静默下载 |

### 实时更新类功能

| 场景 | 说明要点 |
|------|----------|
| 数据刷新机制 | 手动刷新 vs 自动刷新（轮询间隔） |
| 多人协作 | 其他用户操作的实时可见性（是否需要手动刷新页面） |
| 消息通知 | 浏览器内的通知方式（角标/弹窗/声音/震动） |

## 三、不需要写入说明书的内容（网站特有）

以下内容属于技术实现细节，**不要**出现在面向用户的说明书中：

- ❌ 前端框架（React/Vue/Angular 等）
- ❌ CDN / 缓存策略
- ❌ 服务端架构（单体/微服务/Serverless）
- ❌ 数据库类型
- ❌ API 接口地址和参数细节（除非是开发者工具类产品）
- ❌ HTTP 状态码
- ❌ 浏览器 Developer Tools 的使用方法

## 四、截图与示例建议

网站产品说明书中，截图是最重要的辅助手段：

| 截图类型 | 使用场景 | 注意事项 |
|----------|----------|----------|
| 全页截图 | 展示整体页面布局 | 标注主要区域名称 |
| 局部截图 | 展示某个功能的具体操作位置 | 用红框/箭头高亮关键元素 |
| 操作步骤截图 | 分步教程中每一步配一张图 | 保持截图尺寸一致 |
| 对比截图 | 操作前 vs 操作后的效果变化 | 并排展示 |
| 空状态截图 | 无数据时的页面展示 | 帮助用户理解"正常空白"和"异常"的区别 |

**截图规范：**
- 分辨率建议 1280x720 或 1920x1080
- 隐藏敏感信息（真实用户数据、个人信息）
- 隐藏浏览器地址栏中的内部域名/IP（如有）
- 统一浏览器窗口装饰（建议使用 Chrome 无痕模式的简洁界面）
