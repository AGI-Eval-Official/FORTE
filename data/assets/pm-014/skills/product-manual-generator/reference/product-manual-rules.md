# 产品功能说明书规则

> 本文件定义了产品功能说明书的标准内容体系、章节结构和编写规范。
> 支持 Word、Markdown、HTML、PDF 等多格式输出，具体格式选择详见 `create_rules.md` 输出格式章节。
> 面向读者：客户、用户、合作伙伴（非内部研发人员）。

---

## 一、文档基础信息

| 字段 | 说明 |
|------|------|
| 文档名称 | `[产品名称] 功能说明书` |
| 版本号 | 对应产品版本号，如 `V2.1.0` |
| 作者 | 文档撰写人 |
| 日期 | 创建/更新日期 |
| 适用产品版本 | 说明本文档对应的产品版本 |

### 变更记录（必须包含）

| 版本 | 日期 | 修改人 | 修改内容 | 备注 |
|------|------|--------|----------|------|
| V2.1.0 | YYYY-MM-DD | 姓名 | 初始版本 / 新增 XX 功能说明 | 对应产品 V2.1.0 |

---

## 二、标准章节结构（按顺序）

> 以下章节标注了**选填**标识：
> - **【必填】**：几乎所有产品都需要，无特殊情况必须包含
> - **【推荐】**：大部分产品需要，根据实际情况决定是否包含
> - **【选填】**：仅特定类型产品需要，按需决定
> - **（条件）**：满足某条件时必填，否则可跳过

---

### 第 1 章：产品简介 【必填】

#### 1.1 产品概述 【必填】
- 用 2-3 段话介绍产品是什么
- 核心定位和价值主张
- 一句话概括：本产品是面向 [谁] 的 [什么产品]，用于解决 [什么问题]

#### 1.2 产品亮点 / 核心优势 【推荐】
- 列出 3-6 个核心卖点
- 每个卖点用一句话 + 简要说明
- **注意**：从用户获益角度描述，不要堆技术术语

#### 1.3 适用范围 【必填】
- 目标用户群体
- 典型应用场景（举例 3-5 个）
- 不适用的场景（如有）

#### 1.4 版本信息 【条件 — 有版本迭代的产品必填，一次性交付/静态产品可跳过】
- 当前产品版本号
- 本次版本新增/变更的功能概览

#### 1.5 产品环境要求 【条件 — 需要客户端安装或特定运行环境时必填】
- 支持的操作系统及最低版本
- 硬件要求（CPU/内存/存储空间）
- 网络环境要求（是否需要联网、带宽建议）
- 依赖的其他软件/服务（如需安装浏览器插件、依赖某第三方账号）

---

### 第 2 章：功能总览 【必填】

#### 2.1 功能架构图 【推荐】
- 产品功能模块的层级结构（模块 → 子模块 → 功能点）
- 用表格或树形图展示

#### 2.2 功能清单 【必填】
| 模块 | 功能名称 | 功能简介 | 适用对象 |
|------|----------|----------|----------|
| 用户管理 | 用户注册 | 支持手机号和邮箱注册 | 所有用户 |

#### 2.3 功能与角色权限对照表 【条件 — 多角色/多用户类型产品必填，单用户产品可跳过】
| 角色 | 功能A | 功能B | 功能C | ... |
|------|-------|-------|-------|-----|
| 管理员 | ✅ | ✅ | ✅ | |
| 普通用户 | ✅ | ❌ | ✅ | |
| 只读用户 | 👁 | ❌ | 👁 | |

#### 2.4 数据 / 内容结构说明 【条件 — 涉及数据管理/内容展示的产品必填】
- 产品中的核心数据实体及其关系（如：订单包含商品、用户拥有项目等）
- 用简单的 ER 图描述或文字说明即可
- 帮助用户理解产品中的数据组织方式

---

### 第 3 章：详细功能说明（核心章节）【必填】

**每个功能按以下结构描述：**

#### 3.X [功能名称]

##### 功能简介 【必填】
- 1-2 句话说明该功能的用途
- 用户使用此功能能获得什么价值

##### 功能入口 【必填】
- 从哪里可以找到并进入此功能
- 路径说明：如 `首页 → 设置 → 账户安全`

##### 操作步骤 【必填】
- 按 ①②③④ 分步描述操作流程
- 每一步：操作动作 + 界面反馈 + 预期结果
- **关键操作的截图位置预留**（标注"此处插入截图"）

##### 功能详解 【推荐】
- **核心能力列表**：该功能下包含的所有子能力或选项
- **字段/参数说明**（如涉及表单输入、配置项）：
  | 字段/选项 | 说明 | 是否必填 | 示例/可选值 |
  |-----------|------|----------|-------------|
  |           |      |          |             |
- **规则与约束**：
  - 使用限制（如：单次最多上传 XX 个文件）
  - 格式要求（如：支持 jpg/png，最大 5MB）
  - 时间/次数限制（如：每日导出不超过 5 次）

##### 注意事项 【推荐】
- 用户容易踩坑的地方
- 操作不可逆的警告提示
- 与其他功能的关联说明

##### 常见问题（FAQ）【推荐】
- 针对该功能 2-5 个高频问题
- Q&A 形式，简洁回答

---

### 第 4 章：平台特性说明 【条件 — 多端产品必填，单端产品可合并到各功能章节中】

- **注意**：此部分内容由对应平台补充规则文件（web_rules / app_rules / miniprogram_rules）提供具体要求
- 不同平台（网站/APP/小程序）在交互方式、系统权限、使用限制上的差异说明
- 如果产品只在单一平台运行，将平台相关说明直接写入第 3 章对应功能中即可，无需单独成章

---

### 第 5 章：常见问题汇总（FAQ）【推荐】

#### 5.1 通用问题 【条件 — 有则写，无则跳过】
根据产品实际情况选择相关的类别：
- 账号相关（注册、登录、密码找回、账号安全）— **有账号系统的产品**
- 费用/套餐相关（计费方式、升级降级、退款）— **付费产品**
- 数据相关（数据导出、备份、隐私保护）— **涉及用户数据的产品**
- 安装/部署相关（下载安装、环境配置、升级更新）— **需要客户端安装的产品**
- 接入/集成相关（API 对接、嵌入方式、数据同步）— **需要与其他系统集成的产品**

#### 5.2 功能使用问题 【推荐】
- 按功能模块分类的常见问题
- 每个问题：问题描述 + 解决步骤

#### 5.3 故障排查 【推荐】
- 常见错误提示及含义
- 用户可自行处理的异常情况
- 需联系技术支持的判断标准

---

### 第 6 章：附录 【选填 — 按需选取】

以下附录项根据产品特点选择使用，不需要全部包含：

- **附录 A：快捷键 / 手势操作一览表** — 有快捷键或手势操作的产品（桌面端/APP）
- **附录 B：错误码 / 提示信息对照表** — 有明确错误码体系的产品
- **附录 C：名词解释** — 包含大量专业术语或行业特有概念的产品
- **附录 D：联系我们 / 技术支持渠道** — 提供售后/技术支持的产品
- **附录 E：更新日志** — 有持续迭代的产品
- **附录 F：功能对比表（版本/套餐对比）** — 多版本或多套餐的产品
- **附录 G：接口 / 对接说明** — 需要与其他系统对接的技术类产品
- **附录 H：安装 / 部署指南** — 需要本地安装或私有化部署的产品

## 三、不同产品类型的章节裁剪指南

> 编写前先确定产品属于哪种类型，然后参考下表裁剪章节。

| 产品类型 | 特征 | 必须保留 | 可以跳过 | 特别注意 |
|----------|------|----------|----------|----------|
| **SaaS / 在线服务平台** | 有账号、多角色、持续迭代 | 全部章节 | 附录 G/H | 重点写权限对照表、套餐对比、数据相关FAQ |
| **APP（工具/社交/娱乐）** | 移动端为主、有账号系统 | 1.1~1.3, 全部第2~3章, 第5章 | 1.4(可选), 2.3(看情况), 2.4 | 重点写操作步骤+截图、离线使用说明 |
| **小程序** | 轻量级、依托微信生态 | 1.1~1.3, 2.2, 全部第3章 | 1.5, 2.3, 2.4, 第4章(合并到功能中) | 重点写微信授权、分享、扫码等小程序特有能力 |
| **后台管理系统 (B端)** | 多角色、重操作效率 | 1.1~1.3, 2.2~2.3, 全部第3章 | 1.2(弱化), 1.4, 第5章(精简) | 重点写权限差异、批量操作、数据导入导出 |
| **内部工具** | 企业内部使用、无公开售卖 | 1.1, 1.3, 2.2, 全部第3章 | 1.2, 1.4, 2.3, 第5章(大幅精简) | 重点写操作步骤、字段说明，弱化营销性内容 |
| **硬件配套产品** | 配合硬件设备使用 | 1.1, 1.3, 1.5(重点), 2.2, 全部第3章 | 1.2, 2.3 | 重点写设备连接/配对、硬件状态指示、故障排查 |
| **纯内容/展示型产品** | 以浏览查看为主 | 1.1~1.3, 2.2, 第3章(精简) | 2.3, 2.4, 第5章 | 重点写内容分类、导航方式、搜索筛选 |
| **API / 开发者工具** | 面向开发者 | 1.1, 1.3, 1.5, 2.2, 第3章, 附录G | 1.2, 2.3 | 重点写参数说明、调用示例、返回结果、错误码 |

## 四、编写注意事项

### ✅ 必须做到

1. **用户视角**：始终从"用户怎么用"的角度写，而不是"系统怎么实现"
   - ✅ "您可以在设置页面修改头像"
   - ❌ "系统支持头像字段的更新操作"

2. **语言通俗易懂**：
   - 避免专业术语，必须用时在首次出现时注释
   - 用主动语态，直接告诉用户能做什么
   - 句子简短，一段话不超过 3 行

3. **图文结合**：复杂操作流程必须配图（截图/示意图），文档中预留截图位置

4. **示例驱动**：抽象概念给具体例子
   - ✅ "如输入 '2024-01-01' 或点击日历选择"
   - ❌ "输入符合日期格式的字符串"

5. **保持一致性**：
   - 同一功能在全文档中叫同一个名字
   - 操作动词统一（全部用"点击"或全部用"选择"，不要混用）
   - 格式风格统一

6. **明确边界**：写清楚什么能做、什么不能做、有什么限制

7. **按产品类型裁剪**：不要机械地套用所有章节，参考第三节的裁剪指南灵活调整

### ❌ 必须避免

1. **避免内部术语**：不要出现"接口""数据库""后端""缓存""部署"等研发词汇（除非是面向开发者的产品）
2. **避免实现细节**：不要写"基于 xx 算法""采用 xx 架构"，用户不关心
3. **避免主观营销用语**：这是说明书不是广告文案
   - ❌ "极致体验""革命性创新""业界领先"
   - ✅ "支持批量导入，一次最多处理 1000 条"
4. **避免信息过载**：每个功能只写用户需要知道的，不是所有你知道的
5. **避免只有文字没有层次**：善用标题、列表、表格、加粗来提升可扫读性
6. **避免照搬不适用的章节**：没有账号就不要写账号相关的 FAQ，没有多角色就不做权限表

### 📐 格式规范

1. **标题层级**：章（H1）→ 节（H2）→ 小节（H3），最多到 H4
2. **操作步骤编号**：使用 ①②③④ 或 1) 2) 3)，不用 (1)(2)(3)
3. **界面元素引用**：界面上的文字用「」引用，如「确认」「取消」
4. **按钮/菜单名**：用加粗标示，如**点击【提交】按钮**
5. **路径表示**：用 `>` 或 `→` 表示导航路径，如 `首页 > 个人中心 > 设置`
6. **提示/警告分级**：
   - 💡 **提示**：有用的小技巧
   - ⚠️ **注意**：可能导致问题的操作
   - 🔴 **重要**：不可逆操作或关键约束
7. **Word 排版**：
   - 使用目录（TOC）
   - 页眉显示产品名称和文档版本
   - 页脚显示页码
   - 正文使用宋体/微软雅黑 10.5pt，英文使用 Arial
   - 一级标题 16pt 加粗，二级标题 14pt 加粗，三级标题 12pt 加粗
   - 行间距 1.5 倍

### 🔍 自检清单（输出前逐条确认）

**基础检查：**
- [ ] 文档基础信息完整（名称、版本、作者、日期、适用产品版本）
- [ ] 变更记录已填写
- [ ] 已根据产品类型完成章节裁剪（参考第三节裁剪指南）
- [ ] 不适用的章节已合理跳过，未生搬硬套

**内容质量：**
- [ ] 产品简介能让新用户在 3 分钟内了解产品是什么
- [ ] 功能清单无遗漏，覆盖产品的全部可见功能
- [ ] 每个功能都包含：简介、入口、操作步骤（至少这三项）
- [ ] 操作步骤分步清晰，新手能跟着做下来
- [ ] 字段/参数/限制有明确说明（格式、长度、数量等约束）

**用户体验：**
- [ ] 语言是目标读者友好的（面向用户则通俗，面向开发者则专业）
- [ ] 无营销性虚词，内容实在可验证
- [ ] 注意事项覆盖了易错点和不可逆操作
- [ ] FAQ 覆盖了高频问题（且都是该产品真正会遇到的问题）

**格式规范：**
- [ ] 全文术语一致、格式统一、排版整洁
- [ ] 选填标识已清理（最终输出文档不应出现"【必填】【选填】"等标记）
