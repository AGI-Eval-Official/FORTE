# 版本管理规则

本文档定义产品功能说明书的版本管理规范，包括版本号规则、升级判断、修订记录和历史版本保留策略。

---

## 版本管理开关

### 默认行为
版本管理**默认开启**，除非用户明确关闭。

**开启条件**（满足任一）：
- 用户未提及版本管理相关要求
- 用户说"要版本管理"/"加上版本号"

**关闭条件**（用户明确说）：
- "不要版本管理"
- "不用管版本"
- "只更新内容不改版本号"
- "不需要版本记录"

### 开关影响

| 版本管理状态 | 创建文档 | 更新文档 |
|--------------|----------|----------|
| **开启** | 初始版本 V1.0，包含修订记录表 | 自动升级版本号，新增修订记录 |
| **关闭** | 文件名用日期，无版本号和修订记录 | 保持原版本不变，不修改修订记录 |

---

## 版本号规范

### 版本号格式
采用三段式版本号：`V主版本.次版本.修订号`

示例：`V1.2.3`

- **V**：版本标识前缀（大写）
- **主版本**（第 1 段）：产品大版本，重大架构变更时递增
- **次版本**（第 2 段）：功能模块版本，新增功能时递增
- **修订号**（第 3 段）：内容修订版本，小幅修正时递增

### 版本号位置
版本号需出现在以下位置：
- **文档封面**：产品名称下方
- **页眉**：产品名称 + V 版本号
- **修订记录表**：第一条记录

## 版本升级判断

### 升级规则

| 更新类型 | 版本升级方式 | 示例 |
|----------|--------------|------|
| 产品大改版、架构重大变更 | 主版本递增，次版本和修订号归零 | V1.2.3 → V2.0.0 |
| 新增功能模块、较大功能更新 | 次版本递增，修订号归零 | V1.2.3 → V1.3.0 |
| 修改功能描述、内容勘误、小幅修正 | 修订号递增 | V1.2.3 → V1.2.4 |
| 仅排版/错别字修正 | 可不升级版本号 | V1.2.3 保持不变 |

### 升级判断流程

```
收到更新请求
    ↓
判断更新类型
    ↓
┌─────────────────────────────────────────┐
│ 产品大改版？                              │
│   ├─ 是 → 主版本递增（V1.x → V2.0.0）     │
│   └─ 否 ↓                                │
│ 新增功能模块？                            │
│   ├─ 是 → 次版本递增（V1.1.x → V1.2.0）   │
│   └─ 否 ↓                                │
│ 修改功能描述或内容勘误？                   │
│   ├─ 是 → 修订号递增（V1.1.1 → V1.1.2）   │
│   └─ 否 → 不升级版本号                    │
└─────────────────────────────────────────┘
    ↓
更新版本号和修订记录
```

### 具体场景示例

**主版本递增**：
- 产品从 1.0 重构到 2.0
- 核心业务逻辑重大变更
- 用户界面全面改版
- 技术架构迁移（如单页应用重构）

**次版本递增**：
- 新增「数据导出」模块
- 新增「短信验证码登录」功能
- 新增「操作日志」查询功能
- 现有模块新增重要功能点

**修订号递增**：
- 修正「登录入口位置」描述
- 更新「导出上限」从 5 万改为 10 万
- 补充遗漏的操作步骤
- 修正错别字和格式问题

**不升级版本号**：
- 仅调整排版格式
- 修正标点符号
- 调整段落间距

## 修订记录

### 修订记录表格式

修订记录表位于文档目录之后、正文之前，格式如下：

| 版本号 | 修订日期 | 修订内容 | 编写人 |
|--------|----------|----------|--------|
| V1.0.0 | 2024-01-15 | 初始版本创建 | [待补充] |
| V1.1.0 | 2024-03-20 | 新增「数据导出」功能说明 | [待补充] |
| V1.1.1 | 2024-03-25 | 修正「登录入口」位置描述 | [待补充] |

### 字段说明

- **版本号**：本次修订对应的版本号
- **修订日期**：格式为 YYYY-MM-DD
- **修订内容**：简述本次修订的主要内容（建议不超过 50 字）
- **编写人**：文档编写者姓名，若未知使用 `[待补充]`

### 修订内容编写规范

**好的修订内容描述**：
- ✅ "新增「数据导出」功能说明"
- ✅ "修正「登录入口」位置描述，更新至右上角头像菜单"
- ✅ "补充「批量导入」注意事项"

**不好的修订内容描述**：
- ❌ "更新了一些内容"（过于笼统）
- ❌ "修复 bug"（未说明具体内容）
- ❌ "第 3.2 章节修改了操作步骤，从原来的 5 步改为 6 步，新增了确认环节"（过于冗长）

## 文件命名规则

### 命名格式

| 场景 | 命名格式 | 示例 |
|------|----------|------|
| 创建文档（版本管理开启） | `[产品名称]_功能说明书_V[版本号].docx` | `订单管理系统_功能说明书_V1.0.0.docx` |
| 更新文档（版本管理开启） | `[产品名称]_功能说明书_V[新版本号].docx` | `订单管理系统_功能说明书_V1.1.0.docx` |
| 版本管理关闭 | `[产品名称]_功能说明书_YYYYMMDD.docx` | `订单管理系统_功能说明书_20240320.docx` |

### 命名注意事项

- 产品名称中不包含空格和特殊字符
- 版本号保持三段式完整格式（V1.0.0 而非 V1）
- 日期格式为 YYYYMMDD（年月日，无分隔符）

## 历史版本保留

### 默认策略

**版本管理开启时**：
- 默认输出新版本文件，原文档保持不变
- 不自动覆盖，除非用户明确要求

**版本管理关闭时**：
- 询问用户是否覆盖原文件
- 用户确认后执行覆盖

### 版本覆盖确认

当用户要求覆盖原文件时，需确认：

```
⚠️ 确认覆盖

即将覆盖原文件：[产品名称]_功能说明书_V1.0.0.docx

原文件内容将无法恢复，是否继续？

[确认覆盖] [取消，输出新文件]
```

### 版本存档建议

建议用户建立版本存档目录：

```
产品说明书/
├── 存档/
│   ├── V1.0.0/
│   │   └── 订单管理系统_功能说明书_V1.0.0.docx
│   └── V1.1.0/
│       └── 订单管理系统_功能说明书_V1.1.0.docx
└── 最新版本/
    └── 订单管理系统_功能说明书_V1.2.0.docx
```

## 单独执行版本管理操作

除创建/更新时的版本管理外，支持单独执行版本管理操作：

### 仅升级版本号

用户指令示例：
- "把版本号升级到 V1.2"
- "更新一下版本号"

执行步骤：
1. 读取当前文档版本号
2. 按规则升级版本号
3. 更新页眉、封面版本号
4. 修订记录新增一条"版本号升级"记录
5. 输出新文件

### 仅输出变更记录

用户指令示例：
- "输出这个文档的修订记录"
- "给我看一下版本历史"

执行步骤：
1. 读取文档修订记录表
2. 格式化输出为可读文本或表格

### 版本号修正

用户指令示例：
- "把版本号修正为 V2.0.0"
- "版本号写错了，改成 V1.5.0"

执行步骤：
1. 按用户指定版本号更新
2. 更新页眉、封面版本号
3. 修订记录新增一条"版本号修正"记录

## 质量检查清单

版本管理相关检查项：

- [ ] 版本号格式是否符合三段式规范
- [ ] 版本号是否出现在封面、页眉、修订记录三处
- [ ] 版本升级判断是否正确
- [ ] 修订记录是否已新增本次变更记录
- [ ] 修订内容描述是否清晰简洁
- [ ] 文件命名是否符合规范
- [ ] 历史版本是否按要求保留/覆盖
