# 小程序平台补充规则

> 本文件定义了微信小程序类产品的功能说明书中，需要特别说明的平台特性和注意事项。
> 与通用模板 `product_manual_rules.md` 配合使用。

---

## 一、需要在说明书中额外覆盖的内容

### 1. 访问与入口

| 项目 | 说明要点 |
|------|----------|
| 打开方式 | 微信内搜索小程序名称 / 扫描小程序码 / 分享卡片 / 公众号菜单 / 对话嵌入 / 最近使用列表 |
| 添加到桌面 | "添加到我的小程序"后可在微信顶栏快速访问 |
| 小程序码 | 小程序码的位置和用途（如：打印张贴供线下扫码） |
| 关联公众号 | 是否有关联的公众号，两者之间的关系和跳转方式 |

### 2. 微信授权相关（核心！小程序必须详细说明）

小程序重度依赖微信生态，授权是用户最关心也最容易困惑的部分：

#### 2.1 微信登录

| 项目 | 说明要点 |
|------|----------|
| 登录流程 | 点击授权按钮 → 微信弹出授权确认 → 授权成功后自动登录 |
| 授权信息 | 登录时获取的用户信息（昵称/头像/地区等） |
| 拒绝授权 | 用户拒绝微信授权后的处理方式和替代方案 |
| 切换账号 | 微信切换登录账号后，小程序内的账号跟随情况 |
| 解绑/更换微信号 | 是否支持更换绑定的微信号 |

#### 2.2 用户信息授权

| 授权项 | 用途 | 是否必须 | 拒绝后的影响 |
|--------|------|----------|--------------|
| 昵称与头像 | 建立用户身份标识 | 必须 | 无法使用需登录的功能 |
| 手机号 | 账号绑定 / 身份验证 | 视功能而定 | 无法使用需手机号的功能 |
| 地理位置 | 获取周边服务 / 打卡 | 视功能而定 | 无法使用基于位置的功能 |
| 通讯录 | 邀请好友 / 关联联系人 | 可选 | 无法快捷邀请好友 |

**特别注意：**
- 微信小程序的授权是**分级弹出**的（不是一次性全部申请），每次用到某项能力时才会触发对应的授权弹窗
- 需要在说明书中让用户理解这一点，避免用户困惑"为什么之前授权过现在又要授权"
- 部分授权（如手机号）在 2022 年后微信规则变更，需要用户主动触发（点击按钮），不能自动弹出

### 3. 分享能力

| 能力 | 说明要点 |
|------|----------|
| 分享给好友 | 分享卡片展示的内容（标题/图片/描述）、能否自定义分享文案 |
| 分享到朋友圈 | 是否支持（大部分小程序不支持直接分享到朋友圈，只能生成海报图片） |
| 分享卡片打开后的落地页 | 好友从分享卡片进入时看到的页面 |
| 分享追踪 | 分享带来的收益/积分（如有裂变机制） |

### 4. 支付能力

| 项目 | 说明要点 |
|------|----------|
| 支付方式 | 微信支付（小程序内唯一支付方式） |
| 支付流程 | 在小程序内唤起微信支付 → 确认金额 → 输入密码/指纹 → 支付完成 |
| 发票 | 电子发票的获取方式（小程序内查看 / 微信支付服务通知 / 邮件发送） |
| 退款 | 退款流程和到账时间 |

### 5. 订阅消息

| 项目 | 说明要点 |
|------|----------|
| 消息类型 | 服务通知（订单状态/预约提醒/到账通知等） |
| 订阅方式 | 用户主动点击允许 → 可接收一条或多条消息 |
| 一次订阅 vs 长期订阅 | 说明订阅的有效期（一般单次订阅只能发一次） |
| 消息接收位置 | 「服务通知」会话中接收 |
| 取消订阅 | 如何管理订阅偏好 |

### 6. 扫码能力

| 场景 | 说明要点 |
|------|----------|
| 扫码入口 | 小程序内扫码按钮 / 微信右上角 "+" 扫一扫（需配置） |
| 支持的码制 | 二维码 / 条形码 / 小程序码 |
| 扫码后的行为 | 识别后跳转到对应页面或执行对应操作 |
| 相册选图识别 | 是否支持从相册选取二维码图片识别 |

## 二、小程序特有的限制与边界（必须在说明书中说明）

这些是用户在使用过程中一定会遇到的问题，提前说明可以减少大量客服咨询：

| 限制类别 | 具体限制 | 用户侧表现 | 建议说明方式 |
|----------|----------|------------|--------------|
| **包体积限制** | 主包 ≤ 2MB，整个小程序（含分包）≤ 20MB | 一般不影响用户体验 | 通常无需说明，除非涉及大量媒体资源 |
| **存储限制** | 单个小程序本地存储 ≤ 10MB | 缓存过多时可能出现异常 | 在"存储与缓存"中说明清理方式 |
| **并发网络请求** | 最多 10 个并发请求 | 一般不影响用户体验 | 通常无需说明 |
| **页面栈限制** | 最多 10 层页面栈 | 深层跳转后无法继续跳转 | 如产品层级深，需说明 |
| **播放限制** | 同时只能播放一个音频/视频 | 播放新内容时自动停止之前的 | 在多媒体功能中说明 |
| **后台运行限制** | 切到前台后只有约 3 秒后台时间 | 切出微信后操作可能中断 | 在相关功能中提示"请勿切出" |
| **分享限制** | 不能直接分享到朋友圈 | 用户尝试分享到朋友圈时不可用 | 明确告知替代方案（生成海报） |
| **文件下载** | 不能直接下载文件到手机本地 | 需通过"用其他应用打开"中转 | 在下载类功能中说明操作路径 |
| **打开外部链接** | 有限制（需配置业务域名白名单） | 部分链接可能无法打开 | 如有外部链接功能，需说明限制 |

## 三、小程序与 H5 / APP 的区别说明

如果用户可能混淆小程序和其他形态，建议增加对比说明：

| 维度 | 小程序 | H5 网页 | 原生 APP |
|------|--------|---------|----------|
| 安装 | 无需安装，微信内即开即用 | 无需安装，浏览器打开 | 需下载安装 |
| 存储占用 | 几乎不占空间（<10MB 缓存） | 浏览器缓存 | 几十 MB 到几百 MB |
| 启动速度 | 快（秒开） | 取决于网速 | 最快（本地运行） |
| 推送通知 | 通过订阅消息（有限制） | 不支持 | 完整推送能力 |
| 离线使用 | 有限支持（需特殊开发） | 一般不支持 | 可完整离线 |
| 分享 | 微信生态内流畅 | 可分享任意链接 | 通过 DeepLink / 分享 SDK |
| 支付 | 仅微信支付 | 调起微信/支付宝 H5 支付 | 微信/支付宝/Apple Pay 等 |
| 获取手机号 | 需用户主动授权 | 需用户手动输入 | 可选择授权或手动输入 |

**注意**：仅在产品同时存在多种形态时才需要此对比表，否则不需要。

**截图规范：**
- 使用微信正式版截图（非开发者工具模拟器）
- 保留微信导航栏（胶囊按钮区域），这是小程序的重要视觉特征
- 状态栏显示正常的信号/WiFi、时间、电量
- 隐藏用户的真实昵称、头像、个人信息
- 截图比例建议接近真实手机屏幕比例（9:19.5 或类似）
