# A/B 实验分析

A/B 实验的核心关系：**对照组是基准，实验组是变量**。分析的本质不是"实验组的数字好不好看"，而是"实验组相对对照组变化了多少、变化是否符合预期、变化背后的原因是什么"。

---

## 工作流程

```
输入文件 → 数据探查与清洗 → 场景识别与指标路由 → 六步分析 → 输出报告
```

**用户指令优先原则贯穿整个 SKILL，适用于以下三个方面：**

1. **所有分析内容用户指令优先**——用户明确指定了场景、指标定义、分群维度、决策选项、阈值等任何参数时，严格以用户指令为准，本 SKILL 的默认设置仅作为用户未指定时的兜底
2. **输出要求严格遵照**——用户对输出格式、章节结构、文件命名有任何要求时，严格按用户要求输出，不增不减（不额外添加用户未要求的章节，也不省略用户要求的章节）
3. **计算指标用户指定优先**——用户指定了要计算哪些指标时，只计算用户指定的指标；用户未指定时，由模型根据场景规则文件自主决定计算哪些指标，但应分析到的都要分析到，不得遗漏

---

## 一、输入与输出

支持输入：`.csv`（自动检测 UTF-8/GBK）、`.tsv`、`.xlsx/.xlsm`（多 Sheet 默认读所有sheet）、`.json`（自动展平嵌套）。

支持输出：`.md`（默认）、`.docx`、`.xlsx`（每个分析步骤一个 Sheet）、`.html`。

用户指定文件名时以扩展名为准；未指定时默认 Markdown，输出到输入文件同目录。

---

## 二、数据探查与清洗

**在执行任何业务计算之前，必须先完成数据探查。** 用户未对输出章节做要求时，将探查结果作为报告的独立章节输出；用户指定了输出章节时，将探查发现融入用户要求的对应章节中，不额外添加章节。

必须检查：文件可读性、数据规模、字段结构与类型、分组字段识别、缺失值、重复数据、异常值（3 倍 IQR）、数据一致性（如 is_paid=0 但 is_refunded=1）、编码问题。

默认执行：去除空行、去除重复行（基于 user_id 或全行）、字段类型修正、分组字段标准化为 `control` / `experiment`。

需要用户确认或报告中标注：删除异常值行、填充缺失值、合并多文件。

多文件时：同一实验不同时间段 → 拼接；同一实验不同维度（有共同 user_id）→ 合并；不同实验 → 多任务模式。无法判断时按最合理推断执行并注明。

---

## 三、场景识别与指标路由

按以下优先级确定场景，然后读取对应的 references 文件执行分析：

1. **用户明确指定** → 直接采用
2. **用户描述中可推断** → 根据关键词判断
3. **从数据字段推断** → 根据字段名判断
4. **无法判断** → 使用本文件第四节的通用分析框架

| 场景 | 规则文件 |
|------|---------|
| 电商/交易 | `references/e-commerce-rules.md` |
| 推荐算法 | `references/recommendation-rules.md` |
| 推送策略 | `references/push-notification-rules.md` |
| 定价/促销 | `references/pricing-promotion-rules.md` |
| 新用户引导 | `references/onboarding-rules.md` |
| 内容/文案/UI | `references/content-ui-rules.md` |

每个规则文件包含该场景的：核心指标与公式、护栏指标与风险方向、风险判定阈值、常见分群维度、典型矛盾信号与应对、场景特殊考量。

---

## 四、六步分析框架

每次分析必须按顺序执行。用户指定跳过某些步骤时，注明"按用户要求跳过"及可能的影响。

### 第零步：实验背景确认

提取：实验改动了什么、核心假设、预期指标方向。用户未提供时注明背景缺失，解读结论标注"基于字段推断，建议结合实验背景复核"。

### 第一步：SRM 检查

计算实验组与对照组的实际人数比例，与预期比例对比。默认按 50:50 检验，偏差超过 5% 时标注 SRM 风险。用户可指定非 50:50 比例。

### 第二步：整体指标计算

**计算指标用户指定优先**——用户指定了要计算哪些指标时，只计算用户指定的指标及其公式；用户未指定时，根据场景规则文件中的指标体系，选择数据字段能支撑的指标全部计算（应分析到的都要分析到，不得遗漏），并注明无法计算的指标及原因。输出绝对值和 Lift = (实验组 - 对照组) / 对照组。分母与指标定义严格一致；缺失值不计入；用户未明确公式时注明计算口径。

### 第三步：护栏指标检查

护栏指标是"不能变差"的底线。用户指定了护栏指标时，只检查用户指定的；用户未指定时，按场景规则文件中的护栏指标全部检查。正常则标注"通过"；出现显著负向变化则标注风险，在第五步重点解读。具体异常模式参见各场景规则文件。

### 第四步：分群拆解

**每次分析都必须执行。** 用户指定单个维度 → 按该维度；多个维度 → 默认单维度分别拆解（用户要求交叉时检查子群样本量 < 30 注明存疑）；未指定 → 自动识别可用分群字段，无则注明无法执行。

### 第五步：结果解读与风险识别

数据干净时：回答提升来源、幅度是否符合预期、新鲜效应风险、可持续性。存在矛盾信号时：检查指标间矛盾、分群间矛盾、判断冲突，给出明确数据支撑。具体矛盾模式参见各场景规则文件。

### 第六步：差异化决策

用户指定了决策选项时，严格从用户给定的选项中选择（如用户要求三选一则三选一，不可自行添加其他选项）。

用户未指定决策选项时，从以下默认选项中选择：

| 决策类型 | 条件 |
|---------|------|
| 全量上线 | 核心正向，护栏无异常，分群无明显问题 |
| 分群差异化上线 | 不同分群结论不同 |
| 条件上线 | 核心正向但存在可控风险 |
| 暂缓，补充实验 | 结论不明确 |
| 暂缓，排查根因 | 护栏异常或 SRM 未通过 |

每条建议须包含：支撑数据、建议方向、执行路径。

## 五、用户可指定参数

用户可通过自然语言指定分析参数，未指定时使用默认值：

- 实验场景：自动识别（参见第三节）
- 核心指标 / 护栏指标：按场景规则文件
- 分群维度：自动识别可用字段；是否交叉：默认否
- 预期分流比例：50:50；SRM 阈值：5%
- 最小样本量：总样本 200，单组 100，分群子群 30
- 输出格式 / 路径 / 文件名：默认 Markdown，同目录
- 跳过步骤 / 额外要求：无

---

## 六、输出规范

输出须覆盖六步分析的完整结果，数据与结论一一对应，不得因"无异常"而省略分群和解读。

**用户对输出有要求时，严格遵照，不增不减**——用户指定了章节结构时，只输出用户要求的章节，不额外添加数据探查、SRM 检查等用户未要求的章节，也不省略用户要求的任何章节。

用户未指定输出结构时，默认章节：实验背景 → 数据探查报告 → 数据质量检验 → 核心数据总览 → 护栏指标检查 → 分群细节 → 结论 → 隐藏风险（≥3条，含风险等级/描述/影响）→ 业务建议 → 核心建议（引用块格式，3-4条）。

格式细节：百分比保留两位小数，Lift 保留两位小数。

---

## 七、单任务与多任务

**单任务**：一个数据文件，或多个文件但属于同一实验，输出一份报告。

**多任务**：多个属于不同实验的文件，每个实验独立执行六步分析、独立输出报告，额外输出一份对比摘要。用户指定合并输出时合并为一份文档。

---

## 八、异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 缺失值 | 不计入分子分母，备注说明 |
| 整体样本不足（单组<100 或总<200） | 注明可靠性存疑 |
| 分群样本不足（子群<30） | 注明可靠性存疑 |
| SRM 偏差超阈值 | 标注分流异常风险 |
| 指标定义与字段不一致 | 以实际字段为准，注明差异 |
| 实验周期过短 | 注明新鲜效应风险 |
| 无法识别分组字段 | 终止，提示用户指定 |
| 文件无法读取 | 终止并报错 |
