# 指标维度拆解标准映射表

本文件定义各类核心业务指标的标准拆解维度、常见根因与分析示例，供 pm_metric_rca SKILL 在"维度拆解"步骤中读取使用。

---

## 流量类指标

### DAU / MAU / 活跃用户数

**优先拆解维度（按优先级排序）**：
1. 渠道来源（自然流量 / 付费投放 / 推送召回 / 分享裂变）
2. 用户类型（新用户 / 回流用户 / 活跃老用户）
3. 平台/设备（iOS / Android / Web / 小程序）
4. 地域（一线城市 / 新线城市 / 海外）
5. 时间粒度（分时段看是否有特定时间段异常）

**常见根因与分析思路**：

| 根因类型 | 具体原因 | 分析方式 | 示例 |
|----------|----------|----------|------|
| 投放变化 | 付费投放预算削减、素材效果下降 | 对比投放渠道新增用户数变化 | 某周 DAU 下跌 8%，拆解发现信息流投放新增用户减少 40%，其余渠道正常 |
| 召回效率 | Push 到达率下降、内容点击率低 | 查 Push 发送量/到达率/点击率漏斗 | iOS 系统升级后用户关闭通知权限比例上升，Push 到达率从 60% 降至 35% |
| 季节性效应 | 节假日、开学季、淡旺季 | 与去年同期对比（同比），排除季节性 | 春节期间 DAU 下跌属正常，需与去年春节同期对比判断是否超出正常范围 |
| 竞品冲击 | 竞品上线新功能或大促活动 | 结合竞品动态与用户流失时间节点对比 | 竞品发布重大版本后，产品 DAU 连续 3 天下跌，流失用户中高频用户占比高 |
| 版本强制升级 | 强制升级导致部分用户流失 | 对比升级前后各版本活跃用户数 | 强制升级后低端机型用户流失，因新版本对低端机性能要求更高 |

## 转化类指标

### 转化率（注册转化 / 购买转化 / 功能使用转化）

**优先拆解维度**：
1. 漏斗节点（哪一步转化率下降最显著）→ 自动进入漏斗子路径
2. 流量来源（不同来源用户的转化率差异）
3. 用户分层（新用户 vs 老用户转化率对比）
4. 页面/版本（是否某个 AB 版本或新页面导致）
5. 设备/网络（是否特定机型或网络环境下转化异常）

**三维根因分析框架**：

**① 看输入（用户池质量）**

用户池的质量决定了转化的天花板，同样的产品对不同质量的流量转化率差异可达 3-5 倍。

| 根因 | 分析方式 | 示例 |
|------|----------|------|
| 流量来源结构变化 | 对比各渠道用户转化率，计算渠道结构变化对整体转化率的影响 | 付费投放占比从 30% 升至 50%，而投放用户转化率仅为自然用户的 1/3，导致整体转化率下跌 |
| 用户意图不匹配 | 查看搜索词/推荐内容与落地页的相关性 | 推荐算法调整后，商品详情页流量增加但转化率下跌，因推荐内容与用户历史偏好匹配度降低 |
| 淡旺季效应 | 同比去年同期转化率 | 双十一后转化率下跌，高意图用户已在活动期间消耗完，剩余用户池购买意愿低 |
| 新老用户结构变化 | Cohort 分析，对比新老用户转化率及占比变化 | 新用户占比从 20% 升至 40%，而新用户转化率仅为老用户的 1/2，拉低整体转化率 |

**② 看业务（业务逻辑与规则变动）**

| 根因 | 分析方式 | 示例 |
|------|----------|------|
| 价格/优惠策略调整 | 对比调整前后转化率，结合价格弹性分析 | 满减门槛从 99 元提高至 149 元，加购→下单转化率下跌 12% |
| 供给侧变化 | 查看商品/服务数量、质量评分、库存状态 | 头部商家下架导致热门品类商品数减少 30%，该品类转化率下跌 |
| 新增业务节点 | 对比新节点上线前后的漏斗转化率 | 新增手机号验证步骤后，注册转化率下跌 18%，流失集中在验证码发送环节 |
| 风控策略收紧 | 查看风控拦截率变化 | 反欺诈策略升级后，正常用户误拦截率上升，支付成功率下跌 5% |

**③ 看流程（用户交互体验）**

| 根因 | 分析方式 | 示例 |
|------|----------|------|
| 页面加载速度劣化 | 查看关键页面 LCP/FCP 指标变化 | 商品详情页图片未压缩，加载时长从 1.2s 升至 3.5s，跳出率上升 25% |
| 交互步骤增加 | 对比新旧版本操作路径步骤数 | 结算页重构后操作步骤从 3 步增至 5 步，提交订单转化率下跌 9% |
| CTA 文案不清晰 | 用户研究/热力图分析关键按钮点击率 | "立即体验"改为"免费试用 7 天"后，注册按钮点击率提升 22% |
| 错误提示不友好 | 查看表单提交失败率及错误类型分布 | 密码格式要求未在输入框旁说明，用户反复提交失败后放弃注册 |

## 留存类指标

### 次日留存 / 7日留存 / 30日留存

**优先拆解维度**：
1. 用户注册时间（按注册日期分 Cohort，看是否特定批次用户留存差）
2. 首次使用功能（不同首次体验路径的留存率对比）
3. 渠道来源（不同渠道获取用户的留存质量差异）
4. 用户行为深度（首日完成核心动作 vs 未完成的留存对比）

**常见根因与示例**：

| 根因类型 | 具体原因 | 分析方式 | 示例 |
|----------|----------|----------|------|
| 新用户引导变更 | Onboarding 流程调整导致用户未能到达 Aha Moment | 对比新旧版本新用户首日核心动作完成率 | 新手引导步骤增加后，首日完成核心动作的用户比例从 45% 降至 28%，次日留存同步下跌 |
| 渠道质量下降 | 新渠道带来的用户与产品核心价值不匹配 | Cohort 分析各渠道用户留存曲线 | 某信息流渠道带来大量用户，但 7 日留存仅 5%，远低于自然用户的 22% |
| 召回机制失效 | Push 权限丢失、内容质量下降 | 查看 Push 到达率、点击率、带来的回访用户数 | iOS 升级后 Push 到达率下降，7 日留存中回流用户贡献减少 40% |
| 核心功能体验劣化 | 高频功能出现 Bug 或性能下降 | 对比核心功能使用率与留存率的相关性 | 搜索功能响应时间增加后，高频使用搜索的用户 30 日留存下跌 15% |
| 竞品替代 | 竞品推出更有吸引力的功能 | 结合竞品动态与用户流失时间节点 | 竞品上线独家内容后，产品内容消费类用户 30 日留存持续下滑 |

## 收入类指标

### GMV / 订单量 / ARPU / 付费率

**指标拆解公式**：GMV = 订单量 × 客单价 = DAU × 转化率 × 客单价

**优先拆解维度**：
1. 品类/业务线（哪个品类贡献了主要波动）
2. 用户价值分层（高价值用户 / 普通用户 / 新用户）
3. 订单结构（客单价变化 vs 订单量变化）
4. 促销活动（有无促销叠加或促销结束效应）
5. 支付环节（支付成功率是否有变化）

**拆解优先级判断**：

| 现象 | 优先排查方向 |
|------|-------------|
| 订单量下跌，客单价稳定 | 重点看转化率和流量（用户池问题） |
| 客单价下跌，订单量稳定 | 重点看品类结构和促销策略 |
| 两者都跌 | 可能是系统性问题（支付链路故障、大规模用户流失） |
| GMV 涨但利润跌 | 看促销成本和品类结构（低毛利品类占比上升） |

**常见根因与示例**：

| 根因类型 | 具体原因 | 示例 |
|----------|----------|------|
| 促销效应 | 大促结束后的自然回落 | 双十一后 GMV 下跌 40%，属正常回落，需与去年双十一后同期对比 |
| 竞品价格战 | 竞品大幅降价或补贴 | 竞品推出 9.9 包邮活动，产品同品类订单量下跌 25% |
| 支付链路故障 | 支付成功率下跌 | 第三方支付接口超时，支付成功率从 92% 降至 78%，GMV 损失约 15% |
| 供给侧变化 | 头部商家/内容下架 | 头部商家因合同到期下架，该品类 GMV 下跌 30% |
| 高价值用户流失 | 高消费用户活跃度下降 | Top 10% 用户贡献 60% GMV，该群体 30 日活跃率下跌 10%，GMV 下跌 8% |

## 质量类指标

### 崩溃率 / 错误率 / 加载时长 / ANR 率

**优先拆解维度**：
1. 版本号（是否特定版本引入）
2. 设备型号/系统版本（是否集中在特定机型）
3. 功能模块（哪个页面或功能的错误率最高）
4. 网络环境（Wi-Fi vs 移动网络）

**常见根因与示例**：

| 根因类型 | 具体原因 | 示例 |
|----------|----------|------|
| 新版本 Bug | 特定版本引入的代码问题 | v3.2.0 上线后崩溃率从 0.1% 升至 1.8%，集中在支付页面，定位为新引入的 SDK 兼容性问题 |
| 第三方 SDK 更新 | SDK 升级引入不兼容问题 | 地图 SDK 升级后，Android 8.0 以下机型 ANR 率上升 |
| CDN/服务端故障 | 资源加载失败或接口超时 | 图片 CDN 节点故障，首页加载时长从 1.5s 升至 8s，跳出率上升 40% |
| 特定机型兼容性 | 新机型或系统版本适配问题 | 某品牌新旗舰机型发布后，该机型崩溃率是平均水平的 5 倍 |

## 通用注意事项

- 拆解时优先找"贡献度最大的子维度"，而非逐一罗列所有维度数据
- 若某维度数据不可获取，在输出中注明"该维度数据待补充"，不得跳过
- 维度拆解结论应指向"波动集中在 X 维度的 Y 子类"，而非模糊描述
- 同比与环比结论不一致时，优先排查季节性因素，再看业务变化
- 单一维度拆解无法定位时，尝试交叉维度（如"iOS 新用户的转化率"）
