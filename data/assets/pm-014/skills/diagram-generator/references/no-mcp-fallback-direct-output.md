# 直出兜底规范（区分 B1 / B2）

本文件定义直出通道的标准执行方案，并明确区分两类触发场景：
- **B1：无 MCP 预检直出**（启动阶段未检测到可用 MCP）
- **B2：MCP 失效降级直出**（已进入 A，但 MCP 在执行中失败）

## 适用条件

满足任一条件进入直出：

### B1 触发条件（无 MCP）
- 未检测到图表生成 MCP 工具
- 关键工具（如 `generate_diagram`）不存在或不可调用

### B2 触发条件（MCP 失效）
- 已检测到 MCP 可用并进入 A
- MCP 调用报错
- MCP 调用超时
- MCP 返回无效结果（空结果/结构不完整/落盘失败）

## 目标

在无法完成 A 通道生成时，仍稳定产出可编辑流程图文件，确保任务不中断。

## B1 与 B2 的执行差异

- **B1（无 MCP）**：从需求抽取开始，完整执行直出链路。
- **B2（失效降级）**：优先复用 A 通道已完成的需求理解与结构化规格，减少信息损失与重复解析。

## 直出执行步骤

### 步骤 1：需求抽取（B1 必做，B2 按需补做）

从用户输入提取：
- 流程目标
- 关键步骤
- 决策分支与标签
- 角色参与关系（泳道/时序）
- 异常路径（失败/重试/回退）
- 输出格式与路径

补充规则：
- B1：信息不足时按行业通用流程补全初稿
- B2：优先沿用 A 通道中间结果，仅补齐缺失字段

### 步骤 2：构建或复用中间结构

统一结构 `elements`：
- 节点：`start/process/decision/end/state`
- 连线：`normal/yes/no/timeout/retry/exception`

约束：
- 节点 id 唯一
- 边必须指向已存在节点
- 决策节点至少两条分支且有语义标签
- 至少 1 个起点和 1 个终点

差异规则：
- B1：从零构建 `elements`
- B2：优先复用 A 通道已有 `elements`，禁止无故删减关键分支

### 步骤 3：按格式直出

- `mermaid`：生成 Mermaid 并写入 `.mmd/.md`
- `drawio`：生成可编辑 draw.io 内容并写入 `.drawio`
- `excalidraw`：生成可编辑 Excalidraw JSON 并写入 `.excalidraw`

默认策略：
- 用户未指定格式时默认 `mermaid`
- 用户指定格式时按指定格式输出
- B2 如用户在 A 通道已明确格式，则降级后不得擅自改格式

### 步骤 4：落盘与冲突处理

- 目录不存在则自动创建
- 默认目录：
  - `drawio` -> `diagrams/drawio/`
  - `mermaid` -> `diagrams/mermaid/`
  - `excalidraw` -> `diagrams/excalidraw/`
- 文件重名且未要求覆盖：自动追加时间戳

## 质量检查清单（B1/B2 必检）

- 文件写入成功
- 扩展名与格式一致
- 连线无悬空引用
- 决策分支语义清楚（通过/不通过/异常等）
- 主链路从起点可达终点
- 异常链路可闭环

B2 额外检查：
- A 通道已解析的关键节点未丢失
- 降级后格式/路径/文件名与用户要求保持一致
- 记录降级原因（建议写入 runtime.degradeReason）

## 常见失败与修复

- **失败：流程断链**
  - 修复：检查每条边的 source/target 是否存在
- **失败：决策分支不足**
  - 修复：补齐至少两条分支，并补标签
- **失败：输出格式不符合用户要求**
  - 修复：按用户格式重生成，不擅自改格式
- **失败：图过于复杂难阅读**
  - 修复：分段拆图，保留主流程 + 子流程链接
- **失败：B2 降级后信息回退**
  - 修复：回填 A 通道中间规格，重新导出

## 快速模板

### 审批流程模板
- 提交 -> 经理审批 -> 财务复核 -> 审批通过/驳回 -> 通知

### 故障应急模板
- 告警 -> 影响评估 -> 自动恢复/人工介入 -> 验证恢复 -> 复盘

### 发布流程模板
- 提交发布 -> 检查 -> 灰度 -> 监控 -> 全量/回滚
