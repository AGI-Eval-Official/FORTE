# 流程图执行策略指南（三通道：A / B1 / B2）

本技能采用固定优先级，不要求用户做技术选择：
1. 先做环境评估
2. 能走 MCP 就优先走工业级通道 A
3. 不能走 A 时，进入直出通道（B1 或 B2）

## 三个通道的定义

- **通道 A（MCP 工业级生成）**：检测到可用 MCP 工具，并且 MCP 调用成功。
- **通道 B1（无 MCP 预检直出）**：在第 0 步就没检测到可用 MCP 工具，直接直出。
- **通道 B2（MCP 失效降级直出）**：第 0 步检测到 MCP 可用，但在执行 A 过程中调用失败/超时/返回无效，再降级直出。

> 关键区别：B1 是“起跑前就无 MCP”，B2 是“起跑后 MCP 掉线或失效”。

## 第 0 步：环境评估（必须先执行）

检测工具（示例）：
- `mcp__mcp-diagram-generator__get_config`
- `mcp__mcp-diagram-generator__generate_diagram`
- `mcp__mcp-diagram-generator__init_config`

判定规则：
- `generate_diagram` 不存在或不可调用 -> 直接进入 **B1**
- `generate_diagram` 可调用 -> 进入 **A**

## 通道 A：MCP 工业级稳定生成

适用：
- 复杂流程（多分支、多角色、多异常路径）
- 对结构一致性、可重复性要求高

执行要点：
- 先构建结构化流程规格
- 调用 MCP 生成最终文件
- 校验结果（文件存在、格式正确、结构完整）
- 若调用失败，按失败类型切换到 **B2**

## 通道 B1：无 MCP 预检直出

触发条件：
- 启动阶段未检测到可用 MCP 工具

执行要点：
- 不再重复探测 MCP，直接走直出生成
- 直接生成 Mermaid/Drawio/Excalidraw 内容并写文件
- 优先保证流程完整、可读、可编辑

## 通道 B2：MCP 失效降级直出

触发条件：
- 已进入 A，但 MCP 调用报错/超时/返回无效结果

执行要点：
- 继承 A 通道已解析出的结构化规格
- 保持用户要求的格式、路径、文件名优先级不变
- 记录失败原因并完成同轮次直出交付

## 格式落地规则（A/B1/B2 通用）

- 用户未指定格式：默认 `mermaid`（`.mmd`）
- 用户指定格式：按指定格式输出
- 复杂泳道/强视觉排版：优先 `drawio`
- 草图表达：`excalidraw`

## 输出路径与命名

默认目录：
- `drawio` -> `diagrams/drawio/`
- `mermaid` -> `diagrams/mermaid/`
- `excalidraw` -> `diagrams/excalidraw/`

命名规则：
- 优先用户指定文件名
- 未指定：`流程图主题-日期`
- 同名冲突且未要求覆盖：自动加时间戳

## 质量基线（A/B1/B2 都必须满足）

- 节点/连线不悬空
- 决策节点至少两条分支且有语义标签
- 流程从起点可达终点
- 异常链路可闭环
- 文件真实写入成功

## 常见触发表达（命中即执行）

- “画流程图 / 改流程图 / 转流程图”
- “做审批流 / SOP / 时序图 / 泳道图 / 状态图”
- “梳理流程并导出 drawio 或 mermaid”
- “先尝试工业级生成，不行就直接给我可编辑图”
