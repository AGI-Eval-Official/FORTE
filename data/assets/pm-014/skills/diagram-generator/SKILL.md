# diagram-generator（MCP优先 + 直出兜底）

## 技能目标

将用户自然语言需求转换为可编辑流程图文件，并支持在已有图表上修改。

本技能采用双通道执行：
- 通道 A：MCP 可用 -> 工业级稳定生成
- 通道 B：MCP 不可用 -> 直出生成（当前版本兜底能力）

## 强触发规则

只要命中流程图相关关键词或同义表达，即必须触发本技能。

典型触发词：
- 流程图、审批流、SOP、业务流程、系统流程、流程梳理、流程设计
- 时序图、泳道图、状态图、活动图、数据流程图
- 画图、改图、转图、流程可视化、导出 drawio、导出 mermaid、导出 excalidraw

## 支持范围

- 新建：从文字直接生成流程图
- 改图：读取已有图并按要求增删改
- 导出：`.drawio` / `.mmd` / `.md` / `.excalidraw`

图类型覆盖：
- 流程图、审批流程、SOP流程
- 时序流程、泳道流程、状态流转图、活动图
- 异常处理流程、故障应急流程、发布上线流程
- 工单流转流程、订单履约流程、权限申请流程、报销流程

## 默认输出规则

- 用户未指定格式：默认输出 `mermaid`（`.mmd`）
- 用户指定格式：按指定格式输出
- 默认目录：
  - `drawio` -> `diagrams/drawio/`
  - `mermaid` -> `diagrams/mermaid/`
  - `excalidraw` -> `diagrams/excalidraw/`
- 文件名：优先用户指定，否则 `流程图主题-日期`
- 同名冲突且未要求覆盖：自动追加时间戳

## 执行流程

### 第 0 步：先评估环境是否可用 MCP

检测是否可调用图表生成相关工具（示例）：
- `mcp__mcp-diagram-generator__get_config`
- `mcp__mcp-diagram-generator__generate_diagram`
- `mcp__mcp-diagram-generator__init_config`

判定规则：
- 三者可用（或至少 `generate_diagram` 可用） -> 走通道 A
- 不可用或调用失败 -> 走通道 B

注意：
- 不要求用户预先配置环境
- 不因 MCP 不可用而中断任务

### 第 1 步：理解用户需求

提取：
- 流程目标
- 关键步骤
- 决策分支
- 参与角色（泳道/时序）
- 异常路径（失败/超时/重试/回退）
- 输出路径/文件名/格式

信息不完整时：
- 先按通用业务常识补全，快速产出初版
- 再根据反馈迭代

### 第 2 步：构建结构化流程规格

统一组织为节点 + 连线：
- 节点类型：`start/process/decision/end/state`
- 连线类型：`normal/yes/no/timeout/retry/exception`
- 约束：
  - 节点 id 唯一
  - 每条边必须指向存在节点
  - 决策节点至少两条带标签分支
  - 流程可从起点走到终点

### 第 3A 步（通道 A）：MCP 工业级生成

当 MCP 可用时：
1. 将结构化流程规格传入 `generate_diagram`
2. 使用用户指定路径/文件名；未指定则按默认规则
3. 校验生成结果（文件是否存在、格式是否正确、结构是否完整）
4. 如 MCP 调用失败，立即切换到通道 B 兜底

### 第 3B 步（通道 B）：直出生成

当 MCP 不可用时：
- `mermaid`：直接生成 Mermaid 文本并写入 `.mmd/.md`
- `drawio`：生成可编辑 draw.io 内容并写入 `.drawio`
- `excalidraw`：生成可编辑 Excalidraw JSON 并写入 `.excalidraw`

详细兜底执行规范见：`references/no-mcp-fallback-direct-output.md`

### 第 4 步：改图流程

当用户要求修改已有图：
1. 读取原图
2. 保留主干结构
3. 按需求增删节点、分支、连线、文案
4. 输出覆盖版或新文件

## 质量标准

每次输出前必须自检：
- 文件真实写入成功
- 扩展名与格式一致
- 无悬空节点与断裂连线
- 决策分支语义清晰（如“通过/不通过”）
- 异常链路可闭环
- 文案命名一致、可读性高

## 常用模板（自动套用）

- 审批流程：提交 -> 审核 -> 审批 -> 通知
- 登录流程：输入 -> 校验 -> 成功/失败 -> 重试
- 工单流程：创建 -> 分派 -> 处理 -> 验收 -> 关闭
- 异常流程：告警 -> 判定 -> 自动恢复/人工介入 -> 复盘
- 发布流程：检查 -> 灰度 -> 监控 -> 全量/回滚

## 行为边界

- 不伪造结果：必须产出真实文件
- 不因环境差异中断：MCP 不可用时自动走直出通道
- 不强迫用户做技术选择：技能内部自动判断与执行
- 不偏离用户目标：范围调整时需明确说明
