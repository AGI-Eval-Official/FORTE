# 员工绩效考核评估报告生成

本技能帮助HR根据不同的绩效考核方式，动态路由到对应的考核规则，生成专业的绩效评估报告。支持六种主流考核方式：平衡计分卡（BSC）、关键绩效指标（KPI）、360度考核、目标与关键成果法（OKR）、目标管理法（MBO）、薪酬全绩效（KSF）。

## 为什么需要这个技能

绩效考核是一项系统性、专业性很强的工作。不同考核方式有不同的评估维度、评分标准和报告结构。使用错误的考核框架会导致评估偏差，影响员工发展和组织决策。本技能确保每次考核都严格遵循对应的方法论，输出规范、可追溯的评估报告。

## 工作流程（SOP）

### 第一步：需求分析与考核方式识别

分析用户输入的需求文本，识别出绩效考核方式。判断依据如下：

| 关键词线索 | 路由到的考核方式 |
|-----------|---------------|
| 平衡计分卡、BSC、财务/客户/内部/学习四个维度、战略目标分解 | BSC（读取 `references/bsc.md`） |
| 关键绩效指标、KPI、量化指标、目标值、完成率、指标权重 | KPI（读取 `references/kpi.md`） |
| 360度、全方位评价、多源反馈、上级/下级/同事/自评、周围人评价 | 360度考核（读取 `references/360-degree.md`） |
| OKR、目标与关键成果、Objective、Key Result、挑战性目标、对齐 | OKR（读取 `references/okr.md`） |
| 目标管理法、MBO、上下协商目标、目标分解、目标承诺、目标体系 | MBO（读取 `references/mbo.md`） |
| 薪酬全绩效、KSF、关键成功因子、平衡点、薪酬绩效融合、增值分配 | KSF（读取 `references/ksf.md`） |

如果用户未明确指定考核方式，根据以下优先级推断：
1. 需求文本中包含的显式关键词 → 直接匹配
2. 描述了"战略分解""多维度平衡"等概念 → 推断为BSC
3. 描述了"量化指标""达标率"等概念 → 推断为KPI
4. 描述了"多方评价""不同视角"等概念 → 推断为360度考核
5. 描述了"目标对齐""挑战性成果"等概念 → 推断为OKR
6. 描述了"上下协商目标""目标分解""目标承诺"等概念 → 推断为MBO
7. 描述了"薪酬与绩效挂钩""平衡点""增值分配"等概念 → 推断为KSF

### 第二步：加载对应的Reference规则

根据第一步识别出的考核方式，读取对应的reference文件。**只读取与当前考核方式对应的reference文件**，不要加载无关的reference：

- BSC → 读取 `references/bsc.md`
- KPI → 读取 `references/kpi.md`
- 360度考核 → 读取 `references/360-degree.md`
- OKR → 读取 `references/okr.md`
- MBO → 读取 `references/mbo.md`
- KSF → 读取 `references/ksf.md`

### 第三步：基于用户场景执行评估（保护层）

**⚠️ 优先级原则：用户提供的场景/规则 > Skill通用方法**

- **用户优先**：首先严格遵循用户提供的考核维度、评分标准、权重分配、计算方法等具体要求
- **查漏补缺**：当用户提供的规则存在缺失或不完整时，才使用reference中的通用规则进行补充
- **冲突处理**：若用户提供的规则与reference存在冲突，以用户提供的为准，并在报告中注明

具体执行顺序：
1. **解析用户规则**：提取用户输入中明确指定的考核维度、评分标准、权重、计算公式等
2. **对比reference**：与加载的reference规则进行对照
3. **查漏补缺**：仅对用户未覆盖的部分，使用reference规则进行补充
4. **标记来源**：在评估报告中标注各规则的来源（用户提供/补充来源）

### 第四步：执行评估计算

基于用户提供的规则（优先级高）结合reference中的通用方法（查漏补缺），执行以下操作：
1. 按照用户规则（优先）或reference规则（补充）中的评估维度和评分标准，对输入数据进行分析
2. 按照用户指定的或reference中的计算方法，计算各项得分和综合得分
3. 按照用户指定的或reference中的等级划分标准，确定绩效等级
4. 生成改进建议和发展计划

### 第五步：生成绩效考核评估报告

若input提供了固定报告框架，以input框架为准；若input无报告框架要求，则使用以下默认模板
默认模板必须包含以下结构：

```
# 绩效考核评估报告

## 基本信息
- 被考核人：[姓名]
- 考核周期：[时间范围]
- 考核方式：[BSC/KPI/360度/OKR/MBO/KSF]
- 评估日期：[日期]

## 考核概述
[简要说明本次考核的背景、目的和采用的考核方法]

## 评估维度与得分
[按照对应考核方式的reference规则，列出各维度的评分详情]
**⚠️ 规则来源标注**：各评估维度需标注来源（用户提供/通用方法补充）

## 综合评分与等级
- 综合得分：[分数]
- 绩效等级：[等级]

## 优势分析
[列出被考核人的核心优势]

## 待改进领域
[列出需要改进的方面]

## 改进建议与发展计划
[针对待改进领域，提出具体的改进措施和发展建议]

## 评估说明
[说明本次评估的局限性、数据来源等注意事项]
```

## 使用模式与示例

### 示例1：明确的BSC考核需求

**输入：**
> 请用平衡计分卡的方式对技术部张三进行本季度绩效考核。他在财务维度完成了3个项目收入200万，客户维度满意度92分，内部流程维度项目按时交付率95%，学习成长维度完成了2项技术认证。

**处理流程：**
1. 识别关键词"平衡计分卡" → 路由到BSC
2. 读取 `references/bsc.md`
3. 按BSC四维度评分规则计算各维度得分
4. 按报告模板生成BSC绩效评估报告

### 示例2：模糊需求需要推断

**输入：**
> 我要对销售团队进行考核，主要看他们的业绩指标完成情况，比如销售额达成率、客户开发数量等。

**处理流程：**
1. 关键词"业绩指标""达成率" → 推断为KPI考核
2. 读取 `references/kpi.md`
3. 按KPI指标评分规则计算各项得分
4. 按报告模板生成KPI绩效评估报告

### 示例3：MBO考核需求

**输入：**
> 请用目标管理法对产品部李四进行年度绩效考核。年初协商设定了5项目标：产品上线2个、用户增长率30%、NPS达到50、团队培训4次、成本节约10%。目前产品上线2个完成，用户增长率25%，NPS达到48，团队培训3次，成本节约12%。

**处理流程：**
1. 识别关键词"目标管理法" → 路由到MBO
2. 读取 `references/mbo.md`
3. 按MBO目标达成度评分规则计算各目标得分
4. 按报告模板生成MBO绩效评估报告

### 示例4：KSF考核需求

**输入：**
> 用薪酬全绩效方式对销售部王五进行考核。他的KSF指标：销售额平衡点50万实际60万，毛利率平衡点35%实际38%，新客户数平衡点5个实际7个，客户满意度平衡点85分实际82分，回款率平衡点95%实际97%。

**处理流程：**
1. 识别关键词"薪酬全绩效" → 路由到KSF
2. 读取 `references/ksf.md`
3. 按KSF平衡点和分配规则计算各指标浮动薪酬
4. 按报告模板生成KSF绩效评估报告，含薪酬测算明细


## 注意事项

- **⚠️ 用户规则优先**：用户提供的考核规则具有最高优先级，skill通用方法仅用于查漏补缺
- 每次考核只使用一种考核方式，不要混合多种方式
- 评分必须严格遵循reference中定义的标准，不要凭主观判断调整分数
- 报告中的改进建议要具体、可执行，避免空洞的套话
- 综合得分计算方法必须与reference中的权重和公式一致
