# 目标与关键成果法（OKR）考核规则参考

## 概述

目标与关键成果法（Objectives and Key Results，OKR）是一套明确和跟踪目标及其完成情况的管理工具和方法。OKR由英特尔公司创始人安迪·格鲁夫发明，后被约翰·杜尔推广至谷歌等企业。OKR的核心思想是：通过设定挑战性的目标（Objective）和可衡量的关键成果（Key Results），激发员工潜力和创新力，推动组织目标对齐和聚焦。

## OKR与KPI的区别

理解OKR与KPI的本质区别，是正确使用OKR考核的前提：

| 维度 | OKR | KPI |
|------|-----|-----|
| 目标性质 | 挑战性目标，鼓励突破 | 可达成目标，确保完成 |
| 评分期望 | 0.6-0.7分为理想，1.0分可能目标过低 | 1.0分为理想，低于1.0为未达标 |
| 制定方向 | 自下而上为主，员工参与制定 | 自上而下为主，组织分解下达 |
| 更新频率 | 通常季度更新，每周跟踪 | 通常年度/半年度设定 |
| 结果应用 | 不直接挂钩薪酬，侧重发展 | 直接挂钩薪酬和晋升 |
| 公开透明 | 全员公开，互相可见 | 通常仅上级可见 |

## OKR评分规则

### 评分范围

OKR评分采用0-1.0分制（也常用0-100%制，本规则统一采用0-1.0分制）：

| 评分 | 含义 | 解读 |
|------|------|------|
| 0.9 - 1.0 | 完全达成 | 目标可能设定过低，缺乏挑战性 |
| 0.7 - 0.8 | 大部分达成 | 理想得分区间，目标具有适当挑战性 |
| 0.5 - 0.6 | 部分达成 | 目标具有较高挑战性，需评估是否需调整 |
| 0.3 - 0.4 | 少部分达成 | 目标过于激进或执行出现严重问题 |
| 0.0 - 0.2 | 几乎未达成 | 需要深入反思和调整 |

> **关键原则：** OKR得分为1.0并不一定是好事——如果所有OKR都是1.0，说明目标设定过于保守，没有发挥OKR推动突破的作用。理想状态是综合得分在0.6-0.7之间。

### 关键成果（KR）评分

每个KR的评分基于完成进度：

```
KR得分 = 实际完成值 ÷ 目标值
```

KR得分上限为1.0（超过100%仍记为1.0，因为OKR鼓励的是挑战性而非无限超越）。

### 目标（Objective）评分

```
O得分 = Σ（KRi得分 × KRi权重）
```

若KR无特殊权重，则：
```
O得分 = Σ（KRi得分）÷ KR数量
```

### 综合OKR评分

```
综合OKR得分 = Σ（Oi得分 × Oi权重）÷ 权重总和
```

若各Objective权重相同：
```
综合OKR得分 = Σ（Oi得分）÷ O数量
```

## OKR设定规范

### 目标（Objective）设定要求

1. **方向明确：** 描述要达成的方向，而非具体数字
2. **鼓舞人心：** 使用有感染力的语言，激发团队动力
3. **可定性判断：** 即使没有具体数字，也能判断方向是否正确
4. **可控可影响：** 在团队或个人的能力范围和影响力之内

**好的O示例：** "打造行业领先的用户体验"
**不好的O示例：** "用户满意度达到95%"（这是KR，不是O）

### 关键成果（KR）设定要求

1. **可量化：** 必须有明确的数字指标
2. **结果导向：** 描述成果而非行动过程
3. **完整覆盖：** 一组KR需完整支撑对应O的达成
4. **数量适中：** 每个O下设2-5个KR

**好的KR示例：** "用户NPS从30提升至50"
**不好的KR示例：** "优化用户界面"（这是行动，不是成果）

## OKR评估维度

### 1. 目标达成度（权重：40%）

评估OKR的最终完成情况。

| 评估项 | 说明 |
|--------|------|
| KR完成进度 | 各KR的实际完成率 |
| 综合OKR得分 | 加权平均后的整体完成度 |
| 得分分布合理性 | 各O的得分是否落在合理区间 |

### 2. 目标挑战性（权重：20%）

评估目标的进取程度。

| 评估项 | 评分标准 |
|--------|---------|
| 高挑战 | 目标超出当前能力范围但通过努力可能达成（5分） |
| 适中挑战 | 目标需要明显努力才能达成（4分） |
| 一般挑战 | 目标在现有能力范围内需适度努力（3分） |
| 低挑战 | 目标几乎不需额外努力即可达成（2分） |
| 无挑战 | 目标明显过于保守（1分） |

### 3. 对齐与协同（权重：20%）

评估个人/团队OKR与组织OKR的对齐程度。

| 评估项 | 评分标准 |
|--------|---------|
| 完全对齐 | 所有OKR与上级/组织OKR强关联（5分） |
| 高度对齐 | 大部分OKR与上级/组织OKR关联（4分） |
| 基本对齐 | 关键OKR与组织方向一致（3分） |
| 部分偏离 | 部分OKR与组织目标关联弱（2分） |
| 明显偏离 | OKR与组织方向不一致（1分） |

### 4. 执行过程（权重：20%）

评估OKR执行过程中的表现。

| 评估项 | 评分标准 |
|--------|---------|
| 持续跟踪 | 每周检查OKR进度，及时调整策略（5分） |
| 定期复盘 | 双周/月度复盘，主动暴露问题（4分） |
| 基本跟进 | 能按时检查但不深入分析（3分） |
| 偶尔关注 | 检查频率低，问题发现滞后（2分） |
| 缺乏跟踪 | 无人关注进度，OKR形同虚设（1分） |

## 综合评估得分计算

```
综合评估得分 = 目标达成度得分 × 40% + 目标挑战性得分 × 20% 
             + 对齐与协同得分 × 20% + 执行过程得分 × 20%
```

## 绩效等级划分

基于综合评估得分（注意：此等级体系与OKR原始得分不同，是对OKR执行质量的综合评定）：

| 等级 | 得分范围 | 描述 |
|------|---------|------|
| S（卓越） | 4.5 - 5.0 | OKR设定合理且高挑战，执行有力，达成度在理想区间 |
| A（优秀） | 3.8 - 4.4 | OKR设定具有挑战性，执行良好，大部分O达成 |
| B（良好） | 3.0 - 3.7 | OKR设定基本合理，执行正常，核心O部分达成 |
| C（待改进） | 2.0 - 2.9 | OKR设定或执行存在问题，需改进目标管理能力 |
| D（不合格） | 1.0 - 1.9 | OKR管理流于形式，需全面反思和改进 |

## OKR特有的报告要求

1. **OKR清单与评分表：** 逐项列出每个Objective及其下属Key Results，标注目标值、实际值、完成率和KR得分
2. **得分分析专节：** 分析综合OKR得分是否落在理想区间（0.6-0.7），如偏离需说明原因
3. **对齐映射图：** 展示个人OKR与团队/组织OKR的对齐关系
4. **挑战性评估：** 对每个O的挑战性进行评估，判断目标设定是否合理
5. **过程回顾：** 回顾OKR执行过程中的关键事件、调整记录和复盘情况
6. **下期OKR建议：** 基于本期OKR的执行经验，对下期OKR的设定提出建议
