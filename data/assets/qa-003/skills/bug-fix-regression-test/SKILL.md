# Skill: 缺陷定位修复 + 回归测试编写

## 概述

本技能用于根据提供的 bug 报告，在 Python 项目中定位并修复源码缺陷，然后编写 pytest 回归测试确保缺陷不再复现。

## 工作流程

### Phase 1：理解 Bug 报告

1. **通读所有 bug 报告**：逐个阅读输入中提供的 bug 报告（可来自目录、单文件或工单描述），理解每个 bug 的复现步骤、期望结果和实际结果
2. **按模块分组**：将 bug 按所属源码模块归类，同一模块的 bug 一起处理
3. **评估影响面**：判断修复某个 bug 是否可能影响同模块的其他函数

### Phase 2：定位根因

对每个 bug：

1. **阅读相关源码**：打开 bug 报告中指明的模块和函数
2. **对照复现步骤**：用 bug 报告中的输入值在脑中"走读"代码逻辑
3. **找到根因**：精确定位导致实际输出偏离期望输出的那行代码

### Phase 3：修复缺陷

1. **最小化修改**：只修改导致 bug 的代码，不做无关重构
2. **保持函数签名不变**：修复不应改变函数的输入输出接口
3. **注意副作用**：确认修复不会破坏函数的其他正常行为

### Phase 4：编写回归测试

1. **每个 bug 至少一个测试**：确保每个修复都有对应的回归测试
2. **直接使用 bug 报告中的复现用例**：bug 报告的复现步骤就是最好的测试用例
3. **验证期望结果**：断言应该匹配 bug 报告中描述的期望结果
4. **额外覆盖**：如果修复涉及边界条件，可以追加相关边界测试

### Phase 5：验证

运行全部测试，确保所有回归测试通过。

## 常见缺陷类型

### 字符串处理缺陷

- **简单替换不彻底**：`str.replace('  ', ' ')` 只替换一次，无法处理 3 个以上连续空格 → 应使用 `re.sub(r'\s+', ' ', text)` 正则替换
- **索引偏移**：字符串切片的起止位置差一位 → 仔细核对需要保留/隐藏的字符范围

### 格式转换缺陷

- **输出格式写错**：`strftime` 的格式字符串与预期输出不匹配 → 逐字符核对格式占位符
- **缺少绝对值**：计算差值时忘记取绝对值，输入顺序颠倒时返回负数 → 用 `abs()` 包裹

### 正则表达式缺陷

- **匹配范围不全**：正则只覆盖部分情况（如只匹配开标签不匹配闭标签）
- **量词误用**：`+`（一次以上）和 `*`（零次以上）的语义区别

## pytest 回归测试编写

### 基本结构示例

```python
import pytest
from src.text_cleaner import normalize_whitespace, mask_phone


class TestBug001Regression:
    """BUG-xxx 回归测试: normalize_whitespace 连续多空格"""

    def test_triple_spaces_collapsed(self):
        assert normalize_whitespace("hello   world") == "hello world"

    def test_multiple_groups_of_spaces(self):
        assert normalize_whitespace("a  b   c    d") == "a b c d"
```

### 命名规范

- 测试类名：`TestBugXXXRegression`
- 测试方法名：`test_<场景描述>`
- 文件名：`test_<模块名>.py` 或 `test_regression.py`

### 断言

- 使用 `assert` 语句直接断言
- 期望值直接来源于 bug 报告中的"期望结果"

## 最佳实践

1. **先读完所有 bug 报告再动手**：建立全局视角
2. **一次修一个 bug**：避免多处修改互相干扰
3. **修完即测**：每修一个 bug 就运行一次测试
4. **不改函数签名**：回归修复不应改变公开接口
5. **回归测试要精准**：直接用 bug 报告的复现数据作为测试输入
