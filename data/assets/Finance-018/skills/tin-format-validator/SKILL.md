# 各国/地区企业税务识别号码（TIN）格式验证

## 概述

对企业提供的 TIN 号码进行格式合规性验证：读取输入文件 → 根据国家字段路由到对应大洲规则文档中的该国规则 → 逐条验证 TIN 格式 → 输出判定结果。

## 规则文档路由机制

各国/地区的 TIN 格式规则存储在 skill 目录下的 `references/` 中，按大洲组织。执行验证时，**必须**按以下路由逻辑加载对应规则：

### 路由表

| 大洲 | 规则文件路径 |
|------|------------|
| Asia | `references/Asia.md` |
| Europe | `references/Europe.md` |
| North America | `references/North-America.md` |
| South America | `references/South-America.md` |
| Africa | `references/Africa.md` |
| Oceania | `references/Oceania.md` |

### 路由流程

1. 根据输入记录中的 **国家** 字段，确定该国家所属的大洲
2. 加载对应大洲的规则文件，查找该国家的 TIN 格式规则
3. 使用查找到的规则对 TIN 号码进行格式验证
4. 若同一大洲文件中包含多个国家规则，仅使用与当前记录国家匹配的规则

### 兜底机制

当出现以下情况时，按兜底策略处理：

| 场景 | 判定结果 |
|------|---------|
| 国家字段为空或无法识别 | 不符合（原因：无法识别国家） |
| 国家所属大洲的规则文件不存在 | 不符合（原因：缺少该国规则） |
| 规则文件存在但无对应国家条目 | 不符合（原因：缺少该国规则） |
| TIN 号码为空 | 不符合（原因：TIN号码为空） |
| 输入文件格式无法读取 | 尝试以常见表格格式逐一解析；均失败则报错终止 |

> **兜底原则：** 遇到任何异常情况不得中断整体流程，应继续处理其余记录，并在结果中对异常记录给出明确原因说明。

## 输入输出

**输入：** 任意格式的数据文件（`.xlsx`、`.xls`、`.csv`、`.txt`、`.json` 等表格类或结构化数据文件），每条记录至少包含：**国家**、**TIN 号码**

**输出：** 在输入数据基础上增加一列验证结果：
- 新增列名为 `是否符合格式`，取值为 **「是」** 或 **「否」**
- **是**：TIN 号码符合该国家现行有效的格式规则
- **否**：TIN 号码不符合该国家现行有效的格式规则；若因异常情况导致无法判定，括号内注明具体原因

输出文件格式、文件命名等以用户指定的要求为优先原则；若用户未指定，则根据输入文件格式合理确定。

## 执行步骤

1. 接收用户提供的数据文件
2. 读取文件内容，识别每条记录的序号、国家和 TIN 号码
3. 对每条记录，通过**路由机制**定位该国 TIN 规则并执行格式验证
4. 填入验证结果；异常记录按**兜底机制**处理
5. 按用户要求（若有）的格式和命名生成结果文件，保持与输入文件相同的序号顺序
