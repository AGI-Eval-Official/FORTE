# 发票要素合规性检查

## 功能说明

对各类发票的关键字段进行单张发票的**内部一致性检查**，判断发票是否存在格式错误、税率不合法、金额计算错误、数据缺失等问题。本工具专注于单张发票的基础合规验证，不涉及多方数据关联（如与订单、合同的匹配）。当输入仅包含部分字段时，自动进入“字段命中校验”模式：先识别可映射字段，再仅对命中字段执行对应校验，并在结果中明确提示未命中字段。

本技能采用**通用字段验证 + 发票类型差异化配置**模式：
- 通用字段验证规则在本文档定义，适用于所有发票类型
- 各发票类型的特殊规则（如税率范围、必填字段差异）在各类型配置文件中定义
- 系统根据发票类型自动加载对应的配置文件

## 规则优先级

为避免规则冲突，按以下优先级执行：

1. **用户输入的要求规则与格式规范**：包括用户指定的校验字段范围、必填口径、字段格式、输出格式、票种指定。
2. **票种配置规则**：`references/ordinary-invoice.md` 与 `references/special-invoice.md` 的差异化规则。
3. **主文档通用规则**：本文件中的通用字段校验、金额计算、日期校验等。
4. **兜底执行规则**：无法命中字段时仅标记“未执行”，不直接计为失败。

> 执行原则：优先满足用户输入的规则与格式规范；在不违反法律法规强制约束前提下完成校验。

## 发票校验项目

对发票进行以下项目的合规性检查：

| 序号 | 校验项目 | 校验内容 | 错误示例 |
|-----|---------|---------|---------|
| 1 | **字段命中识别** | 识别输入字段与标准字段的映射关系，生成命中字段/未命中字段清单 | 表头命名不规范导致字段未命中 |
| 2 | **发票号码格式** | 检查发票号码是否符合该类型发票的编码规则（字段命中时执行） | 号码位数不符、包含非法字符 |
| 3 | **纳税人识别号格式** | 检查销售方和购买方的纳税人识别号是否为15-20位字母数字（字段命中时执行） | 税号长度不足、包含非法字符 |
| 4 | **税率合法性** | 检查税率是否属于该类型发票的法定税率集合（字段命中时执行） | 使用了不支持的税率 |
| 5 | **金额税额计算一致性** | 验证税额等于金额乘税率后四舍五入保留两位小数，且价税合计等于金额加税额（相关字段均命中时执行） | 计算结果与票面金额不符 |
| 6 | **金额合理性** | 检查金额、税额、价税合计是否为正数且在合理范围内（字段命中时执行） | 金额为零或负数、超过限额 |
| 7 | **开票日期合理性** | 检查开票日期是否为有效日期且不超过基准日期（字段命中时执行） | 日期格式错误、为未来日期 |
| 8 | **专票地址电话格式** | 仅在专用发票且字段命中时，检查地址电话信息完整性 | 地址电话信息缺失或格式不完整 |
| 9 | **专票开户行及账号格式** | 仅在专用发票且字段命中时，检查开户行及账号信息完整性 | 开户行名称或银行账号缺失 |
| 10 | **数据完整性（命中范围）** | 仅对命中字段检查非空性；未命中字段不直接判失败，在结果中提示补充 | 命中字段为空、无法完成对应校验 |

## 发票类型识别与规则加载

### 支持的发票类型

系统在执行校验前先确定发票类型，然后从规则配置目录加载对应的规则配置：

| 类型标识 | 发票种类 | 配置文件 | 说明 |
|---------|---------|---------|------|
| 普通发票 | 增值税普通发票、电子普通发票、数电普票、通用机打发票等 | 规则配置目录/ordinary-invoice.md | 默认类型，不可抵扣进项税 |
| 专用发票 | 增值税专用发票、电子专用发票、数电专票等 | 规则配置目录/special-invoice.md | 用于进项税额抵扣 |

### 类型识别优先级

1. **用户显式指定**（如指定类型为专用发票）
2. **输入数据中包含类型标识列或字段**（如发票类型列值为增值税专用发票）
3. **根据数据特征自动推断**（如字段组成、关键字段名）
4. **无法确定时**：使用默认配置（普通发票配置），并在报告中提示类型未确认，按默认规则校验

### 通用默认参数

以下参数适用于所有发票类型（除非对应配置文件明确覆盖）：

| 参数 | 取值策略 | 说明 |
|------|---------|------|
| **基准日期** | 优先用户指定，否则使用当前日期 | 用于日期逻辑校验 |
| **金额上限** | 默认1000万元 | 经验阈值，可按业务调整 |
| **容差** | ±0.01元 | 算术关系校验浮点精度 |
| **税率归一化** | 6%→0.06，免税转为特殊标记 | 百分比字符串与小数自动转换 |

> **注意**：免税、星号、不征税等特殊标记应视为合法的非数值税率值。

## 通用字段验证规则

以下字段验证规则适用于所有发票类型，在主文档中统一定义：

### 1. 发票基本信息字段

| 字段名 | 校验规则 | 说明 |
|--------|---------|------|
| **发票号码** | 格式依票种而定，见各配置文件 | 发票的唯一标识 |
| **发票代码** | 10-12位数字（如有） | 部分地区或类型使用 |
| **开票日期** | 日期格式年月日，小于等于基准日期 | 不得为未来日期 |
| **发票名称** | 非空，长度2-100字符 | 发票抬头名称 |

### 2. 纳税人信息字段

| 字段名 | 校验规则 | 说明 |
|--------|---------|------|
| **销售方名称** | 非空，长度2-100字符 | 销售方企业或个人名称 |
| **销售方纳税人识别号** | 15-20位字母数字（统一社会信用代码或纳税人识别号） | 格式校验见下文 |
| **购买方名称** | 非空，长度2-100字符 | 购买方企业或个人名称 |
| **购买方纳税人识别号** | 15-20位字母数字（企业必须，个人可为空） | 企业购买方必填 |

### 纳税人识别号格式规则

统一社会信用代码或纳税人识别号校验规则：
- **长度**：15-20位
- **组成**：字母（不含大写艾欧贼艾斯维）和数字
- **统一社会信用代码**（18位）：第1位为登记管理部门代码（1-9或外），第2位为机构类别代码，第3-8位为登记管理机关行政区划码，第9-17位为主体标识码（组织机构代码），第18位为校验码
- **纳税人识别号**（15-20位）：原税务登记号，由数字和大写字母组成

### 3. 金额与税率字段

| 字段名 | 校验规则 | 说明 |
|--------|---------|------|
| **金额（不含税）** | 大于零，小于等于金额上限 | 不含税金额 |
| **税率** | 法定税率集合，见各配置文件 | 如1%、3%、6%、9%、13%等 |
| **税额** | 大于等于零，等于金额乘税率后四舍五入保留两位小数 | 容差±0.01元 |
| **价税合计** | 大于零，等于金额加税额 | 容差±0.01元 |
| **备注** | 长度≤500字符（如有） | 附加说明信息 |

### 4. 专用发票增强字段

| 字段名 | 校验规则 | 说明 |
|--------|---------|------|
| **销售方地址电话** | 专用发票场景下命中即校验完整性 | 地址与电话可同列或拆分列 |
| **销售方开户行及账号** | 专用发票场景下命中即校验完整性 | 开户行名称与账号应可识别 |
| **购买方地址电话** | 专用发票场景下命中即校验完整性 | 地址与电话可同列或拆分列 |
| **购买方开户行及账号** | 专用发票场景下命中即校验完整性 | 开户行名称与账号应可识别 |

## 校验执行顺序

按以下顺序执行各校验项目，确保依赖关系正确：

1. **规则收敛** → 先加载用户输入的要求规则与格式规范，再加载票种配置和通用规则
2. **字段命中识别** → 根据标准字段和别名映射识别可校验字段，形成命中字段清单
3. **数据完整性（命中范围）** → 仅检查命中字段是否为空；未命中字段不直接判失败
4. **发票号码格式** → 命中发票号码字段时，检查格式是否合法
5. **纳税人识别号格式** → 命中税号字段时，检查是否为15-20位字母数字（用户指定其他格式时按用户规则）
6. **税率合法性** → 命中税率字段时，检查是否在法定集合内
7. **金额合理性** → 命中金额相关字段时，检查是否为正数且在合理范围
8. **金额税额计算一致性** → 仅在金额、税率、税额、价税合计相关字段命中时执行计算一致性校验
9. **开票日期合理性** → 命中开票日期字段时，检查是否合法且不超过基准日期
10. **专票增强项** → 专用发票且字段命中时，执行地址电话与开户行及账号格式校验

**综合判定**：所有已执行校验项通过则判定为合规；任一已执行项不通过则判定为不合规，并标注首个不合规项的错误描述。因字段未命中而未执行的校验项应标记为未执行，不计入失败。

**严格规则**：
- 必须使用现行有效的法律规定和标准，不得自行假设或编造规则
- 金额计算严格四舍五入保留两位小数
- 百分比与小数的转换需自动识别并正确处理

### 异常处理总则

| 场景 | 处理策略 |
|------|---------|
| 单条记录解析完全失败 | 跳过该记录，在输出中标记为解析失败并注明原因，不中断整体流程 |
| 校验过程中某字段值非法（如金额为非数字字符串） | 该校验项记为否，错误描述中说明具体原因；不影响其他校验项执行 |
| 输入文件为空或无有效记录 | 输出空结果文件，并在汇总报告中提示输入文件无有效数据 |
| 规则参数冲突（如用户指定了不合法的基准日期） | 优先按用户输入格式规范解析；仍无法解析时回退默认参数并在结果中标注“参数回退” |
| 发票类型无法识别 | 使用默认配置执行校验，在输出中标注类型未确认，按默认规则校验 |
| 配置文件缺失或无法读取 | 报告用户，回退到通用默认参数执行基础校验 |
| 部分字段无法映射 | 记录命中字段与未命中字段；仅对命中字段执行对应校验，未命中字段不计入失败并在结果中提示补充 |

## 输入要求

### 支持的输入格式

系统根据文件扩展名和内容特征**自动识别格式**并选择解析策略：

| 类别 | 格式 | 扩展名 | 解析方式 | 优先级 |
|------|------|--------|---------|--------|
| 表格类 | Excel、CSV、TSV | 表格格式扩展名 | 结构化读取 | 高 |
| 结构化数据类 | JSON | 点JSON | 字段映射解析 | 中 |

> 高优先级为推荐格式（无损）

### 数据预处理

- **空白值处理**：纯空白字符串视为空值
- **数值标准化**：去除千分位逗号、货币符号（人民币符号、美元符号）、前后空格
- **税率归一化**：6%或0.06统一转为0.06；免税或星号转为特殊标记
- **日期标准化**：统一转为年月日格式比较
- **纳税人识别号清理**：去除空格、横杠等分隔符，统一转为大写

### 兜底机制

| 异常情况 | 兜底策略 |
|---------|---------|
| 多工作表 Excel | 默认读第一个工作表；可指定工作表编号或全部合并 |
| 合并单元格 | 表头合并格向下填充；数据区纵向或横向合并格取相同值或首子格值 |
| 多级表头 | 自动合并单级（如金额加元转为金额） |
| 前若干行空行 | 自动跳过，从首个非空行开始 |
| 中间空行 | 保留记录，标记数据不完整 |
| 列顺序不同 | 通过列名映射适配，不依赖固定列序 |
| 仅保留部分字段 | 自动识别命中字段并执行局部校验；未提供字段标记为未命中，不直接判失败 |
| 额外无关列 | 忽略无法映射到标准字段的列，不影响校验 |
| 汇总或合计行 | 检测合计、总计、求和关键字自动排除 |
| 缺少序号 | 按行号自动生成；其余未命中字段标记为未执行，不直接判失败 |
| 编码检测失败 | 依次尝试 UTF-8-BOM、UTF-8、GBK、GB18030、Latin-1 |

## 输出要求

输出格式不做严格规定，由执行者根据实际情况灵活选择。核心要求如下：

- **优先遵循用户要求**：如用户指定了输出格式、输出字段、排序方式等要求，必须优先按照用户要求执行；用户未指定时，按以下默认格式执行
- **默认格式-必须包含的内容**：每条原始记录的校验结果，至少涵盖字段命中情况（命中字段、未命中字段）、各校验项目的判定结果（发票号码合规、纳税人识别号合规、税率合规、金额税额一致性、金额合理性、开票日期合理性、专票增强项、数据完整性），以及一个综合判定结论
- **默认格式-综合判定规则**：所有已执行校验项通过则判定为合规；任一已执行项不通过则判定为不合规；未命中字段对应项标记为未执行，不计入失败
- **默认格式-建议形式**：在输入数据基础上追加校验结果列，保持与输入相同的记录顺序；建议额外输出“未执行校验项及原因（字段未命中）”；也可仅输出不合规记录及原因汇总
- **默认格式-可选增强**：若输出为表格文件，可对不合规单元格标红高亮
- **动态列调整**：校验列会根据实际加载的票种配置动态增减
