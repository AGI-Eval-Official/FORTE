# 发票与合同订单匹配验证

## 功能说明

根据企业的合同、订单和发票数据，对发票进行多维度的匹配验证，识别异常发票、防止重复支付、确保财务合规性。

## 校验规则

支持用户自定义校验规则，可指定启用的校验维度及各维度的判定条件；未指定时使用默认规则。

### 默认规则

默认规则涵盖 14 个校验维度、共 56 条规则，详见 reference/default-rules.md。

| 维度 | 编号 | 核心校验点 |
|------|------|------------|
| 供应商匹配 | R1 | 供应商名称、纳税人识别号、地址一致性 |
| 商品/服务匹配 | R2 | 商品范围、分类、规格型号、单位 |
| 订单关联 | R3 | 订单有效性、重复发票、跨订单关联 |
| 金额合规 | R4 | 金额上限、累计超额、含税/不含税/税额三角校验 |
| 数量匹配 | R5 | 数量上限、累计超量、单价×数量=金额校验 |
| 时间合规 | R6 | 合同有效期、订单日期、未来日期、到期预警 |
| 税率一致性 | R7 | 税率一致、法定税率、免税校验 |
| 发票类型合规 | R8 | 发票类型合法性、专票/普票一致性、红字发票 |
| 合同状态校验 | R9 | 合同生效状态、终止合同新发票 |
| 付款条件匹配 | R10 | 付款节点、预付款比例、验收进度 |
| 币种与汇率校验 | R11 | 币种一致、汇率合理性、折算准确性 |
| 折扣与折让合规 | R12 | 折扣范围、折扣率、折扣后金额 |
| 重复发票检测 | R13 | 发票号码重复、相同供应商+金额+日期 |
| 备注与附注校验 | R14 | 合同编号、项目名称、关键备注信息 |

> 当输入数据缺少某规则所需字段时，该规则自动跳过，不影响综合判定。详见 reference/default-rules.md 的「规则跳过机制」。

### 自定义规则

1. **启用/禁用维度**：指定仅启用部分校验维度
2. **调整判定条件**：对某维度的判定条件进行调整（如金额允许偏差比例、时间允许宽限期等）
3. **新增维度**：提出默认规则以外的校验需求，工具根据数据可行性支持

> **规则优先级**：自定义规则优先于默认规则。仅指定部分维度时，未提及的维度默认不校验；若需其余维度按默认执行，须显式声明。

## 输入要求

### 文件格式

支持 `.xlsx`、`.xlsm`、`.csv`、`.tsv` 格式，文件数量不限。数据可分布在多个文件中，也可合并在同一文件的不同工作表中。

### 数据类别

所需数据类别取决于启用的校验维度：

- **合同数据**：合同相关字段（按需提供）
- **订单数据**：订单相关字段（按需提供）
- **发票数据**：发票相关字段（始终必需）

各类数据间通过关联字段建立关系。关联字段确定优先级：

1. **用户指定**：用户指定字段间的关联关系
2. **自动识别**：未指定时，工具根据字段名称及数据内容推断

某类数据在所有启用维度中均无字段需求时，可不提供。

### 必需字段

所需字段取决于启用的校验维度：

| 校验维度 | 合同数据需含字段 | 订单数据需含字段 | 发票数据需含字段 |
|---|---|---|---|
| 供应商匹配(R1) | 合同标识、供应商、纳税人识别号（R1.3）、地址/联系方式（R1.4） | — | 发票标识、供应商、合同关联字段、纳税人识别号（R1.3）、地址/联系方式（R1.4） |
| 商品匹配(R2) | 合同标识、商品/服务、商品分类（R2.2）、规格型号（R2.3）、单位（R2.4） | — | 发票标识、商品/服务、合同关联字段、规格型号（R2.3）、单位（R2.4） |
| 订单关联(R3) | 合同标识（R3.4） | 订单标识、合同关联字段（R3.4） | 发票标识、订单关联字段 |
| 金额合规(R4) | 合同标识、合同金额 | 订单标识、合同关联字段、订单金额 | 发票标识、发票金额、订单关联字段、不含税金额（R4.6/R4.7）、含税金额（R4.6）、税额（R4.7） |
| 数量匹配(R5) | 合同标识、合同数量（R5.3）、合同单价（R5.7） | 订单标识、订单数量 | 发票标识、发票数量、订单关联字段、发票单价（R5.6/R5.7） |
| 时间合规(R6) | 合同标识、生效日期、失效日期 | 订单标识、合同关联字段、订单日期 | 发票标识、发票日期、订单关联字段 |
| 税率一致性(R7) | 合同标识、税率 | — | 发票标识、税率、合同关联字段 |
| 发票类型合规(R8) | 合同标识、纳税人类型（R8.2） | — | 发票标识、发票类型、合同关联字段 |
| 合同状态校验(R9) | 合同标识、合同状态（R9.1）、合同终止日期（R9.2） | — | 发票标识、合同关联字段 |
| 付款条件匹配(R10) | 合同标识、付款条件/节点、预付款比例（R10.3） | — | 发票标识、发票金额、合同关联字段、验收进度（R10.2） |
| 币种与汇率校验(R11) | 合同标识、币种 | — | 发票标识、币种、合同关联字段、汇率（R11.2/R11.3/R11.4）、本位币金额（R11.3） |
| 折扣与折让合规(R12) | 合同标识、折扣约定、折扣率（R12.3） | — | 发票标识、折扣金额（R12.1/R12.2）、合同关联字段 |
| 重复发票检测(R13) | — | — | 发票标识、发票号码、供应商、发票金额、发票日期 |
| 备注与附注校验(R14) | 合同标识、备注要求 | — | 发票标识、备注、合同关联字段 |

> 表中"—"表示该类别非该维度的必需项。**合同标识**、**订单标识**、**发票标识**分别为各类数据的唯一标识字段；**合同关联字段**、**订单关联字段**为跨类别关联字段，具体对应实际字段时用户指定优先、未指定时自动识别。

**通用约束：**
- 默认规则下需满足全部维度的字段要求
- 自定义规则下仅需满足所启用维度的字段要求
- 自定义维度所需字段由工具与用户确认
- 日期格式：`YYYY-MM-DD`；金额、数量须为数值类型
- 发票数据始终需含**序号**字段（用于结果排序）

## 输出格式

如果用户对输出有格式和内容要求，严格按照用户要求输出，如没有指定，则按照默认模式输出xlsx文件（`.xlsx`格式）。

### 表头结构

输出表头 = 固定列 + 校验列（动态） + 综合判定列：

**固定列**：`序号,发票号码,供应商,商品/服务`

**校验列**：按启用的校验维度逐一生成，每维度一列。默认规则下为：`供应商匹配,商品匹配,订单关联,金额合规,数量匹配,时间合规,税率一致性,发票类型合规,合同状态校验,付款条件匹配,币种与汇率校验,折扣与折让合规,重复发票检测,备注与附注校验`

**综合判定列**：`综合判定`

**取值说明：**
- 各校验列：`是` / `否` / `跳过`（该维度下所有规则均被跳过时标记为"跳过"；部分规则跳过时仅基于已执行规则判定）；自定义维度可按用户指定取值
- **综合判定**：`匹配` / `匹配（预警：原因）` / `不匹配（原因）` / `不匹配（原因；预警：预警原因）`。原因格式为：维度名称+规则编号（如 `金额超额R4.1`），多项原因以分号分隔。预警规则不作为"否"判定，详见 reference/default-rules.md 的「预警规则」；另有前置校验（如 R6.5）不按逐张发票判定，详见 reference/default-rules.md 的「前置校验」

## 执行步骤

1. 接收用户提供的文件
2. 确定校验规则：用户自定义优先，未指定则使用默认规则
3. 根据校验规则确定所需字段，校验输入数据完整性
4. 识别并提取合同、订单、发票数据，建立关联关系
5. 按启用的维度逐张发票执行匹配验证
6. 填入校验结果，给出综合判定
7. 生成结果文件（保持原序号顺序）
8. 列出不匹配记录及异常原因
