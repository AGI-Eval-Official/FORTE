# 发票与合同订单匹配校验 — 默认规则

> 本文档为 `invoice-contract-matcher` 技能的默认校验规则库。用户未指定自定义规则时，按本规则执行。
> 规则编号格式：`R{维度编号}.{规则序号}`，便于在结果中精确引用。
> 本文档中"金额"如未特别注明含税/不含税，均指发票票面金额字段（即用户数据中"发票金额"字段所对应的金额）。

---

## 规则跳过机制

以下情况下，相应规则自动跳过（该规则不参与维度判定，也不计入综合判定）：

1. **数据字段缺失**：规则所需字段在输入数据中不存在时，该规则跳过
2. **红字发票**：金额/数量非正规则（R4.8、R4.9、R5.4、R5.5、R12.2）对红字发票跳过
3. **用户声明**：用户在自定义规则中明确排除的规则跳过
4. **条件未满足**：规则的前置条件未满足时跳过（如 R10.2 需有验收进度字段、R11.2 对本位币发票跳过、R13.1 对同号码不同金额跳过等）
5. **业务不适用**：规则不适用的业务场景（如合同未约定折扣时 R12.1/R12.3 跳过、合同未约定总数量时 R5.3 跳过）

> 跳过的规则在输出结果中标注为"跳过"，与"是"/"否"区分。

### 部分跳过时的维度判定

当维度下部分规则跳过时，维度判定仅基于已执行规则的结果：
- 所有已执行规则均为"是" → 维度判定为"匹配"
- 任一已执行规则为"否" → 维度判定为"不匹配"
- 所有规则均跳过 → 维度判定为"跳过"

## 预警规则

以下规则为预警规则，其判定结果不作为"否"计入综合判定，而是在综合判定原因中以"预警"标注：

| 编号 | 规则 | 预警含义 |
|------|------|----------|
| R4.10 | 合同剩余金额 < 0 | 累计开票已超合同金额 |
| R6.6 | 发票日期距合同失效日期 < 30天 | 即将到期合同的开票需关注 |
| R13.2 | 同一供应商 + 同一金额 + 同一日期 | 可能的重复开票 |

> 预警规则触发时，综合判定格式为：`匹配（预警：原因）` 或 `不匹配（原因；预警：预警原因）`。

## 前置校验

以下规则为前置校验，在逐张发票校验之前执行，不按逐张发票判定，仅校验一次：

| 编号 | 规则 | 不通过时的影响 |
|------|------|----------------|
| R6.5 | 合同生效日期 ≤ 合同失效日期 | 该合同下所有发票的时间合规维度均标记为"否" |

## 综合判定规则

### 维度级判定

- **匹配**：该维度下所有已执行的规则结果均为"是"（预警规则除外）
- **不匹配**：任一已执行规则（预警规则除外）结果为"否"，标注具体原因（引用规则编号）
- **跳过**：该维度下所有规则均被跳过

### 发票级综合判定

- `匹配`：全部维度均为"匹配"或"跳过"，且无预警
- `匹配（预警：原因）`：全部维度均为"匹配"或"跳过"，但存在预警规则触发
- `不匹配（原因）`：任一维度为"不匹配"，原因格式为：维度名称 + 规则编号，多项以分号分隔
- `不匹配（原因；预警：预警原因）`：不匹配且存在预警规则触发

### 综合判定原因示例

- `匹配（预警：合同剩余金额为负R4.10）`
- `不匹配（金额超额R4.1；时间超期R6.2）`
- `不匹配（供应商不符R1.1；预警：重复发票可疑R13.2）`

## 校验规则详情

### 一、供应商匹配（R1）

校验发票供应商与合同供应商的一致性，防止虚假开票和供应商冒名。

| 编号 | 规则 | 说明 |
|------|------|------|
| R1.1 | 发票供应商名称 = 合同供应商名称（精确匹配） | 全称一致，不可用简称替代 |
| R1.2 | 供应商名称变更须有合同变更文件 | 如供应商因更名、合并、分立等原因名称变化，需提供合同变更协议；当数据中无变更标识字段时跳过 |
| R1.3 | 发票供应商纳税人识别号 = 合同供应商纳税人识别号 | 名称一致但税号不同视为不匹配，防止同名不同主体；当数据中无纳税人识别号字段时跳过 |
| R1.4 | 发票供应商地址/联系方式应与合同供应商一致 | 当数据中包含地址/联系方式字段时校验；不含则跳过 |
| R1.5 | 同一合同下不应出现多个不同供应商的发票 | 防止发票错配合同 |

---

### 二、商品/服务匹配（R2）

校验发票商品/服务是否在合同约定范围内。

| 编号 | 规则 | 说明 |
|------|------|------|
| R2.1 | 发票商品/服务 ⊆ 合同约定商品/服务 | 发票中每种商品/服务须在合同约定范围内；当合同商品为文本描述时，按语义包含关系判定 |
| R2.2 | 发票商品分类应与合同商品分类一致 | 当合同按大类约定时，发票商品须归属该大类；当数据中无分类字段时跳过 |
| R2.3 | 发票商品规格型号应与合同一致 | 当数据中包含规格型号字段时校验；不含则跳过 |
| R2.4 | 发票商品单位应与合同约定单位一致 | 如合同约定"台"，发票不应为"套"；不含则跳过 |
| R2.5 | 发票不应包含合同未约定的商品/服务 | 单项校验，任何超出合同范围的商品均标记异常；与 R2.1 为双向校验（R2.1 校验发票商品是否在合同范围内，R2.5 单独标注越界商品明细） |

---

### 三、订单关联（R3）

校验发票与订单的关联完整性和唯一性。

| 编号 | 规则 | 说明 |
|------|------|------|
| R3.1 | 每张发票须关联至有效订单 | 发票必须有对应的订单，无关联订单的发票视为异常 |
| R3.2 | 同一订单不允许多张相同金额的发票 | 防止重复开票/重复支付（不含红字冲红场景） |
| R3.3 | 同一订单不允许多张相同发票号码的发票 | 防止重复录入 |
| R3.4 | 发票关联的订单须属于对应合同 | 订单的合同归属应与发票的合同归属一致；需合同数据中包含合同标识、订单数据中包含合同关联字段方可校验 |
| R3.5 | 同一发票不应关联多个不同订单 | 一张发票原则上对应一个订单；拆票场景除外（需用户声明） |

---

### 四、金额合规（R4）

校验发票金额与合同、订单金额的关系，防止超额支付。

| 编号 | 规则 | 说明 |
|------|------|------|
| R4.1 | 发票金额 ≤ 订单金额 | 单张发票金额不应超过其关联订单的金额 |
| R4.2 | 发票金额 ≤ 合同金额 | 单张发票金额不应超过合同总金额 |
| R4.3 | 同一合同下累计发票金额 ≤ 合同金额 | 防止超额开票（事后校验：统计已入账的全部发票） |
| R4.4 | 同一订单下累计发票金额 ≤ 订单金额 | 防止超额开票 |
| R4.5 | 该发票金额 + 同合同下已开票金额 ≤ 合同金额 | 增量式校验：当前发票入账后累计不超额（与 R4.3 互补，R4.3 校验历史累计，R4.5 校验加入当前发票后的累计） |
| R4.6 | 发票不含税金额 × (1 + 税率) = 发票含税金额 | 金额内部一致性校验，允许尾差（≤0.01元）；当数据中无不含税金额或含税金额字段时跳过 |
| R4.7 | 发票含税金额 - 发票税额 = 发票不含税金额 | 税额与含税/不含税金额的三角校验，允许尾差（≤0.01元）；当数据中无含税金额、税额或不含税金额字段时跳过 |
| R4.8 | 发票金额不应为负数 | 红字发票跳过此规则 |
| R4.9 | 发票金额不应为零 | 零金额发票视为异常；红字发票跳过此规则 |
| R4.10 | 合同剩余金额 < 0 时标记预警 | 合同剩余金额 = 合同金额 - 同合同下累计发票金额；此为预警规则，不作为"否"判定，仅在综合判定原因中标注"预警" |

---

### 五、数量匹配（R5）

校验发票数量与订单/合同数量的关系。

| 编号 | 规则 | 说明 |
|------|------|------|
| R5.1 | 发票数量 ≤ 订单数量 | 单张发票数量不应超过其关联订单的数量 |
| R5.2 | 同一订单下累计发票数量 ≤ 订单总数量 | 防止超量开票 |
| R5.3 | 同一合同下累计发票数量 ≤ 合同总数量 | 当合同约定总数量时校验；未约定则跳过 |
| R5.4 | 发票数量不应为负数 | 红字发票跳过此规则 |
| R5.5 | 发票数量不应为零 | 零数量发票视为异常；红字发票跳过此规则 |
| R5.6 | 发票单价 × 数量 = 发票金额（不含税） | 单价、数量、金额三者一致性校验，允许尾差（≤0.01元）；当数据中无发票单价字段时跳过 |
| R5.7 | 发票单价应与合同约定单价一致 | 当合同约定单价时校验；允许尾差（≤0.01元）；合同未约定单价或数据中无发票单价字段时跳过 |

---

### 六、时间合规（R6）

校验发票日期与合同、订单时间的关系。

| 编号 | 规则 | 说明 |
|------|------|------|
| R6.1 | 发票日期 ≥ 合同生效日期 | 合同未生效期间的开票视为异常 |
| R6.2 | 发票日期 ≤ 合同失效日期 | 合同失效后的开票视为异常 |
| R6.3 | 发票日期 ≥ 订单日期 | 订单日期之前的开票视为异常 |
| R6.4 | 发票日期不应晚于当前日期 | 未来日期的发票视为异常；当前日期以校验执行时的系统日期为准 |
| R6.5 | 合同生效日期 ≤ 合同失效日期 | 合同日期自身合理性校验（前置校验，不按逐张发票判定，仅校验一次）；若不通过，该合同下所有发票的时间合规维度均标记为"否" |
| R6.6 | 发票日期距合同失效日期 < 30天时标记预警 | 即将到期合同的开票需关注；此为预警规则，不作为"否"判定，仅在综合判定原因中标注"预警" |
| R6.7 | 同一订单下多张发票日期不应倒序 | 后开票日期早于先开票日期可能存在异常（按发票号码升序排列后检查日期是否单调不减） |

---

### 七、税率一致性（R7）

校验发票税率与合同税率的一致性。

| 编号 | 规则 | 说明 |
|------|------|------|
| R7.1 | 发票税率 = 合同税率 | 税率必须完全一致 |
| R7.2 | 发票税率应为法定有效税率 | 检查税率是否为增值税法定税率（0%、1%、3%、5%、6%、9%、13%；其中1%为小规模纳税人减按征收率）；不在上述范围内的税率视为异常 |
| R7.3 | 同一合同下所有发票税率应一致 | 如合同约定统一税率，发票间税率不应不同；当合同允许混合税率（如不同商品不同税率）时不适用 |
| R7.4 | 免税商品不应有非零税率 | 合同约定免税时，发票税率应为0%或免税标识 |

---

### 八、发票类型合规（R8）

校验发票类型本身的合法性和适用性。

| 编号 | 规则 | 说明 |
|------|------|------|
| R8.1 | 发票类型应为合法类型 | 合法类型包括：增值税专用发票、增值税普通发票、增值税电子专用发票、增值税电子普通发票、机动车销售统一发票、二手车销售统一发票、通行费发票等；不在上述范围内的类型标记异常 |
| R8.2 | 一般纳税人应取得增值税专用发票 | 合同约定一般纳税人时，发票类型应为专票或电子专票；当数据中无纳税人类型字段时跳过 |
| R8.3 | 红字发票须有对应的蓝字发票 | 红字发票（冲红）应能追溯到原蓝字发票（按发票号码关联或原发票号码字段匹配）；当数据中无法识别红蓝字关联关系时跳过 |
| R8.4 | 同一笔业务不应既有专票又有普票 | 同一订单/合同的发票类型应统一；当数据中无法按订单/合同归集时跳过 |

---

### 九、合同状态校验（R9）

校验发票对应的合同是否处于有效状态。

| 编号 | 规则 | 说明 |
|------|------|------|
| R9.1 | 合同状态应为"生效" | 非生效状态（如草稿、已终止、已作废）的合同不应有发票；当数据中无合同状态字段时跳过 |
| R9.2 | 已终止合同不应有新发票 | 合同终止日期之后不应产生新发票；当数据中无合同终止日期字段时跳过 |
| R9.3 | 合同不应存在审批未通过的开票 | 当数据中包含审批状态字段时校验；不含则跳过 |

---

### 十、付款条件匹配（R10）

校验发票金额与合同付款条件的匹配关系。

| 编号 | 规则 | 说明 |
|------|------|------|
| R10.1 | 发票金额应与合同付款节点金额匹配 | 如合同约定分期付款，发票金额应与某期付款金额对应；当合同无付款节点字段或付款方式为一次性付款时跳过 |
| R10.2 | 付款进度不应超过项目验收进度 | 当数据中包含验收进度字段时校验；不含则跳过 |
| R10.3 | 预付款发票金额不应超过合同预付款比例 | 当合同约定预付款比例时校验；未约定则跳过 |

---

### 十一、币种与汇率校验（R11）

校验多币种场景下的金额准确性。

| 编号 | 规则 | 说明 |
|------|------|------|
| R11.1 | 发票币种 = 合同币种 | 币种必须一致；当合同允许外币结算且约定了结算币种时，以约定为准 |
| R11.2 | 外币发票应有汇率 | 当发票币种非本位币时，须有汇率字段；本位币发票跳过此规则 |
| R11.3 | 外币折算本位币金额应与汇率一致 | 外币金额 × 汇率 = 本位币金额，允许尾差（≤0.01元）；当数据中无本位币金额字段时跳过 |
| R11.4 | 汇率应在合理范围内 | 不应偏离同期市场汇率超过5%；当无法获取同期市场汇率参考时跳过 |

---

### 十二、折扣与折让合规（R12）

校验发票中的折扣、折让信息。

| 编号 | 规则 | 说明 |
|------|------|------|
| R12.1 | 发票折扣应在合同约定范围内 | 合同未约定折扣时，发票不应有折扣；当数据中无折扣金额字段时跳过 |
| R12.2 | 折扣后的发票金额不应为负 | 折扣金额不应超过商品金额；红字发票跳过此规则 |
| R12.3 | 折扣率应与合同约定一致 | 当合同约定固定折扣率时校验；合同未约定折扣率或数据中无折扣率字段时跳过 |

---

### 十三、重复发票检测（R13）

检测可能重复录入或重复开票的情况。

| 编号 | 规则 | 说明 |
|------|------|------|
| R13.1 | 同一发票号码不应出现多次 | 防止重复录入；号码相同且金额相同视为重复，号码相同但金额不同（可能为红字冲红）跳过此规则 |
| R13.2 | 同一供应商 + 同一金额 + 同一日期 的发票应标记可疑 | 可能的重复开票；此为预警规则，不作为"否"判定，仅在综合判定原因中标注"预警" |
| R13.3 | 同一发票号码 + 不同供应商 应标记异常 | 同一发票号码不应归属不同供应商 |

---

### 十四、备注与附注校验（R14）

校验发票备注信息与合同约定是否一致。

| 编号 | 规则 | 说明 |
|------|------|------|
| R14.1 | 发票备注应包含合同编号 | 当合同约定发票须注明合同编号时校验；未约定则跳过 |
| R14.2 | 发票备注应包含项目名称 | 当合同约定发票须注明项目名称时校验；未约定则跳过 |
| R14.3 | 发票备注信息应与合同约定一致 | 备注中的关键信息（如项目编号、阶段等）应与合同一致；当数据中无备注字段或合同无备注要求时跳过 |
