# 数据清洗规则

> 本文件定义 finance_data_analyzer SKILL 在数据探查与预处理阶段必须遵守的规则。
> **无论用户是否明确要求清洗，数据探查阶段必须强制执行；清洗操作是否执行取决于探查结论。**
> 本文件中涉及输出结构与格式的描述均为默认示例；若用户明确指定输出格式（字段顺序、命名、样式），必须以用户要求为准。

---

## 一、支持的输入格式

| 格式 | 读取方式 | 备注 |
|------|----------|------|
| `.xlsx` / `.xls` | `pandas.read_excel()` | 多 Sheet 时默认读取第一个 Sheet，可通过参数 `sheet_name` 指定 |
| `.csv` | `pandas.read_csv()` | 自动检测编码（优先 UTF-8，fallback GBK） |
| `.tsv` | `pandas.read_csv(sep='\t')` | — |
| `.json` | `pandas.read_json()` | 仅支持 records / columns 两种 orient |

> 若文件格式不在上表中，终止执行并提示用户转换格式后重试。

---

## 二、数据探查（强制执行，不可跳过）

无论用户是否要求清洗，**必须先完成以下探查**，并将探查结论写入 `probe_report`：

### 2.1 基础结构探查

```
探查项：
├── 行数、列数
├── 列名列表及数据类型
├── 是否存在完全重复行（duplicate rows）
└── 是否存在全空列或全空行
```

### 2.2 关键字段识别

尝试自动识别以下字段类型，识别失败时记录到 `probe_report.unresolved_fields`：

| 字段类型 | 识别规则 |
|----------|----------|
| 科目/项目列 | 列名含"项目"、"科目"、"名称"、"类别"、"project"、"category" |
| 时间列 | 列名含"日期"、"月"、"年"、"date"、"month"、"year"，或值可解析为日期 |
| 金额列 | 数值类型列，列名含"金额"、"收入"、"支出"、"成本"、"费用"、"amount"、"value" |
| 汇总列 | 列名含"合计"、"总计"、"total"、"sum" |

### 2.3 数据质量探查

```
探查项：
├── 各列空值数量及占比
├── 金额列中的负值数量及所在行
├── 金额列中的异常值（超过均值 ± 3σ 的值）
├── 金额列格式混用情况（千分位、货币符号、括号负数、单位后缀）
├── 时间列的覆盖范围（最早/最晚日期，是否有断档）
└── 科目列的唯一值列表（用于后续会计分类）
```

### 2.4 探查结论输出格式（默认示例）

> 以下结构仅为默认示例；若用户明确指定 `probe_report` 输出格式，按用户要求输出，但不得省略必要探查结论。

```json
{
  "probe_report": {
    "shape": {"rows": 11, "cols": 14},
    "identified_fields": {
      "project_col": "项目",
      "time_cols": ["1月", "2月", "..."],
      "amount_cols": ["1月", "2月", "..."],
      "total_col": "合计"
    },
    "quality_issues": [
      {"type": "null_value", "field": "10月", "count": 1, "action": "fill_zero"},
      {"type": "negative_value", "field": "投资收益", "month": "2月", "value": -500},
      {"type": "amount_format_mixed", "field": "3月", "examples": ["￥1,234.50", "(2,500)", "3.2万"]}
    ],
    "unresolved_fields": [],
    "cleaning_required": true,
    "cleaning_actions": ["fill_null", "type_cast"]
  }
}
```

## 三、清洗决策路由

根据探查结论，按以下规则决定是否执行清洗：

```
if cleaning_required == false:
    → 跳过清洗，直接进入下一步
    → 在执行日志中记录"数据质量良好，无需清洗"

if cleaning_required == true AND 用户参数 auto_clean == true（默认）:
    → 自动执行清洗，记录所有操作到 notes
    → 清洗完成后继续执行后续任务

if cleaning_required == true AND 用户参数 auto_clean == false:
    → 暂停执行，向用户展示 probe_report 中的质量问题
    → 等待用户确认后再执行清洗
```

## 四、清洗操作规则

### 4.1 空值处理

| 字段类型 | 默认处理方式 | 用户可覆盖参数 |
|----------|-------------|--------------|
| 金额列 | 填充为 `0` | `null_fill_value`（如填 `NaN` 保留） |
| 科目列 | 终止并提示用户，科目名不可为空 | — |
| 时间列 | 标记为缺失期，不填充 | — |
| 汇总列 | 忽略（汇总列不参与计算） | — |

### 4.2 数据类型转换

- 金额列：先执行会计金额格式统一，再按货币精度处理（优先 `Decimal`，或等价地先转为"分"的 `int64`）
- 会计金额格式统一规则：去除首尾空格与全角空格、全角数字转半角、去除货币符号（`￥`/`¥`/`RMB`/`CNY`）、去除千分位分隔符、识别括号负数（如`(1,234.56)`→`-1234.56`）、识别并剥离末尾单位（如`万`/`万元`/`亿`）
- 金额列转换失败的单元格记录到 `notes` 并按 `Decimal('0')` 处理
- 时间列（宽表月份列名）：保持字符串，不做转换
- 科目列：保持字符串，去除首尾空格

### 4.3 重复行处理

- 完全重复行：默认保留第一行，删除后续重复行，记录到 `notes`
- 科目名重复但数值不同：**不自动合并**，在 `notes` 中标注"发现同名科目，请人工确认是否合并"，并继续执行（两行均参与计算）

### 4.4 异常值处理

- 负值：**保留原值**，不修改，在 `notes` 中标注"[科目名] [月份] 出现负值，请核查"
- 超过 3σ 的异常值：**保留原值**，在 `notes` 中标注"[科目名] [月份] 数值异常（超过均值±3σ），请核查"
- 不自动修改任何异常值，所有修改必须由用户明确指示

### 4.5 单位统一

- 清洗后金额统一为**元**，作为唯一计算基准
- 若单元格中已包含明确单位（如`万`/`万元`/`亿`），优先按单元格单位换算为元
- 若单元格未包含单位，则按参数 `amount_unit`（默认"元"）解释并换算
- 若发现单元格单位与 `amount_unit` 冲突，以单元格单位为准并在 `notes` 中记录
- 转换后计算过程始终使用元，展示时可按量级换算

## 五、清洗操作记录规范（默认示例）

所有清洗操作必须写入 `notes`。以下格式为默认示例；若用户指定 `notes` 输出格式（字段顺序、命名、样式），按用户要求输出：

```
[数据清洗记录]
- 空值填充：{科目名} {月份} 数据缺失，已按 {fill_value} 处理
- 金额格式统一：{列名} "{原值}" 已规范为 "{标准值}"（元）
- 类型转换：{列名} 列存在非数值内容 "{原值}"，已按 Decimal('0') 处理
- 重复行删除：第 {行号} 行与第 {行号} 行完全重复，已删除后者
- 异常值标注：{科目名} {月份} 值为 {值}，超过均值±3σ，已保留原值
```

---

## 六、用户可配置参数汇总

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `auto_clean` | bool | `true` | 是否自动执行清洗（false 时暂停等待确认） |
| `null_fill_value` | float | `0` | 金额列空值填充值 |
| `amount_unit` | str | `"元"` | 输入数据的金额单位，支持"元"/"万元"/"百万元" |
| `sheet_name` | str/int | `0`（第一个Sheet） | 多 Sheet 文件时指定读取的 Sheet |
| `encoding` | str | `"auto"` | CSV 文件编码，auto 时自动检测 |
