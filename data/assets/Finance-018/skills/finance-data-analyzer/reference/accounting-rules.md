# 会计计算规则

> 本文件定义 finance_data_analyzer SKILL 在执行财务指标聚合时必须遵守的字段分类规则与计算公式。
> 所有计算必须以清洗后的源数据为唯一依据，禁止虚构或估算任何数值。
> 本文件中涉及输出展示的格式要求均为默认示例；若用户明确指定输出格式（单位、小数位、字段顺序、命名或样式），必须以用户要求为准。

---

## 一、数据表结构约定

本 SKILL 支持两种数据表格式，由数据探查阶段自动识别：

### 宽表格式（Wide Format）— 推荐

```
行：每行一个财务科目
列：项目列 + 各月份金额列（1月~12月）+ 合计列（可选）

示例：
项目       | 1月    | 2月    | ... | 12月   | 合计
主营收入   | 8856   | 9938   | ... | 6666   | 97759
主营成本   | 2536   | 3364   | ... | 4333   | 41642
```

### 长表格式（Long Format）— 兼容

```
列：科目列 + 时间列（日期/月份/年份）+ 金额列 + 收支类型列（可选）

示例：
项目     | 月份  | 金额   | 类型
主营收入 | 2025-01 | 8856 | 收入
主营成本 | 2025-01 | 2536 | 支出
```

> 格式识别规则：若存在多个数值列且列名为月份格式 → 宽表；若存在独立时间列 → 长表。识别失败时在 `notes` 中提示用户确认格式。

---

## 二、财务科目分类规则

### 2.1 收入科目（income）

以下科目归入**收入**类别，参与收入侧聚合计算：

| 科目名称 | 说明 |
|----------|------|
| 主营收入 | 核心业务产生的营业收入 |
| 投资收益 | 对外投资取得的收益 |
| 补贴收入 | 政府补贴或平台补贴收入 |
| 营业外收入 | 非主营业务产生的其他收入 |

### 2.2 支出科目（expense）

以下科目归入**支出**类别，参与支出侧聚合计算：

| 科目名称 | 说明 |
|----------|------|
| 主营成本 | 直接与主营业务相关的成本 |
| 主营税金及附加 | 与主营业务相关的税费 |
| 营业费用 | 销售与市场推广费用 |
| 管理费用 | 行政管理相关费用 |
| 财务费用 | 融资、利息等财务相关费用 |
| 营业外支出 | 非主营业务产生的其他支出 |
| 所得税 | 企业所得税 |

### 2.3 科目扩展识别规则

若源数据中出现上表以外的科目名称，按以下规则自动归类：

```
名称含"收入"、"收益"、"补贴"、"income"、"revenue"  → 归入收入
名称含"成本"、"费用"、"税"、"支出"、"cost"、"expense"、"tax"  → 归入支出
无法判断  → 在 notes 中记录"[科目名] 无法自动归类，已跳过，请人工确认"
```

> 用户可通过参数 `custom_income_items` 和 `custom_expense_items` 传入自定义科目列表，优先级高于自动识别规则。

## 三、时间维度划分规则

### 3.1 季度划分

| 季度 | 月份范围 |
|------|----------|
| Q1 | 1月、2月、3月 |
| Q2 | 4月、5月、6月 |
| Q3 | 7月、8月、9月 |
| Q4 | 10月、11月、12月 |

### 3.2 数据期不完整时的处理

- 若数据仅覆盖部分月份（如 1月~9月），仍可执行计算，但：
  - 缺失月份的金额视为 0 参与汇总
  - 在所有输出（图表标题、PPT 页面）中注明"数据期：{实际起止月份}"
  - 不得将不完整季度的数据与完整季度直接对比

## 四、核心指标计算公式

### 4.0 任务联动关系（与 SKILL 同步）

- T3：收入结构占比
- T4：支出结构占比
- T5：净利润分析
- 当识别到 T5 时，必须联动执行 T3 + T4（或复用同周期聚合结果），以保证净利润口径与结构口径一致

### 4.1 月度汇总

```
月总收入(m) = Σ 所有收入科目在第 m 月的金额
月总支出(m) = Σ 所有支出科目在第 m 月的金额
月净利润(m) = 月总收入(m) - 月总支出(m)
```

### 4.2 季度汇总

```
季度总收入(Qx) = Σ 该季度各月的月总收入
季度总支出(Qx) = Σ 该季度各月的月总支出
季度净利润(Qx) = 季度总收入(Qx) - 季度总支出(Qx)
```

### 4.3 全年汇总

```
全年总收入 = Σ 各月月总收入（或 Σ Q1~Q4 季度总收入，结果应一致）
全年总支出 = Σ 各月月总支出
全年净利润 = 全年总收入 - 全年总支出
全年净利润率 = 全年净利润 / 全年总收入 × 100%
```

### 4.3.1 T5 净利润分析（与 T3/T4 联动）

```
T5 依赖同周期的收入总额与支出总额，且口径与 T3/T4 保持一致：
净利润(T) = 总收入(T) - 总支出(T)
净利润率(T) = 净利润(T) / 总收入(T) × 100%

执行 T5 时必须联动执行 T3/T4 或复用其同周期聚合结果。
若仅触发 T5，也应自动补齐 T3/T4 后再输出净利润结果。
```

### 4.4 收入结构占比

用于展示各收入科目在指定时间维度（月/季/年）总收入中的构成比例：

```
某收入科目在时间段 T 的金额 = Σ 该科目在 T 内各月的金额
某收入科目在时间段 T 的占比 = 该科目金额 / 时间段 T 总收入 × 100%

校验条件：所有收入科目占比之和 = 100%（允许四舍五入误差 ±0.1%）
```

### 4.5 支出结构占比

用于展示各支出科目在指定时间维度总支出中的构成比例：

```
某支出科目在时间段 T 的占比 = 该科目金额 / 时间段 T 总支出 × 100%

校验条件：所有支出科目占比之和 = 100%（允许四舍五入误差 ±0.1%）
```

## 五、计算结果校验规则

每完成一个维度的计算，必须执行以下校验，不通过时记录到 `notes` 并继续执行（不中断）：

| 校验项 | 通过条件 | 不通过时的处理 |
|--------|----------|--------------|
| 收入占比合计 | 99.9% ≤ 合计 ≤ 100.1% | 记录差值，标注"精度误差超限，请核查" |
| 支出占比合计 | 99.9% ≤ 合计 ≤ 100.1% | 同上 |
| T5联动一致性 | T5 使用的总收入/总支出口径与 T3/T4 一致 | 不一致则记录"T5与T3/T4口径不一致，请核查分类与周期" |
| 月度加总与合计列一致性 | 差异 ≤ 1% | 以月度加总为准，记录差异值 |
| 季度加总与全年一致性 | Q1+Q2+Q3+Q4 = 全年（允许精度误差） | 记录差异，标注"请核查数据完整性" |

## 六、数值格式规范（默认示例）

> 以下格式为默认展示示例；若用户明确指定输出格式，以用户要求为准。

| 类型 | 格式规范 |
|------|----------|
| 金额（计算过程） | 始终使用元，保留两位小数 |
| 金额（展示） | 根据量级自动换算：< 1万 显示元；1万~1亿 显示万元（保留两位小数）；≥ 1亿 显示亿元 |
| 百分比 | 保留两位小数，如 `80.86%` |
| 图表数据标签 | 金额类：整数或一位小数 + 单位；占比类：两位小数 + `%` |

> 用户可通过参数 `display_unit` 强制指定展示单位（"元"/"万元"/"亿元"），覆盖自动换算逻辑。

---

## 七、用户可配置参数汇总

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `custom_income_items` | list | `[]` | 追加到收入科目的自定义科目名称列表 |
| `custom_expense_items` | list | `[]` | 追加到支出科目的自定义科目名称列表 |
| `display_unit` | str | `"auto"` | 金额展示单位，支持"元"/"万元"/"亿元"/"auto" |
| `time_granularity` | str | `"month"` | 计算粒度，支持"month"/"quarter"/"year" |
