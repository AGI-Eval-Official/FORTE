name: finance-data-analyzer
description: 当用户需要对财务数据文件进行探查清洗与指标计算时触发，包括数据探查与清洗、财务科目分类、收入/支出/净利润的月度或季度汇总、结构占比计算等场景。触发关键词包括："财务分析"、"收入趋势"、"支出分析"、"净利润"、"收入占比"、"支出占比"、"季度汇总"、"帮我分析财务数据"。该 SKILL 依赖用户提供的 .xlsx 或 .csv 财务明细文件，将执行数据探查清洗、会计科目分类与指标聚合，输出结构化的财务指标计算结果。不适用于：生成图表、生成 PPT、生成标准财务报表（资产负债表/利润表）、非财务类数据分析。
---

## instruction

### 第一步：读取 reference 规则文件

在执行任何操作前，必须先读取以下规则文件，作为后续所有步骤的执行依据：

```
reference/data-cleaning-rules.md  → 数据探查与清洗规则
reference/accounting-rules.md     → 会计科目分类与指标计算规则
```

> 规则文件路径相对于本 SKILL.md 所在目录。

---

### 第二步：解析用户指令，确定任务列表

从用户指令中提取以下信息，**用户明确指定的内容优先级最高，覆盖所有默认值**：

#### 2.1 输入文件

```
- 文件路径：用户提供的数据文件路径（必须存在）
- 若用户未提供文件路径，询问用户提供文件后再继续
```

#### 2.2 任务类型识别

根据用户指令，识别需要执行的任务（支持多任务组合）：

| 任务 ID | 任务名称 | 触发关键词 |
|---------|---------|-----------|
| `T1` | 月度汇总计算 | "月度趋势"、"每月变化"、"月度汇总" |
| `T2` | 季度汇总计算 | "季度"、"Q1~Q4"、"按季度" |
| `T3` | 收入结构占比 | "收入占比"、"收入结构"、"收入构成" |
| `T4` | 支出结构占比 | "支出占比"、"支出结构"、"支出构成" |
| `T5` | 净利润分析 | "净利润"、"利润趋势"、"盈亏" |

> 若用户指令模糊（如"帮我分析这份财务数据"），默认执行 T1 + T3 + T4 + T5，并在输出中说明已执行的任务。
> T5 与 T3/T4 为联动任务：当识别到 T5 时，自动补齐并执行 T3 + T4，复用同周期收入/支出聚合结果。

#### 2.3 参数收集

从用户指令中提取以下参数，未提及的参数使用默认值（各参数默认值见对应 reference 文件）：

```
数据清洗参数：auto_clean, null_fill_value, amount_unit, sheet_name
会计计算参数：custom_income_items, custom_expense_items, display_unit, time_granularity
```

---

### 第三步：数据探查与清洗（强制执行）

**无论用户是否提及数据清洗，本步骤必须执行。**

按照 `data-cleaning-rules.md` 中的规则：

1. 读取输入文件，识别数据格式（宽表/长表）
2. 执行强制数据探查，生成 `probe_report`
3. 根据清洗决策路由（`auto_clean` 参数）决定是否执行清洗，清洗内容必须包含会计金额格式统一（货币符号/千分位/括号负数/单位后缀标准化）
4. 将探查结论和清洗记录写入 `notes`

---

### 第四步：财务指标计算

按照 `accounting-rules.md` 中的规则：

1. 识别并分类会计科目（收入/支出）
2. 根据任务列表确定需要计算的指标和时间维度：
   - T1：月度维度（收入/支出/净利润）
   - T2：季度维度
   - T3/T4：全年或用户指定时间段的占比
   - T5：净利润分析（默认月度，可按用户指定周期计算），并强制联动 T3/T4
3. 执行计算，完成校验，将校验结果写入 `notes`

---

### 第五步：输出结果汇总

所有任务执行完毕后，向用户输出以下内容：

> 以下输出结构与格式仅为默认示例；若用户明确指定输出格式（字段顺序、命名、单位、小数位、样式），必须以用户要求为准。

#### 5.1 执行摘要

```
已完成任务：[T1] 月度汇总计算、[T3] 收入结构占比 ...
数据期：{实际数据起止月份}
数据单位：{展示单位}
```

#### 5.2 关键指标（文字形式）

```
全年总收入：XX 万元
全年总支出：XX 万元
全年净利润：XX 万元（净利润率 XX%）
```

#### 5.3 计算结果明细

按任务输出对应的结构化数据表格：

- T1：各月收入/支出/净利润明细表
- T2：各季度收入/支出/净利润汇总表
- T3：各收入科目金额及占比表
- T4：各支出科目金额及占比表
- T5：各月（或用户指定周期）净利润及净利润率表（与 T3/T4 联动口径）

#### 5.4 注意事项（notes）

将所有 `notes` 中记录的内容（数据清洗记录、异常值标注、校验警告等）汇总展示，格式：

```
[注意事项]
1. {note 内容}
2. {note 内容}
...
```

若 `notes` 为空，输出"数据质量良好，无异常记录。"

---

## 执行约束

1. **用户指令优先**：用户明确指定的任何参数（包括单位、科目分类、输出格式、字段顺序与命名等）必须严格遵守，不得以"默认规则"覆盖用户指令。
2. **禁止虚构数值**：所有计算结果必须来源于输入数据，禁止估算、插值或虚构任何财务数值。
3. **数据探查不可跳过**：即使用户明确说"不需要清洗"，探查步骤仍必须执行，但可跳过清洗操作。
4. **notes 必须记录**：所有数据质量问题、清洗操作、校验警告必须记录到 `notes`，不得静默忽略。
5. **金额格式必须统一**：金额字段在进入计算前必须完成会计格式标准化并统一换算为元，禁止以原始混合格式直接参与计算。
6. **文件路径验证**：在读取输入文件前，必须验证路径有效性，失败时立即提示用户。
7. **多任务顺序执行**：多任务组合时，严格按照"探查→计算→输出"的顺序执行，不得跳步。
