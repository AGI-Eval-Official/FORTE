# 财务报表通用生成

## 概述

本技能基于原始财务数据（科目余额表或记账凭证等），经数据探查、清洗后，自动生成符合会计准则的标准财务报表。支持多种输入格式，覆盖三大核心报表，内置校验与兜底机制。

## 0. 前置条件与环境依赖

### 0.1 必需的运行环境

| 依赖项 | 版本要求 | 用途 |
|-------|---------|------|
| Python | ≥ 3.8 | 运行基础 |
| pandas | ≥ 1.5 | 数据读取与处理 |
| openpyxl | ≥ 3.1 | Excel 文件读写与格式化 |
| pdfplumber | ≥ 0.9 | PDF 表格提取（仅处理 PDF 输入时需要） |
| pdfplumber 或 tabula | 任选其一 | PDF 表格提取备选方案 |

> 若当前环境缺少上述依赖，在执行前先自动安装：`pip install pandas openpyxl pdfplumber`

### 0.2 标准输入参数

> **核心原则：以用户的 instruction 为准。** 

本技能只需关注三个核心参数，其余参数均从上下文自动推断或使用默认值：

| 参数 | 含义 | 未指定时的默认行为 |
|------|------|------------------|
| `input_file` | 输入数据文件路径 | **必填**——用户必须提供（路径或直接上传） |
| `output_dir` | 输出目录路径 | 使用输入文件所在目录 |
| `report_types` | 需要生成的报表列表 | 从用户指令关键词解析；未提及时默认仅生成资产负债表 |

## 1. 数据探查

收到输入文件后，**必须先执行数据探查**，再进入后续环节。探查内容包括：

### 1.1 数据质量检查

逐项检查以下内容，记录发现的问题：

| 检查项 | 判定标准 | 不通过时的处理 |
|-------|---------|--------------|
| **完整性** | 关键列（科目编码/名称、金额/借贷方向）是否存在 | 缺失列则尝试从相邻列推断或提示用户 |
| **一致性** | 同一科目是否重复出现且金额矛盾 | 保留明细行并汇总，冲突时以借方合计为准 |
| **借贷平衡** | 全部科目借方余额合计 ≈ 贷方余额合计（容差≤0.01） | 不平衡时发出警告，但仍继续生成报表 |
| **合理性** | 是否存在异常值（如负数存货、超额应收等） | 标记异常科目，不阻断流程 |
| **期间匹配** | 数据是否属于同一会计期间 | 混合期间数据按科目编码分组警告 |

> 探查结论分为三类，按结论跳转到对应环节：
> - **直接可用** → 跳过清洗，直接进入第 3 节（报表模板与结构）
> - **需清洗后使用** → 进入第 2 节（数据清洗），完成后再进入第 3 节
> - **无法处理**（数据严重缺失）→ 中断流程，告知用户补充数据，不继续执行

## 2. 数据清洗

以下规则仅在探查结论为「需清洗后使用」时执行：

| 清洗动作 | 说明 |
|---------|------|
| 表头标准化 | 统一映射常见变体列名：如"会计科目"/"科目名称"→"科目名称"，"期末余额"/"金额"→"金额" |
| 空值处理 | 剔除完全空白的行；金额为空的视为 0 |
| 方向统一 | 根据科目属性自动判定余额方向：资产类科目默认借方余额为正，负债/权益类科目默认贷方余额为正 |
| 明细合并 | 将同一总账科目的多个明细科目（如"银行存款-工行"+"银行存款-建行"）汇总为总账科目 |
| 去除辅助列 | 删除序号、备注等非计算相关列 |

## 3. 报表模板与结构

各报表的标准项目分类、核心计算公式及输出格式定义，分别存储在以下独立参考文件中：

| 报表 | 参考文件 | 内容概要 |
|------|---------|---------|
| 资产负债表 | **`reference/ref-balance-sheet.md`** | 资产（流动/非流动）12+12 项明细、负债（流动/非流动）12+8 项明细、权益 7 项、计算公式、精简版与完整版两种输出格式 |
| 利润表 | **`reference/ref-income-statement.md`** | 多步式 7 大板块取数说明（含常见科目别名）、四步计算链条、标准多步式输出格式 |
| 现金流量表 | **`reference/ref-cash-flow-statement.md`** | 经营/投资/筹资三大活动共 30+ 行项目、各项目倒算公式、辅助数据要求说明、交叉校验规则 |

实际生成时根据用户指令选择所需报表对应的参考文件，**未要求的报表不生成**。未指定则默认仅使用 `reference/ref-balance-sheet.md`。

## 4. 输出规范

> **优先级原则：用户指令 > 以下默认规范。** 如果用户在 instruction 中明确指定了输出格式（如文件格式、小数位数、列名、是否加边框等），以用户的指令为准。本节所有格式要求均为**未指定时的默认行为**。

### 4.1 通用格式要求

| 要求项 | 标准 |
|-------|------|
| 文件格式 | Excel（.xlsx），每种报表一个 Sheet |
| 金额格式 | 数值格式，四舍五入保留 2 位小数 |
| 对齐方式 | 居中对齐 |
| 边框 | 全表细线边框 |
| 表头 | 加粗字体，字号略大于数据行 |
| 列宽 | 自适应内容，项目列≥18字符宽，金额列≥14字符宽 |

### 4.2 各报表具体输出格式

各报表的列名、行项目顺序及示例，详见对应参考文件的「输出格式」章节：资产负债表见 `reference/ref-balance-sheet.md` 第6节、利润表见 `reference/ref-income-statement.md` 第4节、现金流量表见 `reference/ref-cash-flow-statement.md` 第6节。生成时应严格遵循对应 reference 中定义的格式，不得擅自增减行项或修改列名（除非用户明确要求调整）。

## 5. 输出 Checklist 与兜底机制

### 5.1 生成完成后必须逐项核对

| # | 检查项 | 通过标准 |
|---|-------|---------|
| 1 | 文件已保存至指定路径 | 输出路径与用户要求完全一致 |
| 2 | 金额列单元格格式 | 数值格式，小数位数为 2 |
| 3 | 资产负债表平衡校验 | \|资产总计 - (负债总计 + 所有者权益合计)\| ≤ 0.01 |
| 4 | 无空值/无 #REF! 等错误 | 所有数据单元格均为有效数值 |
| 5 | 科目匹配覆盖率 | 已匹配金额占输入总金额的比例 ≥ 90%；若低于此比例则在输出中附《未匹配科目清单》 |
| 6 | 借贷平衡（输入侧） | \|∑借方 - ∑贷方\| ≤ 0.01；不平衡时在输出文件中新增"校验说明"Sheet 记录差异 |

### 5.2 兜底机制

当遇到以下情况时，按对应策略处理，**不得中断或询问用户**：

| 异常场景 | 兜底策略 |
|---------|---------|
| 科目名称模糊匹配到多个候选 | 取金额绝对值最大的一项 |
| 输入文件中缺少某报表必需的关键科目 | 该项目填 0，并在输出中标注"数据缺失" |
| 用户未指定需要哪些报表 | 默认只生成资产负债表（最基础、最通用） |
| 用户指定的输出目录不存在 | 自动创建目录 |
| 输入数据借贷不平衡差异 > 0.01 | 仍正常生成报表，将差额挂入"未分配利润"调整项并记录到校验说明 |
| 生成的报表平衡校验不通过 | 按差异方向调整"未分配利润"使报表强制平衡，同时在独立Sheet中记录原始数据与调整额 |
| PDF/图片识别成功率 < 80% | 回退到半自动模式：提取可识别部分，对不确定区域标记后请用户确认后再继续 |

## 6. 执行步骤总结

1. **接收输入文件** → 解析 input_file / output_dir / report_types 等参数（第 0.2 节）
2. **数据探查** → 完整性/一致性/合理性/借贷平衡/期间匹配 五维检查（第 1 节）
3. **数据清洗**（如探查结论为需清洗）→ 表头标准化、空值处理、方向统一、明细合并（第 2 节）
4. **确定报表范围** → 根据 report_types 加载对应的 ref 参考文件（第 3 节）
5. **科目映射与计算** → 按模板定义的项目分类，精确匹配+模糊匹配取数，执行公式计算
6. **格式化输出** → 按输出规范生成 Excel 文件（第 4 节）
7. **Checklist 校验** → 逐项执行 5.1 中的 6 项检查（第 5.1 节）
8. **兜底处理** → 对校验不通过项执行 5.2 中的对应策略，确保最终产出可用文件（第 5.2 节）
9. **输出处理日志** → 在输出目录生成 `_processing_log.txt` 记录全链路处理记录（第 7 节）
