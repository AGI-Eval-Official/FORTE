# 国际银行账号（IBAN）格式与校验位验证

## 功能说明

根据 ISO 13616-1:2015 国际标准，对国际银行账号（International Bank Account Number, IBAN）进行多维度的合规性检查，包括长度验证、格式验证和校验码计算验证，确保跨境付款前账号信息的准确性和有效性。

## 输入要求

### 支持的文件格式

支持以下格式的输入文件：`.xlsx`、`.xls`、`.csv`、`.tsv`

### 必需字段

每条记录至少需包含以下信息：

| 必需信息 | 说明 |
|---------|------|
| 国家 | 国家名称（中文或英文）或 ISO 3166-1 国家代码 |
| 收款方名称 | 收款方名称 |
| IBAN | 完整的国际银行账号 |

### 字段映射

输入文件中的列名可能与上表列名不一致。AI 应根据列名语义自动映射到对应的必需字段，而非依赖精确列名匹配。映射示例如下：

| 必需信息 | 可能的列名示例 |
|---------|-------------|
| 序号 | 序号、编号、No.、ID、Index |
| 国家 | 国家、国家/地区、Country、Nation |
| 收款方名称 | 收款方名称、收款人、户名、Payee、Account Name、Beneficiary |
| IBAN | IBAN、国际账号、账号、Account Number |

**若用户在指令中明确指定了列名映射，则优先按用户指定的映射关系执行。**

## 验证规则

### 一、IBAN 的国际标准结构

根据 ISO 13616-1:2015 标准，IBAN 由以下部分组成：

| 部分 | 位数 | 内容 | 格式 |
|------|------|------|------|
| 国家代码 | 2位 | ISO 3166-1 国家码 | 大写字母（A-Z） |
| 校验位 | 2位 | 由 MOD 97-10 算法计算 | 数字（00-99） |
| 基本账号（BBAN） | 可变 | 各国自己的银行账号格式 | 字母+数字 |

**总长度范围**：15-34位

### 二、国家 IBAN 长度参照标准

具体规则和完整国家长度表详见：`reference/iban-length-rules.md`

> **注意**：该文件为本 skill 验证 IBAN 长度的唯一依据，不得使用其他来源的数据替代或补充。

### 三、校验位计算规则（ISO 7064 MOD 97-10）

**计算步骤**：

1. 将 IBAN 的前4位（前2位国家码+后2位校验位）移至末尾
2. 将所有字母转换为数字（A=10, B=11, ..., Z=35）
3. 对得到的数字串进行 MOD 97 运算
4. 若余数为 1，则校验码正确

**示例**：
- 原始 IBAN：`DE89370400440532013000`
- 重新排列：`370400440532013000DE89` → `370400440532013000131489`
- MOD 97 计算 → 余数为 1 ✅

**规则**：校验位必须使用正确的 MOD 97-10 算法计算，不得自行假设。

> **注意**：校验码计算基于预处理后的 IBAN（即转大写、去除空格和特殊字符后的纯大写字母+数字字符串），以确保算法可正确执行。

### 四、三项合规性检查规则

对每条记录，需按以下顺序执行检查和预处理，再进行后续判断：

#### 1. 格式合规（基于原始 IBAN 检查，必须在预处理之前执行）

- 检查原始 IBAN 是否仅包含大写字母（A-Z）和数字（0-9），不包含小写字母、空格、特殊符号
- 检查原始 IBAN 前两位国家码是否在 `reference/iban-length-rules.md` 的国家长度表中（即是否为有效 IBAN 国家码）
- 两项均满足则 `是`，任一不满足则 `否`
- **注意**：格式合规不检查长度是否与该国标准一致（长度由"长度合规"单独判断）

#### 2. 数据预处理（格式合规检查之后执行）

- IBAN 预处理：统一转换为大写 → 去除空格和特殊字符，仅保留字母和数字 → 预处理后的 IBAN 用于后续的长度检查、校验码检查和输出
- 国家名称转换：依据 `reference/country-mapping-rules.md` 将国家名称转换为 ISO 3166-1 代码

#### 3. 长度合规（基于预处理后的 IBAN 检查）

- 根据预处理后 IBAN 前两位国家码，在 `reference/iban-length-rules.md` 中查找该国的标准长度
- 将预处理后 IBAN 的实际字符数与标准长度比较：一致则 `是`，否则 `否`
- 若国家码不在长度表中，判定为 `否`

#### 4. 校验码正确（基于预处理后的 IBAN 检查）

- 按 ISO 7064 MOD 97-10 算法对预处理后的 IBAN 进行校验位计算
- 计算结果余数为 1 则 `是`，否则 `否`
- 即使国家码无效，仍需按算法对预处理后的现有字符进行计算并判定

### 五、综合判定规则

- `合规`：三项检查均为"是"
- `不合规（具体原因）`：任一检查为"否"，括号内列出所有不合规项，用分号分隔

**不合规原因表述**：
- 长度不合规 → `长度不符`
- 格式不合规 → `格式不符`
- 校验码不合规 → `校验码错误`

示例：`不合规（长度不符；校验码错误）`

### 六、特殊情形处理

#### 1. IBAN 国家码与输入国家不一致

- **输出中的"国家"字段**始终取自输入的国家名称转换后的 ISO 代码，不受 IBAN 内容影响
- **格式合规检查**的对象是 IBAN 字符串本身的前两位国家码是否有效，与输入的国家字段无关
- 例如：输入国家为"葡萄牙"→输出国家字段为 `PT`，但 IBAN 为 `XX9121828374961238596366`，则格式合规为 `否`（XX 不是有效国家码）

#### 2. 无效国家码时的判定汇总

当 IBAN 前两位国家码不在 `reference/iban-length-rules.md` 中时（无论字符集是否合规）：

| 检查项 | 判定 | 说明 |
|--------|------|------|
| 格式合规 | `否` | 国家码无效（若字符集同时不合规，也是`否`） |
| 长度合规 | `否` | 无法确定该国家的标准长度 |
| 校验码正确 | 仍需计算 | 对预处理后的现有字符执行 MOD 97-10 |
| 综合判定 | `不合规（格式不符；…）` | 至少包含"格式不符" |

### 七、国家名称 → ISO 3166-1 代码映射参照标准

具体映射表详见：`reference/country-mapping-rules.md`

> **注意**：该文件为本 skill 国家名称转换的唯一依据，不得使用其他来源的数据替代或补充。

## 输出格式

**若用户在指令中指定了输出格式（文件格式、列名、列顺序等），则优先按用户指定的格式输出。** 用户未指定时，按以下默认格式输出。

### 默认输出格式

输出为 CSV 文件（`.csv`格式），CSV 表头必须严格按照以下顺序输出，不得调换或增减列：

```
序号,国家,收款方名称,IBAN,长度合规,格式合规,校验码正确,综合判定
```

各列取值说明：
- **序号**：与输入一致
- **国家**：ISO 3166-1 国家代码（由输入的国家名称转换而来，非国家名称原文）
- **收款方名称**：与输入一致
- **IBAN**：预处理后的大写纯字符形式（转大写、去除空格和特殊字符后）
- **长度合规**：`是` 或 `否`
- **格式合规**：`是` 或 `否`
- **校验码正确**：`是` 或 `否`
- **综合判定**：`合规` 或 `不合规（具体原因）`

## 执行步骤

1. **读取输入文件**：识别文件格式（xlsx/xls/csv/tsv），读取内容
2. **字段映射**：根据列名语义将输入列映射到必需字段（序号、国家、收款方名称、IBAN）；若用户指定了映射关系则优先遵循
3. **格式合规检查**（对每条记录的**原始IBAN**执行，必须在预处理之前）：
   - 检查原始 IBAN 是否仅包含大写字母（A-Z）和数字（0-9），不包含小写字母、空格、特殊符号
   - 检查原始 IBAN 前两位国家码是否在 `reference/iban-length-rules.md` 的国家长度表中
   - **重要**：此步骤必须基于原始输入的 IBAN 判断，不得先进行任何预处理，否则将遗漏小写字母、空格等格式不合规的情况
4. **数据预处理**（对每条记录依次执行）：
   - IBAN 预处理：统一转换为大写 → 去除空格和特殊字符，仅保留字母和数字 → 预处理后的 IBAN 用于后续的长度检查、校验码检查和输出
   - 国家名称转换：依据 `reference/country-mapping-rules.md` 将国家名称转换为 ISO 3166-1 代码
5. **逐条验证**（基于预处理后的IBAN执行以下两项检查）：
   - 长度合规：依据 `reference/iban-length-rules.md` 中的国家长度表，将预处理后 IBAN 的字符数与标准长度比较
   - 校验码正确：按 ISO 7064 MOD 97-10 算法对预处理后的 IBAN 进行校验位计算
6. **综合判定**：根据三项检查结果（步骤3的格式合规 + 步骤5的长度合规和校验码正确）给出综合判定（规则见第五节）
7. **生成输出**：按输出格式生成结果文件，保持与输入相同的序号顺序
