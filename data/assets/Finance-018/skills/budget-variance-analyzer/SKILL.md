name: budget-variance-analyzer
description: 当用户需要对任意周期（月度/季度/年度）的预算执行情况进行差异核算与归因分析时触发。触发关键词包括："预算差异"、"超支分析"、"节约分析"、"预算执行"、"差异率"、"预算与实际对比"、"哪些科目超支了"、"预算执行情况"、"预算偏差"。该 SKILL 依赖用户提供的预算表与实际支出明细表（.xlsx 或 .csv），可选接入营收数据表作为归因辅助，将执行差异计算、超支/节约标记、重点科目筛选与原因归因，输出结构化的预算执行汇总表及差异分析报告。
---

## instruction

### 第一步：读取 reference 规则文件

在执行任何操作前，必须先读取以下规则文件，作为后续所有步骤的执行依据：

```
reference/variance-calculation-rules.md  → 差异计算公式、标记规则、重点科目筛选阈值
reference/variance-cause-taxonomy.md     → 差异原因分类框架与归因依据
```

> 规则文件路径相对于本 SKILL.md 所在目录。

---

### 第二步：解析用户指令，确定任务参数

从用户指令中提取以下参数，**用户明确指定的内容优先级最高，覆盖所有默认值**：

#### 2.1 输入文件

```
- 预算表路径（必填）：包含各业务线/部门、费用科目、预算额度的数据文件
- 实际支出明细表路径（必填）：包含实际支出记录的数据文件
- 营收数据表路径（选填）：用于辅助归因，缺失时跳过营收维度分析
```

若必填文件路径缺失，立即提示用户补充，不得继续执行。

#### 2.2 分析参数

```
- 分析周期：用户指定的时间范围（如"Q1"、"2026年上半年"、"1-6月"），未指定时从数据中自动识别
- 业务线范围：全部业务线（默认）或用户指定的子集
- 重点科目筛选阈值：差异率绝对值的触发阈值，默认值见 variance-calculation-rules.md
- 金额单位：默认万元，用户可指定
```

#### 2.3 输出任务识别

根据用户指令，识别需要执行的输出任务（支持组合）：

| 任务 ID | 任务名称 | 触发关键词 |
|---------|---------|-----------|
| `O1` | 预算执行汇总表 | "汇总表"、"执行表"、"差异表"、"生成表格" |
| `O2` | 重点科目差异分析 | "差异分析"、"原因分析"、"超支原因"、"节约原因" |
| `O3` | 优化建议 | "优化建议"、"改进建议"、"下期预算建议" |

> 若用户指令模糊（如"帮我做预算差异分析"），默认执行 O1 + O2 + O3，并在输出中说明已执行的任务。

---

### 第三步：数据口径对齐

在计算前，必须完成以下口径校验，不通过时记录到 `notes` 并提示用户：

1. **时间范围对齐**：预算表与支出明细表的时间范围必须一致，不一致时以两者交集为准并记录
2. **金额单位统一**：将所有金额统一转换为用户指定单位（默认万元），记录转换操作
3. **科目名称映射**：识别预算科目与支出科目的名称差异，建立映射关系；无法自动映射的科目记录到 `notes`，提示用户确认
4. **业务线名称对齐**：确保预算表与支出表中的业务线名称一致，不一致时记录差异

---

### 第四步：差异计算

按照 `variance-calculation-rules.md` 中的规则执行：

1. 按业务线 × 费用科目维度，计算差异金额与差异率
2. 按规则对每条记录打上超支/节约/持平标记
3. 按阈值筛选重点科目列表
4. 执行校验（汇总差异金额与各行差异金额之和一致性），不通过时记录到 `notes`

---

### 第五步：差异归因分析（仅 O2 任务）

仅针对重点科目（差异率超过阈值的科目）执行归因分析：

1. 读取 `variance-cause-taxonomy.md` 中的原因分类框架
2. 结合支出备注字段、营收数据（若有）、业务线上下文，逐科目判断最可能的原因类别
3. 每个重点科目输出：原因类别 + 具体情形描述 + 分析依据
4. 若支出备注为空且无营收数据，在 `notes` 中标注"[科目名] 归因依据不足，建议人工补充说明"

---

### 第六步：优化建议生成（仅 O3 任务）

针对重点科目，基于归因结论生成可落地的优化建议：

- 超支科目：聚焦管控措施、审批流程优化、下期预算调整方向
- 节约科目：聚焦预算调配建议、成本控制经验固化
- 每条建议必须与具体科目绑定，禁止输出泛化的通用建议

---

### 第七步：输出结果

按用户指定格式（或默认格式）输出，输出内容与格式规范见 `assets/variance-report-template.md`：

#### 7.1 执行摘要（必输出）

```
已完成任务：[O1] 预算执行汇总表、[O2] 重点科目差异分析 ...
分析周期：{实际数据起止时间}
数据单位：{金额单位}
重点科目数：{超过阈值的科目总数}（超支 X 个 / 节约 X 个）
```

#### 7.2 预算执行汇总表（O1）

按 `assets/variance-report-template.md` 中的汇总表模板输出，包含汇总统计行。

#### 7.3 重点科目差异分析（O2）

按 `assets/variance-report-template.md` 中的分析报告模板，逐科目输出归因结论。

#### 7.4 优化建议（O3）

按 `assets/variance-report-template.md` 中的建议模板，逐科目输出可落地建议。

#### 7.5 注意事项（notes）

汇总所有 `notes` 中记录的内容（口径对齐问题、归因依据不足提示、校验警告等）：

```
[注意事项]
1. {note 内容}
2. {note 内容}
...
```

若 `notes` 为空，输出"数据口径一致，无异常记录。"

---

## 执行约束

1. **用户指令优先**：用户明确指定的任何参数（单位、阈值、业务线范围等）必须严格遵守，不得以默认规则覆盖。
2. **禁止虚构数值**：所有计算结果必须来源于输入数据，禁止估算、插值或虚构任何财务数值。
3. **口径对齐不可跳过**：即使用户说"直接算"，第三步的口径校验仍必须执行。
4. **归因不足时不得编造**：若归因依据不足，必须在 notes 中如实标注，不得凭空推断原因。
5. **notes 必须记录**：所有口径问题、映射操作、校验警告必须记录到 `notes`，不得静默忽略。
6. **多任务顺序执行**：严格按照"口径对齐 → 差异计算 → 归因分析 → 建议生成 → 输出"的顺序执行，不得跳步。

---

## 检查清单

在最终输出前，逐项确认以下内容，任一项未通过须返回对应步骤修正后再输出：

**数据层**
- [ ] 预算表与支出明细表的时间范围已对齐，差异已记录到 notes
- [ ] 所有金额已统一转换为用户指定单位
- [ ] 科目名称映射已完成，无法映射的科目已在 notes 中标注
- [ ] 业务线名称已对齐，不一致项已记录

**计算层**
- [ ] 差异金额与差异率计算公式正确（实际支出 - 预算额度）
- [ ] 超支/节约/持平标记已按规则打标
- [ ] 重点科目已按阈值筛选完整，无遗漏
- [ ] 汇总校验已执行，各行差异金额之和与总差异金额一致（误差 ≤ 0.01 万元）

**归因层**（仅 O2 任务）
- [ ] 归因仅针对重点科目，非重点科目未做归因
- [ ] 每个重点科目均输出了原因类别、具体情形、分析依据三要素
- [ ] 归因依据不足的科目已在 notes 中标注，未凭空推断

**输出层**
- [ ] 汇总表表头顺序与模板一致，未自行调整列顺序
- [ ] 金额保留 2 位小数，差异率为百分比格式保留 2 位小数
- [ ] 优化建议与具体科目绑定，无泛化通用建议（仅 O3 任务）
- [ ] notes 已汇总输出，为空时输出"数据口径一致，无异常记录。"
