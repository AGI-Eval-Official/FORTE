# 差异原因分类框架

> 本文件定义 budget_variance_analyzer SKILL 在执行差异归因时使用的原因分类体系与判断依据。
> 归因结论必须基于可观测的数据证据（支出备注、营收变化、业务上下文），禁止在无依据时凭空推断。

---

## 一、超支原因分类

### 1.1 原因类别与判断依据

| 原因类别 | 具体情形 | 主要判断依据 |
|----------|----------|-------------|
| 业务扩张 | 新增业务线、扩大经营规模、新项目启动带来额外支出 | 营收同比增长显著；支出备注含"新增"、"扩张"、"新项目" |
| 市场因素 | 原材料/服务价格上涨、供应商涨价、竞争加剧导致营销投入增加 | 支出备注含"涨价"、"价格调整"；同类科目历史数据对比 |
| 管理因素 | 预算编制偏低、费用管控不足、审批流程缺失 | 差异率持续多期超标；支出备注无特殊说明；历史同期对比偏差大 |
| 临时支出 | 突发事件、临时需求、计划外采购 | 支出备注含"临时"、"紧急"、"突发"、"计划外" |
| 政策/外部因素 | 监管政策变化、不可抗力、汇率波动 | 支出备注含"政策"、"合规"、"汇率"；外部环境变化 |

### 1.2 归因优先级规则

```
1. 支出备注有明确说明 → 直接采用备注中的原因，归入最匹配的类别
2. 支出备注为空，但营收数据显示同期增长 ≥ 20% → 优先归入"业务扩张"
3. 支出备注为空，营收数据无显著变化，且历史同期差异率 < 5% → 归入"管理因素"
4. 以上条件均不满足 → 标注"归因依据不足"，不强行归类
```

---

## 二、节约原因分类

### 2.1 原因类别与判断依据

| 原因类别 | 具体情形 | 主要判断依据 |
|----------|----------|-------------|
| 业务收缩 | 业务线调整、项目暂停、战略收缩导致支出减少 | 营收同比下降显著；支出备注含"暂停"、"收缩"、"取消" |
| 成本控制 | 主动优化流程、降本增效措施落地 | 支出备注含"优化"、"降本"、"谈判"；多科目同步节约 |
| 预算偏高 | 预算编制过于保守，实际需求低于预期 | 节约率持续多期稳定；无特殊业务变化 |
| 采购优化 | 供应商优化、集中采购、价格谈判成功 | 支出备注含"集采"、"谈判"、"供应商优化" |
| 计划延期 | 项目延期、支出推迟至下期 | 支出备注含"延期"、"推迟"、"下期执行" |

### 2.2 归因优先级规则

```
1. 支出备注有明确说明 → 直接采用备注中的原因，归入最匹配的类别
2. 支出备注为空，但营收数据显示同期下降 ≥ 20% → 优先归入"业务收缩"
3. 支出备注为空，营收数据无显著变化，节约率 ≥ 30% → 归入"预算偏高"或"采购优化"，在 notes 中标注需人工确认
4. 以上条件均不满足 → 标注"归因依据不足"，不强行归类
```

---

## 三、归因输出格式规范

每个重点科目的归因结论必须包含以下三个要素：

```
原因类别：[从上述分类中选择最匹配的一个类别]
具体情形：[用一句话描述该科目的具体超支/节约情形]
分析依据：[列出支撑该归因结论的可观测证据，如"支出备注显示XX"、"营收同比增长XX%"]
```

若归因依据不足，输出格式为：

```
原因类别：待确认
具体情形：暂无足够依据判断
分析依据：支出备注为空，且无营收数据辅助，建议业务方补充说明
```

---

## 四、归因置信度说明

| 置信度 | 条件 | 处理方式 |
|--------|------|----------|
| 高 | 支出备注有明确说明，且与营收数据吻合 | 直接输出归因结论 |
| 中 | 仅有营收数据或仅有支出备注，两者之一支持归因 | 输出归因结论，在 notes 中标注"建议人工复核" |
| 低 | 无支出备注，无营收数据，仅凭历史规律推断 | 标注"归因依据不足"，不输出具体原因类别 |
