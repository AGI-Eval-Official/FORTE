# cashflow-forecast-rules

## 1. 目标

定义资金头寸滚动预测的核心业务规则，包括余额汇总口径、账户状态筛选逻辑、收支按日聚合规则、滚动计算公式与起点定义。

本文件只包含跨任务稳定的业务判断规则，不包含字段映射、单位换算、异常处理等数据清洗规范。

---

## 2. 余额汇总规则

### 2.1 账户状态筛选

从银行账户余额表中汇总总可用资金，按以下规则筛选账户：

| 账户状态 | 处理方式 |
|----------|----------|
| 正常 | 全部纳入计算，包括余额为负的透支账户 |
| 冻结 | 剔除，不参与可用资金计算 |
| 久悬 | 剔除，不参与可用资金计算 |
| 其他未知状态 | 标记异常并剔除，在结果说明中列出 |

> 若账户余额表不含状态字段，则全部账户视为正常账户参与计算，并在结果说明中注明。

### 2.2 汇总计算

```
总可用资金 = Σ 所有纳入计算的账户余额
```

- 负余额账户保留参与汇总（不做截断）
- 若汇总结果为负值，照常参与后续预测，并在结果说明中标注"期初余额为负，请核查账户数据"
- 金额单位以账户余额表中标注的单位为准

---

## 3. 收支按日聚合规则

### 3.1 付款聚合

```
每日付款总额 = 该日期下所有付款记录的金额之和
```

- 若某日无付款记录，当日付款总额补 `0`
- 同一笔款项不得重复计入

### 3.2 回款聚合

```
每日回款总额 = 该日期下所有回款记录的金额之和
```

- 若某日无回款记录，当日回款总额补 `0`
- 同一笔回款不得重复计入

### 3.3 日期范围

- 预测日期范围：从 `date_anchor` 起，连续 `forecast_days` 天（默认 7 天）
- 对预测区间内无任何收支记录的日期，付款和回款均补 `0`，正常参与滚动计算
- 超出预测区间的收支记录不纳入本次计算

---

## 4. 滚动预测计算规则

### 4.1 起点定义

```
第 1 天期初余额 = 总可用资金（由余额汇总步骤得出）
```

### 4.2 逐日滚动公式

```
当日期末余额 = 当日期初余额 + 当日回款总额 - 当日付款总额
次日期初余额 = 当日期末余额
```

- 严格按日期顺序迭代，不得跳日
- 期末余额允许为负值，不做截断
- 所有金额默认保留 2 位小数（中间计算过程保持高精度浮点，仅最终输出时统一四舍五入）；若用户明确指定其他精度格式（如整数输出），以用户指定为准

### 4.3 计算完整性校验

输出前应自检：
- 输出行数 = `forecast_days`，日期连续无缺失
- 每行满足：期末余额 = 期初余额 + 回款额 - 付款额（允许浮点误差 ≤ 0.01）
- 第 N+1 天期初余额 = 第 N 天期末余额

若校验不通过，记录异常行并在结果说明中标注。
