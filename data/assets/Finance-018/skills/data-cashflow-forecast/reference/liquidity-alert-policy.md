# liquidity-alert-policy

## 1. 目标

定义资金头寸预测场景的通用预警分级规则、等级标识映射优先级与默认阈值策略，用于统一风险解释口径。

## 2. 适用范围

- 适用于日频资金头寸滚动预测结果
- 输入为按日计算得到的期末余额（万元）
- 输出为风险等级标签

## 3. 默认阈值

- `red_threshold = 200`（万元）
- `safety_reserve = 300`（万元）

阈值含义：

- `red_threshold`: 高风险临界值
- `safety_reserve`: 安全备付目标值

## 4. 分级规则

对于任意日期 `d` 的期末余额 `closing_balance_d`，先判定风险区间，再映射等级标识：

- 高风险区：`closing_balance_d < red_threshold`
- 关注区：`red_threshold <= closing_balance_d < safety_reserve`
- 安全区：`closing_balance_d >= safety_reserve`

默认示例等级标识（仅示例，可被覆盖）：

- 高风险区 -> 红色预警
- 关注区 -> 黄色预警
- 安全区 -> 正常

> 说明：红/黄/正常仅为默认示例；最终输出标识以用户要求为准。

## 5. 配置优先级

### 5.1 阈值配置优先级

按以下顺序应用阈值：

1. 任务输入显式给定阈值
2. 任务输入中的规则文件阈值
3. 本文件默认阈值

当阈值来源冲突时，采用更高优先级来源并记录决策依据。

### 5.2 等级标识映射优先级

按以下顺序确定输出等级标识：

1. 用户明确指定的等级标识（最高优先）
2. 任务输入中的规则文件或参数指定映射
3. 本文件默认示例标识（红色预警/黄色预警/正常）

当标识来源冲突时，采用更高优先级来源并记录决策依据。

## 6. 参数合法性校验

执行分级前应校验：

- `red_threshold >= 0`
- `safety_reserve >= 0`
- `red_threshold <= safety_reserve`

若不满足条件：

- 优先尝试交换错误阈值（仅在明显录入反转时）
- 否则回退默认阈值并记录告警

## 7. 风险说明模板（可复用）

- 高风险区（默认示例标识：红色预警）：资金安全垫不足，需立即控制支出并评估融资/调拨方案
- 关注区（默认示例标识：黄色预警）：接近安全阈值，建议加强回款管理并优化付款节奏
- 安全区（默认示例标识：正常）：处于安全区间，保持常规监控即可

## 8. 输出字段建议

> **输出格式优先级：用户明确指定的列名、字段顺序、值域格式等，以用户指定为准；未指定时，按以下默认建议输出。**

预警结果输出建议至少包含：

- `date`
- `closing_balance_wan`
- `alert_level`（值域以用户要求等级标识为准；未指定时使用默认示例标识）

以下为可选审计字段，默认不在主输出列中，仅在任务有明确需要时附加：

- `threshold_red`
- `threshold_safety`
