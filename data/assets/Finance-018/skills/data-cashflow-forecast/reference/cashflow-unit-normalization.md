# cashflow-unit-normalization

## 1. 目标

定义金额单位识别与标准化规则，将多来源金额统一为“万元”口径，保证可比性与计算稳定性。

## 2. 标准口径

- 统一目标单位：`万元`
- 内部计算单位：`万元`
- 默认保留精度：2 位小数（展示层）

## 3. 输入值清洗

金额字段在转换前先做清洗：

- 去除千分位分隔符（如 `1,234,567.89`）
- 去除货币符号（如 `¥`, `￥`, `$`）
- 去除首尾空格和不可见字符
- 将空字符串、`N/A`、`null`、`--` 识别为空值

## 4. 单位识别规则

按优先级识别单位：

1. 任务输入显式声明的单位（最高优先）
2. 列名单位标记（如 `金额(元)`、`金额(万元)`、`金额(亿)`）
3. 同表单位字段（如 `单位` 列）
4. 规则默认值（未识别时默认按 `元`）

## 5. 换算规则

转换到万元的系数：

- 元 -> 万元: `value / 10000`
- 千元 -> 万元: `value / 10`
- 万元 -> 万元: `value`
- 亿元 -> 万元: `value * 10000`

## 6. 文字金额处理

当金额中包含中文单位后缀时：

- `12.5万` -> `12.5`（万元）
- `0.8亿` -> `8000`（万元）
- `3500元` -> `0.35`（万元）

若出现复杂表达（如“约1.2万-1.5万”），标记为异常值，不参与自动计算。

## 7. 精度与舍入

- 内部过程尽量保留高精度浮点
- 最终输出时统一四舍五入到 2 位小数
- 禁止在中间步骤反复舍入，避免累计误差

## 8. 异常策略

- 无法识别单位：记录告警并按默认单位处理
- 非法金额（无法转数值）：按 0 处理并记录计数
- 单列中混用多单位：按行转换并记录“单位混用”告警

## 9. 审计信息建议

结果说明中建议输出：

- 单位识别依据
- 执行了哪些换算
- 异常金额条数
- 是否存在单位混用
