# 资金头寸预测与流动性预警 Skill

本 Skill 定义"资金头寸预测与预警"的通用业务动作，用于对未来若干天的现金流入流出进行滚动测算，并输出风险分级结果。

## 1. 输入依赖与接口契约

### 1.1 必要输入

至少需要三类数据：

1. 账户余额表（期初资金基础）
2. 付款计划表（未来每日资金流出）
3. 回款计划表（未来每日资金流入）

支持格式：`.xlsx`、`.xls`、`.csv`

### 1.2 字段要求（语义级）

#### 余额表

- 必需语义字段：账户余额
- 可选语义字段：账户状态、账户名称、币种

#### 付款计划表

- 必需语义字段：日期、金额
- 可选语义字段：收款方、用途

#### 回款计划表

- 必需语义字段：日期、金额
- 可选语义字段：付款方、业务类型

说明：字段名允许存在同义表达，执行时应先完成语义映射再计算。

### 1.3 可配置参数

- `forecast_days`：预测天数，默认 `7`
- `safety_reserve`：安全备付额（万元），默认 `300`
- `red_threshold`：红色预警阈值（万元），默认 `200`
- `amount_unit`：金额口径（如元/万元），默认自动识别并统一为万元
- `date_anchor`：预测起始日期，默认由任务输入决定（优先使用数据中的最早预测日）

## 2. 核心规则与流程

### 步骤 1：读取与字段标准化

- 读取三类输入数据
- 执行字段语义映射（日期、金额、余额、状态）
- 将日期统一为 `YYYY-MM-DD`
- 将金额统一为“万元”口径

### 步骤 2：余额汇总

读取 `cashflow-forecast-rules.md` 第 2 节执行，核心规则如下：

- 状态为"正常"的账户全部纳入，**含余额为负的透支账户**，不做截断
- 状态为"冻结"或"久悬"的账户剔除，不参与计算
- 其他未知状态账户标记异常并剔除，在结果说明中列出
- 若余额表无状态字段，则全部账户视为正常，在结果说明中注明
- `总可用资金 = Σ 所有纳入计算的账户余额`，汇总结果为负时照常参与后续预测并标注
- 金额单位以账户余额表中标注的单位为准

### 步骤 3：日级流入流出聚合

读取 `cashflow-forecast-rules.md` 第 3 节执行，核心规则如下：

- `每日付款总额 = 该日期下所有付款记录的金额之和`，同一笔款项不得重复计入
- `每日回款总额 = 该日期下所有回款记录的金额之和`，同一笔回款不得重复计入
- 预测区间内无付款记录的日期，付款总额补 `0`；无回款记录的日期，回款总额补 `0`
- 超出预测区间（`date_anchor` 起 `forecast_days` 天）的收支记录不纳入本次计算

### 步骤 4：滚动预测

读取 `cashflow-forecast-rules.md` 第 4 节执行，核心规则如下：

- **起点**：`第 1 天期初余额 = 总可用资金`（由步骤 2 汇总得出）
- **逐日公式**：`当日期末余额 = 当日期初余额 + 当日回款总额 - 当日付款总额`
- **衔接规则**：`次日期初余额 = 当日期末余额`，严格按日期顺序迭代，不得跳日
- 期末余额允许为负值，不做截断
- 中间计算保持高精度浮点，最终输出统一四舍五入到 2 位小数
- **输出前自检**：输出行数 = `forecast_days`，日期连续无缺失；每行满足期末 = 期初 + 回款 - 付款（浮点误差 ≤ 0.01）；第 N+1 天期初 = 第 N 天期末

### 步骤 5：预警分级

读取 `liquidity-alert-policy.md` 执行，核心规则如下：

**参数合法性校验**（分级前必须执行）：
- 要求 `red_threshold >= 0`、`safety_reserve >= 0`、`red_threshold <= safety_reserve`
- 若明显为录入反转（如 red=300、safety=200），尝试自动交换并记录告警
- 否则回退默认阈值（red=200、safety=300）并记录

**分级与标识规则**：

- 先按阈值判定风险区间（高风险区/关注区/安全区），再映射为输出等级标识
- **等级标识优先级**：用户明确指定 > 任务输入规则文件指定 > 本 Skill 默认示例标识
- 以下“红色预警/黄色预警/正常”仅为默认示例，不作为固定输出值

| 风险区间（内部语义） | 条件 | 默认示例等级标识 | 风险说明 |
|----------|------|----------|----------|
| 高风险区 | `期末余额 < red_threshold` | 红色预警 | 资金安全垫不足，需立即控制支出并评估融资/调拨方案 |
| 关注区 | `red_threshold <= 期末余额 < safety_reserve` | 黄色预警 | 接近安全阈值，建议加强回款管理并优化付款节奏 |
| 安全区 | `期末余额 >= safety_reserve` | 正常 | 处于安全区间，保持常规监控即可 |

### 步骤 6：结果输出

> **输出格式优先级：用户明确指定的列名、列顺序、金额精度、文件格式、编码等，以用户指定为准；未指定时，按以下默认规范输出。**

输出日级预测结果，至少包含以下语义字段（默认列名）：

- 日期
- 期初余额（万元）
- 回款额（万元）
- 付款额（万元）
- 期末余额（万元）
- 预警等级（等级标识以用户要求为准；未指定时使用默认示例标识）

## 3. 异常处理规则

- 缺少必需语义字段：先尝试同义字段映射；仍失败则返回明确缺失项
- 日期不可解析：标记异常行并剔除，记录剔除数量
- 金额为空或非法：按 0 处理并记录异常计数
- 金额单位不一致：执行统一换算并在结果说明中声明换算口径
- 输入表为空：返回结构化空结果并附失败原因，不输出误导性预测

## 4. 配套 Reference 设计

### 4.1 目录建议

```text
reference/
  cashflow-forecast-rules.md
  liquidity-alert-policy.md
  cashflow-field-mapping.md
  cashflow-unit-normalization.md
  cashflow-exception-playbook.md
```

### 4.2 文件职责

- `cashflow-forecast-rules.md`：沉淀余额汇总口径、账户状态筛选逻辑、收支按日聚合规则、滚动预测公式与计算完整性校验
- `liquidity-alert-policy.md`：沉淀预警分级口径、等级标识映射优先级、阈值合法性校验、冲突回退策略与风险说明文本
- `cashflow-field-mapping.md`：沉淀跨任务通用字段同义词映射与必需字段检查规则
- `cashflow-unit-normalization.md`：沉淀金额单位识别、换算顺序、精度处理规则
- `cashflow-exception-playbook.md`：沉淀常见脏数据与缺失值处理优先级、兜底策略

### 4.3 调用路由规则

执行过程中按需读取 reference：

1. 执行余额汇总、收支聚合、滚动预测时，读取 `cashflow-forecast-rules.md`
2. 执行预警分级或需校验阈值参数时，读取 `liquidity-alert-policy.md`
3. 字段名不标准或多种命名混用时，读取 `cashflow-field-mapping.md`
4. 金额单位不明确或混用时，读取 `cashflow-unit-normalization.md`
5. 出现空表、空值、非法日期、异常金额时，读取 `cashflow-exception-playbook.md`

## 5. 质量检查清单

执行完成后应自检：

- 是否完成字段语义映射且关键字段齐全
- 是否统一了日期格式与金额单位
- 是否按预测天数输出连续日期结果
- 是否每行满足期初、流入、流出、期末的计算闭环
- 是否按阈值规则正确分级预警
- 是否记录并说明了关键异常处理
- 是否在需要时正确加载 reference 并遵守优先级规则
