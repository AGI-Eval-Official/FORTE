# 公司行政管理制度规范 Skill

不同公司、不同行业的管理规范内容差异很大，本 Skill 提供通用的制度撰写维度和质量标准，并在用户未明确说明的情况下给出合理的示例内容，让生成结果开箱即用。用户说明的内容和要求优先。

**用户优先原则**：当本 Skill 的默认规则、流程、示例或模板与用户在当前对话中的明确要求发生冲突时，一律以用户要求为准；本 Skill 的内容仅作为兜底参考，不得覆盖用户意图。

---

## 第一步：理解用户意图，直接生成

从用户的描述中提取以下信息，有什么用什么，缺什么给出合理默认值，**不向用户反复确认**：

- **制度主题**：从用户描述中直接识别，模糊时取最合理的宽泛理解（如"安全制度"→生成综合安全管理制度）
- **公司名称**：用户未提供则全文用「公司」二字代替，不用占位符打断阅读
- **适用范围**：用户未提及则默认为全体员工
- **行业/规模背景**：用户未提及则生成适用于大多数普通企业的通用版本
- **文档风格**：用户未指定则根据描述中透露的公司背景判断；完全无线索时默认正式严谨风

对于具体数字、部门名称、人员职务等细节，优先使用合理的示例值（如"行政部""人力资源部""警告、记过、解除劳动合同"），让文档读起来完整流畅；若某些数字确实因公司而异（如具体罚款金额、薪资数字），则用【XX】占位，并在文末提示用户填写。

---

## 第二步：判断制度类型与核心维度

根据用户的描述，判断制度所属的管理类别，并据此确定本份制度需要覆盖的**核心管理维度**。

### 四大类别及对应核心维度

**行为规范类**（如员工行为准则、着装规范、保密制度、社交媒体使用等）
核心维度：行为边界的界定 / 正面要求与禁止行为的列举 / 对外形象与对内协作 / 信息与财产保护 / 违规的认定标准

**安全卫生类**（如消防安全、应急预案、环境卫生、职业健康等）
核心维度：风险识别与预防措施 / 日常检查与维护责任 / 事件处置流程与响应链 / 培训与演练要求 / 事故记录与复盘机制
法规底线：此类制度存在强制性法规要求，内容不得低于法定标准，须在总则中引用对应法规依据。互联网企业适用的具体法规条款已整理在 `references/safety-regulations.md`，生成制度时直接参照该文件中的条款内容，无需自行检索。

**设备设施类**（如IT设备、车辆、会议室、宿舍、办公用品等）
核心维度：资产台账与归属管理 / 申请与审批流程 / 使用规范与禁止行为 / 维护与报修责任 / 归还与损坏赔偿

**其他行政类**（如访客管理、门禁、快递收发、档案管理等）
核心维度：准入或操作的授权机制 / 登记与记录要求 / 流程节点与责任人 / 例外情况处理 / 安全与保密要求

> 如果用户的需求跨多个类别（如"综合员工手册"），则合并相关类别的维度，按逻辑顺序组织章节。

## 第三步：生成制度内容

### 通用文档结构

以下结构仅作为默认示例，用于用户未指定输出格式时兜底。若用户对结构、章节命名、篇幅或呈现形式有明确要求，必须按用户要求输出。

所有制度文档遵循以下结构，章节数量和命名根据制度内容灵活调整：

```
【封面信息】
核心字段（所有公司适用）：文件名称 / 发布日期
可选字段（有文件管理体系的企业按需保留）：文件编号 / 版本号 / 编制-审核-批准部门

第一章  总则
  ├─ 制定目的（为什么要有这份制度）
  ├─ 适用范围（约束哪些人、哪些场景）
  └─ 管理职责（谁来执行、谁来监督）

第二章  [核心内容]（根据制度类型展开，1~3章均可）
  └─ 将第二步中识别的核心维度展开为具体条款

第N章  违规认定与处理
  ├─ 违规行为的分级（轻微 / 一般 / 严重）
  └─ 对应处理措施（须与劳动法规保持一致）

附则
  ├─ 解释权归属
  ├─ 生效日期
  └─ 修订说明
```

### 两种文档风格

根据用户选择或公司背景，采用对应风格：

**正式严谨风**（大型企业、国企、上市公司）
- 使用"第X条"逐条编号，语言严谨规范
- 总则引用相关法律法规依据
- 责任表述完整，含追责条款
- 示例写法："第一条 为规范员工日常行为，维护公司正常工作秩序，依据《中华人民共和国劳动法》及本公司实际情况，特制定本制度。"

**简洁实用风**（中小企业、创业公司、互联网团队）
- 使用一、二、三分级，语言简洁直白；文档结构不强制套用"第一章/第二章"格式，可直接用"一、二、三"或"（一）（二）（三）"展开内容
- 突出"做什么/不做什么"，易于理解和执行
- 可适当加入说明性语言解释规定背后的原因
- 示例写法："一、为什么要有这份规定：让大家清楚基本的行为边界，减少不必要的误解和摩擦。"

用户未明确说明时，根据其描述的公司背景自行判断风格。

### 示例内容参考

以下为各类制度的典型条款示例，生成时可参考此类写法，确保内容具体可执行，而非空泛套话：

**行为规范类示例条款**
> 第X条 员工在工作时间内须保持职业形象，着装整洁得体。工作日不得穿着拖鞋、背心、破洞裤等休闲装进入办公区；有客户来访时须着正装或商务休闲装。

**安全卫生类示例条款**
> 第X条 各部门每月至少开展一次消防安全自查，检查内容包括：消防通道是否畅通、灭火器是否在有效期内、电气线路是否存在老化或私拉乱接情况。发现隐患须在24小时内上报行政部并记录在案。

**设备设施类示例条款**
> 第X条 员工申请使用公司车辆须提前一个工作日填写《用车申请单》，经部门负责人审批后提交行政部统一安排。用车结束后须填写行驶记录，油量不足二分之一时须及时加油，费用凭发票报销。

**其他行政类示例条款**
> 第X条 外来访客须在前台登记姓名、所属单位、来访事由及被访人信息，由被访员工确认后方可进入办公区。访客须全程佩戴访客证，不得单独进入非公开区域。

## 制度内容质量标准

生成内容时，始终对照以下标准自检：

**可执行性**：每条规定必须明确"谁、在什么情况下、做什么或不做什么"。含糊的表述（如"员工应保持职业形象"）是不合格的，应改为具体可操作的描述（如"工作时间内须着职业装，不得穿着拖鞋、背心进入办公区"）。

**完整性**：目的、适用范围、核心条款、违规处理、附则五个要素缺一不可。

**合规性**：制度条款分两个层次处理。第一层是法规强制要求，凡涉及消防、安全生产、职业健康等有国家法规约束的内容，须以对应法规为底线，不得低于法定标准，条款中明确引用法规名称；第二层是企业内部自由裁量，如考勤规则、着装要求、设备使用等内容，企业可自行制定，但涉及惩戒处分的表述须与《劳动法》《劳动合同法》保持一致，避免"直接辞退""无偿加班"等可能违法的表述，应改为"视情节轻重予以警告、记过或依法解除劳动合同"。

**适配性**：制度内容应贴合用户描述的行业和公司规模，不生搬与该公司场景无关的条款。制造业与互联网公司、50人小公司与5000人大公司，制度侧重点和严格程度都应有所不同。文档结构同样允许按规模裁剪，小公司可省略文件编号、版本号等管理体系字段，章节也可适当合并精简，不必强求与大公司格式一致。

**简洁性**：避免堆砌套话和重复表述，每一条款都应有实际意义。

## 常见模糊需求处理

当用户描述模糊时，主动理解意图而非反复追问：

- "帮我写个安全制度" → 直接生成综合性安全管理制度，涵盖消防、人身、信息安全等主要维度
- "制度要全面一点" → 理解为需要覆盖该类别的所有核心维度，章节可适当拆细
- "参考XXX公司的风格" → 理解为对标该类型企业的正式程度，不必照搬
- "内容不用太多" → 精简为核心条款，每章不超过5条，去掉铺陈和说明性语言
- 用户只给了制度名称没有其他信息 → 直接生成通用完整版本，公司名称用"公司"代替，因公司而异的具体数字用【XX】占位

生成的制度文档可直接作为参考版本使用，正式对外发布前建议用户结合本公司实际情况修改，并经法务或HR审核。涉及薪资数字、具体罚款金额等因公司而异的敏感内容，一律用【XX】占位，由用户自行填写。
