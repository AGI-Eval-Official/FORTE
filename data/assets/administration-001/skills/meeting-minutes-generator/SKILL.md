# 会议纪要生成与整理技能

## 技能概述

本技能帮助将会议录音、聊天记录、速记笔记等原始材料整理成规范的书面白话会议纪要。不规定固定格式模板，但约束各领域通用的核心维度和主要内容，确保输出的会议纪要专业、完整、可执行。

## 核心能力

1. **口语转书面语**：将口语化表达转化为规范的专业书面语
2. **结构化整理**：识别并归类会议中的各类信息（决议、待办、讨论等）
3. **必要元素保障**：确保输出包含所有必要的核心项和内容元素
4. **多格式输出**：支持 .docx、.pdf、.md 等格式输出
5. **文本清洗**：自动识别并清洗质量较差的转写文本

## 会议场景支持

本技能覆盖以下常见会议场景：

| 会议类型 | 说明 | 特殊关注点 |
|---------|------|-----------|
| 周会 | 周工作总结与计划同步 | 工作进展、下周计划、需协调事项 |
| 复盘会议 | 项目/季度/月度复盘 | 成果总结、问题分析、改进措施 |
| 例会 | 月度/季度例会 | 周期性总结、既定议题、常规决议 |
| 专题会议 | 特定主题深入讨论 | 主题聚焦、深度讨论、专项决议 |

## 会议纪要核心内容要求

### 必须包含的核心项

一份完整的会议纪要**必须**包含以下核心项：

| 核心项名称 | 说明 | 示例 |
|---------|------|------|
| 会议名称 | 本次会议的主题 | 2024年11月度项目进展会议 |
| 会议时间 | 具体日期和时间 | 2024年11月22日 10:00-11:30 |
| 会议地点 | 会议召开地点 | 3号会议室 / 线上会议 |
| 主持人 | 会议主持/主导人 | 张三（项目经理） |
| 参会人员 | 所有参与人员及角色 | 详见参会人员列表 |
| 议题与讨论内容 | 各议题的讨论要点 | 按议题分条整理 |
| 会议摘要 | 会议核心内容精炼概述 | 5句话及以内概括要点 |
| 会议决议/结论 | 最终决定和结论 | 明确的行动项和决策 |
| 记录人 | 会议记录整理人员 | 李四 |
| 待办事项 | 后续行动项 | 含责任人、截止时间 |

### 会议摘要要求

**会议摘要**是对整场会议核心内容的精炼概括，需满足以下要求：

- **句数要求**：5句话及以内
- **内容要求**：
  - 涵盖会议主要议题
  - 概括关键结论或决议
  - 体现核心待办事项
- **风格要求**：精炼简洁，避免冗余描述

**示例**：

> 原会议讨论了A项目上线时间调整、B功能需求变更、C技术方案选型三个议题。经讨论决定：项目延期至12月15日上线，B功能采用方案二实施。主要待办为小王完成方案细化报告，小李跟进供应商对接。

### 会议决议/结论的内容要素

会议决议/结论**必须**包含以下内容要素：

| 要素名称 | 说明 |
|---------|------|
| 具体内容 | 决议的具体事项或行动项 |
| 责任人 | 负责执行该决议的人员 |
| 截止时间 | 完成的期限 |

### 待办事项格式要求

**待办事项**必须逐条分点列出，每条待办包含以下要素：

- **待办内容**：具体任务描述
- **责任人**：负责执行的人员
- **截止时间**：任务完成的期限
- **优先级**（可选）：高/中/低

**输出格式示例**：

```
待办事项：
1. 完成A模块接口文档编写
   - 责任人：张三
   - 截止时间：2024年11月25日
   - 优先级：高

2. 协调供应商确认交付日期
   - 责任人：李四
   - 截止时间：2024年11月28日
   - 优先级：中

3. 组织下一轮技术评审会议
   - 责任人：王五
   - 截止时间：2024年12月1日
   - 优先级：低
```

## 输入方式支持

本技能支持多种输入方式，用户可根据实际情况选择：

### 1. 文本输入

直接提供会议记录的文字内容，支持以下格式：

| 格式类型 | 说明 | 处理方式 |
|---------|------|---------|
| 纯文本 | 直接粘贴的文字记录 | 直接解析处理 |
| Markdown | 带格式的.md文本 | 解析结构化内容 |
| 聊天记录 | 微信/钉钉/飞书等聊天导出 | 提取关键对话信息 |
| 格式化的文本 | 表格、分点列表等结构化文本 | 识别并提取信息 |

### 2. 文件输入

支持读取以下格式的会议原始文件：

| 文件格式 | 说明 | 处理说明 |
|---------|------|---------|
| .txt | 纯文本文件 | 直接读取解析 |
| .md | Markdown文件 | 解析结构化内容 |
| .docx | Word文档 | 提取文字和表格内容 |
| .pdf | PDF文档 | 提取文字内容（不含加密文件） |
| .csv | 表格数据 | 解析表格形式的会议数据 |

### 3. 音视频输入（需用户自行转写）

**注意**：本技能不直接处理音视频文件。用户需先使用其他工具完成语音转文字（STT），再将转写后的文本提交处理。

支持的转写工具输出格式：
- 语音助手/录音APP的自动转写文本
- 第三方STT服务的输出文件
- 人工转写的会议记录

**建议流程**：
1. 用户使用录音工具录制会议
2. 使用转写工具或服务将音频转为文字
3. 将转写文本提交给本技能生成纪要

## SOP 流程

### 第一步：读取并分析原始材料

1. 读取用户提供的会议原始材料（录音整理、聊天记录、速记等）
2. 识别材料中的关键信息：
   - 会议时间、参与者
   - 讨论的议题
   - 形成的结论和决议
   - 分配的任务和待办

### 第二步：文本质量评估与清洗

**重要**：在开始处理前，需评估输入文本的质量。

#### 文本质量判断标准

| 质量等级 | 特征 | 处理方式 |
|---------|------|---------|
| 良好 | 语句通顺、错别字少、结构清晰 | 直接进入下一步 |
| 较差 | 错别字多、断句混乱、口语化严重、逻辑不清 | 需要文本清洗 |

#### 文本清洗步骤

当文本质量较差时，执行以下清洗操作：

1. **错别字修正**：识别并修正常见错别字和同音字错误
2. **断句整理**：修正混乱的断句，重新组织语句结构
3. **口语过滤**：删除无意义的语气词（"那个"、"嗯"、"就是说"等）
4. **重复去除**：合并重复表达的内容
5. **逻辑梳理**：重新组织混乱的逻辑顺序
6. **信息补全**：对明显遗漏的信息用 [待确认] 标记

**清洗后需确认**：将清洗后的文本呈现给用户确认，必要时进行调整。

### 第三步：提取核心信息

从原始材料中提取以下信息：

```
- 会议基本信息（时间、人物、地点）
- 议题列表（每个议题的讨论内容）
- 决议事项（每个议题的结论）
- 待办任务（责任人、截止时间）
```

### 第四步：生成书面语会议纪要

**语言风格要求**：
- 使用书面语，避免口语化表达
- 客观陈述，避免主观评价
- 明确具体，避免模糊表述
- 逻辑清晰，层次分明

**内容组织原则**：
- 按议题分块组织
- 每个议题包含：背景、讨论、结论
- 决议部分需明确**具体内容**、**责任人**及**截止时间**
- 待办事项按截止日期升序排列
- 会议摘要精炼简洁，5句话及以内概括核心

### 第五步：信息安全检查

**敏感信息提醒**：在生成纪要时，需注意以下敏感信息：

| 敏感类型 | 示例 | 处理方式 |
|---------|------|---------|
| 薪资信息 | 工资、奖金、绩效奖金、股票等 | [敏感信息] 标记，提醒用户确认是否保留 |
| 人事变动 | 人员调整、离职、调岗、晋升等 | [待确认] 标记，由用户确认后处理 |
| 未公开数据 | 财务数据、业务机密、战略规划等 | [保密信息] 标记，提醒用户确认 |
| 个人隐私 | 家庭情况、健康信息、联系方式等 | 删除或脱敏处理 |

### 第六步：输出最终文档

根据用户要求的格式输出最终会议纪要。

## 口语转书面语示例

| 口语表达 | 书面语表达 |
|---------|-----------|
| "那个、嗯、就是说" | 删除或转为正式表述 |
| "我觉得/我认为" | "会议认为" / "经讨论" |
| "能不能/行不行" | "是否" / "建议" |
| "必须得/一定得" | "应" / "须" |
| "大概/可能/差不多" | 具体数值或明确表述 |

**转换示例**：

口语：
> "那个，关于年终大促活动页的事儿，李工说他后端写完了，但小刘的切图还没给他。我就让小刘必须在24号之前把图给李工。"

书面语：
> "**议题一：年终大促活动页上线进度**
> 
> **讨论内容**：研发部李工汇报后端接口开发已完成，等待UI设计小刘提供切图进行联调。因品牌升级需求，页面配色需由红色调整为紫色。
> 
> **会议决议**：UI设计小刘须于2024年11月24日前完成紫色版本切图并交付研发部；研发部李工须于2024年11月26日前完成联调测试并实现上线。"

## 内容忠实性原则

**核心原则**：会议纪要应忠实于原始内容，不添加会议中未讨论的信息。

### 允许的操作

- 将口语表述转化为书面语
- 整理和重组信息使其更清晰
- 补充必要的连接词使逻辑通顺
- 用 [待确认] 标记不确定的信息
- 标注缺失的必填项（如会议地点、主持人等）

### 禁止的操作

- 添加原始材料中不存在的新信息
- 推测或假设未明确表达的内容
- 对未达成共识的事项作出结论性陈述
- 删减重要讨论内容（除非用户明确要求）

### [待确认] 标记使用场景

以下情况应使用 [待确认] 标记：

| 场景 | 示例 |
|-----|------|
| 时间不明确 | "预计 [待确认] 完成" |
| 人员不明确 | "责任人： [待确认]" |
| 结论未明确 | "关于XX问题，结论 [待确认]" |
| 信息有歧义 | "此处表述存在歧义 [待确认]" |

## 特殊处理规则

### 必填核心项检查

输出前必须确认以下核心项均有内容：

- [ ] 会议名称
- [ ] 会议时间
- [ ] 会议地点（如未明确，标注 [待确认]）
- [ ] 主持人（如未明确，标注 [待确认]）
- [ ] 参会人员
- [ ] 议题与讨论内容
- [ ] 会议摘要（5句话及以内）
- [ ] 会议决议/结论（含：具体内容、责任人、截止时间）
- [ ] 记录人（如未明确，标注 [待确认]）
- [ ] 待办事项（逐条列出，含责任人和截止时间）

### 核心项缺失处理

- **会议地点未明确**：标注 [待确认]，或根据上下文推断后标注
- **主持人未明确**：根据讨论主导者推断，或标注 [待确认]
- **参会人员不完整**：已提及的人员均需列出，未提及的可标注 [待确认]
- **待办责任人未明确**：标注 [待确认]
- **截止时间未明确**：标注 [待确认]

### 敏感信息提醒

在处理涉及以下信息的会议内容时，需特别提醒用户：

> ⚠️ **信息安全提醒**：检测到以下敏感信息，请确认处理方式：
> 
> - 薪资/财务相关数据
> - 人事变动信息
> - 未公开的业务数据
> - 其他需保密的内容
